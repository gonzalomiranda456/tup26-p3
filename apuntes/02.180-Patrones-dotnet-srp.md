# Patrones en .NET para aplicar SRP

SRP no significa "una clase, un método" — significa que una clase tiene **una razón clara para cambiar**. Cuando una clase cambia por motivos distintos (cambió la BD, cambió la UI, cambió el proveedor externo), mezcla responsabilidades.

**Regla de oro:** SRP detecta el problema; el patrón ayuda a ubicar la responsabilidad que sobra.

## Mapa rápido

| Si cambia...                                | Patrón útil                   |
| ------------------------------------------- | ----------------------------- |
| La creación/selección de objetos            | Dependency Injection, Factory |
| La construcción compleja de un objeto       | Builder                       |
| La forma de guardar o buscar datos          | Repository, Unit of Work      |
| Un criterio de búsqueda combinable          | Specification                 |
| Un algoritmo o política de negocio          | Strategy                      |
| El comportamiento según el estado interno   | State                         |
| Una acción que debe ejecutarse o deshacerse | Command                       |
| Una cadena de validaciones o filtros        | Chain of Responsibility       |
| Una operación sobre una jerarquía de tipos  | Visitor                       |
| Una API externa                             | Adapter                       |
| Logging, cache, retry, métricas             | Decorator, Proxy              |
| La coordinación entre varios objetos        | Mediator, Facade              |
| Reacciones a eventos                        | Observer / Events             |
| Configuración externa                       | Options Pattern               |
| Responsabilidades HTTP transversales        | Middleware, Filters           |
| Contrato externo vs modelo interno          | DTO + Mapper                  |

## Dependency Injection

Sin DI, el servicio crea sus dependencias (queda acoplado a JSON, SQL, etc.). Con DI, recibe una abstracción:

```csharp
public sealed class ContactoService(IContactoRepository repository) {
    public void Agregar(Contacto contacto) {
        var contactos = repository.ObtenerTodos();
        contactos.Add(contacto);
        repository.GuardarTodos(contactos);
    }
}

// Registro:
builder.Services.AddSingleton<IContactoRepository>(new JsonContactoRepository("contactos.json"));
builder.Services.AddScoped<ContactoService>();
```

## Repository y Unit of Work

```csharp
public interface IContactoRepository {
    List<Contacto> ObtenerTodos();
    void GuardarTodos(List<Contacto> contactos);
}

public sealed class JsonContactoRepository(string path) : IContactoRepository {
    public List<Contacto> ObtenerTodos() {
        if (!File.Exists(path)) return [];
        return JsonSerializer.Deserialize<List<Contacto>>(File.ReadAllText(path)) ?? [];
    }
    public void GuardarTodos(List<Contacto> contactos) =>
        File.WriteAllText(path, JsonSerializer.Serialize(contactos, new JsonSerializerOptions { WriteIndented = true }));
}

// Para tests: InMemoryContactoRepository : IContactoRepository { /* usa List<Contacto> */ }
```

En EF Core, `DbContext` cumple el rol de Repository + Unit of Work: rastrea cambios y los confirma con `SaveChangesAsync()`.

## Strategy

Reemplaza `if/else if` sobre el mismo campo con clases intercambiables:

```csharp
public interface IExportadorContactos { string Formato { get; } void Exportar(IEnumerable<Contacto> c, string path); }
public sealed class CsvExportadorContactos : IExportadorContactos { public string Formato => "csv"; /* ... */ }
public sealed class JsonExportadorContactos : IExportadorContactos { public string Formato => "json"; /* ... */ }

public sealed class ExportacionService(IEnumerable<IExportadorContactos> exportadores) {
    public void Exportar(string formato, IEnumerable<Contacto> c, string path) =>
        exportadores.Single(e => e.Formato == formato).Exportar(c, path);
}
```

## Decorator

Agrega comportamiento transversal (logging, cache) sin modificar la clase original:

```csharp
public sealed class LoggingContactoRepository(IContactoRepository inner, ILogger logger) : IContactoRepository {
    public List<Contacto> ObtenerTodos() {
        logger.LogInformation("Leyendo contactos.");
        return inner.ObtenerTodos();
    }
    public void GuardarTodos(List<Contacto> c) {
        logger.LogInformation("Guardando {n} contactos.", c.Count);
        inner.GuardarTodos(c);
    }
}
```

## State

Elimina `if/else` sobre el mismo campo `estado`:

```csharp
public abstract class EstadoPedido {
    public abstract EstadoPedido Pagar();
    public abstract EstadoPedido Cancelar();
}
public sealed class Pendiente : EstadoPedido {
    public override EstadoPedido Pagar() => new Pagado();
    public override EstadoPedido Cancelar() => new Cancelado();
}
public sealed class Pedido {
    public EstadoPedido Estado { get; private set; } = new Pendiente();
    public void Pagar() => Estado = Estado.Pagar();
}
```

## Options Pattern (configuración tipada)

```csharp
public sealed class AgendaOptions { public string ArchivoContactos { get; set; } = "contactos.json"; }

builder.Services.Configure<AgendaOptions>(builder.Configuration.GetSection("Agenda"));

public sealed class JsonContactoRepository(IOptions<AgendaOptions> options) : IContactoRepository {
    private readonly AgendaOptions opts = options.Value;
}
```

## DTO + Mapper

Separa el contrato externo del modelo interno:

```csharp
public sealed class ContactoDto { public int Id; public string NombreCompleto = ""; public string Email = ""; }

public static class ContactoMapper {
    public static ContactoDto ToDto(Contacto c) => new() {
        Id = c.Id, NombreCompleto = $"{c.Nombre} {c.Apellido}", Email = c.Email
    };
}
```

## Reglas prácticas

Preguntas para detectar responsabilidades mezcladas: ¿La clase cambia si cambia la BD? ¿Si cambia la UI? ¿Si cambia el proveedor externo? Si varias respuestas son sí, hay mezcla.

Nombres que revelan responsabilidad: `JsonContactoRepository`, `CsvExportadorContactos`, `WhatsAppNotificador`. Nombres que esconden mezcla: `ContactoManager`, `AgendaHelper`, `Utils`, `GeneralService`.

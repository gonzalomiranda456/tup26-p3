# Evaluar expresiones matemáticas

En este tutorial vamos a construir un mini lenguaje de expresiones aritméticas en C#.

La idea no es solo obtener un resultado numérico, sino recorrer una secuencia de diseños cada vez más potentes:

1. **tokenizar** una expresión;
2. **interpretarla** directamente con un parser descendente recursivo;
3. **compilarla** a un árbol;
4. implementar un **AST simple** en donde cada nodo tenga `Evaluar` y `ToString`;
5. rehacer ese AST usando el **patrón Visitante**.

La expresión de ejemplo será algo como:

```csharp
2 * x + 3
```

Si `x = 4`, el resultado debe ser `11`.

---

## Paso 1 — Tokenizar la expresión

Antes de evaluar, conviene transformar el texto en piezas más manejables. A esas piezas las llamamos **tokens**.

Para la expresión:

```text
2 * x + 3
```

los tokens son:

```text
2   *   x   +   3
```

Podemos modelarlos así:

```csharp
enum TipoToken {
    Numero,
    Variable,
    Suma,
    Resta,
    Multiplicacion,
    Division,
    ParentesisAbierto,
    ParentesisCerrado,
    Final
}

record Token(TipoToken Tipo, string Texto = "");
```

Y ahora escribimos el tokenizador:

```csharp
List<Token> Tokenizar(string expresion) {
    int posicion = 0;
    List<Token> tokens = [];

    char Consumir() => expresion[posicion++];

    while (posicion < expresion.Length) {
        char c = expresion[posicion];

        if (char.IsWhiteSpace(c)) {
            Consumir();
            continue;
        }

        if (char.IsDigit(c)) {
            string numero = "";
            while (posicion < expresion.Length && char.IsDigit(expresion[posicion])) {
                numero += Consumir();
            }

            tokens.Add(new Token(TipoToken.Numero, numero));
            continue;
        }

        if (char.IsLetter(c)) {
            string variable = "";
            while (posicion < expresion.Length && char.IsLetterOrDigit(expresion[posicion])) {
                variable += Consumir();
            }

            tokens.Add(new Token(TipoToken.Variable, variable));
            continue;
        }

        switch (c) {
            case '+': tokens.Add(new Token(TipoToken.Suma)); break;
            case '-': tokens.Add(new Token(TipoToken.Resta)); break;
            case '*': tokens.Add(new Token(TipoToken.Multiplicacion)); break;
            case '/': tokens.Add(new Token(TipoToken.Division)); break;
            case '(': tokens.Add(new Token(TipoToken.ParentesisAbierto)); break;
            case ')': tokens.Add(new Token(TipoToken.ParentesisCerrado)); break;
            default:
                throw new Exception($"Carácter inesperado: {c}");
        }

        Consumir();
    }

    tokens.Add(new Token(TipoToken.Final));
    return tokens;
}
```

Con esto pasamos de una cadena de caracteres a una lista de objetos con significado.

## Paso 2 — Interpretar directamente con un parser descendente recursivo

Ahora definimos una gramática simple:

```text
expresion = termino { ('+' | '-') termino }
termino   = unario  { ('*' | '/') unario }
unario    = ('+' | '-') unario | factor
factor    = numero | variable | '(' expresion ')'
```

La gracia de esta gramática es que ya incorpora la precedencia:

- `*` y `/` se resuelven dentro de `termino`;
- los operadores unarios `+` y `-` se resuelven dentro de `unario`;
- `+` y `-` binarios se resuelven dentro de `expresion`.

Podemos traducirla casi línea por línea a código:

```csharp
int Evaluar(string expresion, Dictionary<string, int> variables) {
    return Evaluar(Tokenizar(expresion), variables);
}

int Evaluar(List<Token> tokens, Dictionary<string, int> variables) {
    int posicion = 0;
    Token actual = tokens[posicion];

    void Consumir() {
        posicion++;
        actual = tokens[posicion];
    }

    bool Coincide(TipoToken tipo) {
        if (actual.Tipo != tipo) { return false; }
        Consumir();
        return true;
    }

    Token Anterior() => tokens[posicion - 1];

    int Expresion() {
        int valor = Termino();

        while (true) {
            if (Coincide(TipoToken.Suma)) {
                valor += Termino();
            } else if (Coincide(TipoToken.Resta)) {
                valor -= Termino();
            } else {
                break;
            }
        }

        return valor;
    }

    int Termino() {
        int valor = Unario();

        while (true) {
            if (Coincide(TipoToken.Multiplicacion)) {
                valor *= Unario();
            } else if (Coincide(TipoToken.Division)) {
                int divisor = Unario();
                if (divisor == 0) { throw new DivideByZeroException("División por cero."); }
                valor /= divisor;
            } else {
                break;
            }
        }

        return valor;
    }

    int Unario() {
        if (Coincide(TipoToken.Suma)) {
            return Unario();
        }

        if (Coincide(TipoToken.Resta)) {
            return -Unario();
        }

        return Factor();
    }

    int Factor() {
        if (Coincide(TipoToken.Numero)) {
            return int.Parse(Anterior().Texto);
        }

        if (Coincide(TipoToken.Variable)) {
            string nombre = Anterior().Texto;
            if (!variables.TryGetValue(nombre, out int valor)) {
                throw new Exception($"La variable '{nombre}' no tiene valor.");
            }
            return valor;
        }

        if (Coincide(TipoToken.ParentesisAbierto)) {
            int valor = Expresion();
            if (!Coincide(TipoToken.ParentesisCerrado)) {
                throw new Exception("Se esperaba un paréntesis cerrado.");
            }
            return valor;
        }

        throw new Exception("Se esperaba un número, variable o paréntesis.");
    }

    int resultado = Expresion();

    if (actual.Tipo != TipoToken.Final) {
        throw new Exception("Se esperaba el final de la expresión.");
    }

    return resultado;
}
```

Este diseño funciona, pero tiene una limitación importante: **parsea y evalúa al mismo tiempo**.

Eso quiere decir que si más adelante queremos:

- imprimir la expresión con paréntesis;
- optimizarla;
- evaluarla muchas veces con distintos valores;

tenemos que volver a leer todo el texto.

## Paso 3 — Separar el análisis del cálculo

La solución clásica es construir un **AST** (*Abstract Syntax Tree*), o árbol de sintaxis abstracta.

Por ejemplo, para:

```text
2 * x + 3
```

el árbol sería conceptualmente algo así:

```text
      +
     / \
    *   3
   / \
  2   x
```

Ese árbol ya no es texto: es una estructura de objetos.

Una vez construido, podemos recorrerlo de distintas maneras.

---

## Paso 4 — AST simple con `Evaluar` y `ToString`

En esta primera versión del AST, cada nodo se hace cargo de dos responsabilidades:

- **evaluarse** a sí mismo;
- **convertirse a texto** con `ToString()`.

Es una solución muy buena para empezar porque es concreta, fácil de leer y bastante natural.

### 4.1 La clase base

```csharp
abstract record Nodo {
    public abstract int Evaluar(Dictionary<string, int> variables);
}
```

### 4.2 Nodos hoja

```csharp
record Constante(int Valor) : Nodo {
    public override int Evaluar(Dictionary<string, int> variables) => Valor;
    public override string ToString() => Valor.ToString();
}

record Variable(string Nombre) : Nodo {
    public override int Evaluar(Dictionary<string, int> variables) {
        if (!variables.TryGetValue(Nombre, out int valor)) {
            throw new Exception($"La variable '{Nombre}' no tiene valor.");
        }
        return valor;
    }

    public override string ToString() => Nombre;
}
```

### 4.3 Nodos unarios

Podemos aplicar exactamente la misma idea que en los binarios: extraer la estructura común a una clase base abstracta.

```csharp
abstract record Unario(Nodo Operando) : Nodo {
    protected abstract string Operador { get; }
    protected abstract int Aplicar(int operando);

    public override int Evaluar(Dictionary<string, int> variables) {
        int operando = Operando.Evaluar(variables);
        return Aplicar(operando);
    }

    public override string ToString() => $"({Operador}{Operando})";
}

record Positivo(Nodo Operando) : Unario(Operando) {
    protected override string Operador => "+";
    protected override int Aplicar(int operando) => operando;
}

record Negativo(Nodo Operando) : Unario(Operando) {
    protected override string Operador => "-";
    protected override int Aplicar(int operando) => -operando;
}
```

### 4.4 Nodos binarios

En realidad estamos usando el mismo patrón dos veces:

- `Unario`  factoriza la lógica común de los operadores con un operando;
- `Binario` factoriza la lógica común de los operadores con dos operandos.

Para los binarios hacemos lo mismo.

```csharp
abstract record Binario(Nodo Izquierdo, Nodo Derecho) : Nodo {
    protected abstract string Operador { get; }
    protected abstract int Aplicar(int izquierdo, int derecho);

    public override int Evaluar(Dictionary<string, int> variables) {
        int izquierdo = Izquierdo.Evaluar(variables);
        int derecho   = Derecho.Evaluar(variables);
        return Aplicar(izquierdo, derecho);
    }

    public override string ToString() => $"({Izquierdo} {Operador} {Derecho})";
}

record Suma(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    protected override string Operador => "+";
    protected override int Aplicar(int izquierdo, int derecho) => izquierdo + derecho;
}

record Resta(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    protected override string Operador => "-";
    protected override int Aplicar(int izquierdo, int derecho) => izquierdo - derecho;
}

record Multiplicacion(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    protected override string Operador => "*";
    protected override int Aplicar(int izquierdo, int derecho) => izquierdo * derecho;
}

record Division(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    protected override string Operador => "/";

    protected override int Aplicar(int izquierdo, int derecho) {
        if (derecho == 0) {
            throw new DivideByZeroException("División por cero.");
        }

        return izquierdo / derecho;
    }
}
```

### 4.5 Compilar a este AST

El parser es muy parecido al del paso 2. La diferencia es que ahora devuelve un `Nodo` en lugar de un `int`.

```csharp
Nodo Compilar(string expresion) {
    return Compilar(Tokenizar(expresion));
}

Nodo Compilar(List<Token> tokens) {
    int posicion = 0;
    Token actual = tokens[posicion];

    void Consumir() {
        posicion++;
        actual = tokens[posicion];
    }

    bool Coincide(TipoToken tipo) {
        if (actual.Tipo != tipo) { return false; }
        Consumir();
        return true;
    }

    Token Anterior() => tokens[posicion - 1];

    Nodo Expresion() {
        Nodo valor = Termino();

        while (true) {
            if (Coincide(TipoToken.Suma)) {
                valor = new Suma(valor, Termino());
            } else if (Coincide(TipoToken.Resta)) {
                valor = new Resta(valor, Termino());
            } else {
                break;
            }
        }

        return valor;
    }

    Nodo Termino() {
        Nodo valor = Unario();

        while (true) {
            if (Coincide(TipoToken.Multiplicacion)) {
                valor = new Multiplicacion(valor, Unario());
            } else if (Coincide(TipoToken.Division)) {
                valor = new Division(valor, Unario());
            } else {
                break;
            }
        }

        return valor;
    }

    Nodo Unario() {
        if (Coincide(TipoToken.Suma)) {
            return new Positivo(Unario());
        }

        if (Coincide(TipoToken.Resta)) {
            return new Negativo(Unario());
        }

        return Factor();
    }

    Nodo Factor() {
        if (Coincide(TipoToken.Numero)) {
            return new Constante(int.Parse(Anterior().Texto));
        }

        if (Coincide(TipoToken.Variable)) {
            return new Variable(Anterior().Texto);
        }

        if (Coincide(TipoToken.ParentesisAbierto)) {
            Nodo valor = Expresion();
            if (!Coincide(TipoToken.ParentesisCerrado)) {
                throw new Exception("Se esperaba un paréntesis cerrado.");
            }

            return valor;
        }

        throw new Exception("Se esperaba un número, variable o paréntesis.");
    }

    Nodo resultado = Expresion();

    if (actual.Tipo != TipoToken.Final) {
        throw new Exception("Se esperaba el final de la expresión.");
    }

    return resultado;
}
```

### 4.6 Uso del AST simple

```csharp
var variables = new Dictionary<string, int> {
    ["x"] = 4
};

Nodo arbol = Compilar("2 * x + 3");

Console.WriteLine(arbol);                    // ((2 * x) + 3)
Console.WriteLine(arbol.Evaluar(variables)); // 11

variables["x"] = 10;
Console.WriteLine(arbol.Evaluar(variables)); // 23
```

Esta versión ya es mucho mejor que el intérprete puro:

- compilamos una vez;
- evaluamos varias veces;
- y además podemos imprimir la estructura con `ToString()`.

Pero ahora aparece una nueva tensión de diseño.

Si mañana queremos agregar:

- `Optimizar()`
- `ContarNodos()`
- `GenerarCodigo()`

habría que seguir agregando métodos a todos los nodos.

Ahí entra el patrón Visitante.

## Paso 5 — Reimplementar el AST con el patrón Visitante

La idea del patrón Visitante es esta:

- los nodos dejan de saber **cómo** evaluarse o imprimirse;
- los nodos solo saben **aceptar** un visitante;
- las operaciones viven afuera, en clases separadas.

### 5.1 La interfaz del visitante

```csharp
interface IVisitante<T> {
    T VisitarConstante(Constante nodo);
    T VisitarVariable(Variable nodo);
    T VisitarUnario(Unario nodo);
    T VisitarBinario(Binario nodo);
}
```

### 5.2 Nueva clase base del AST

```csharp
abstract record Nodo {
    public abstract T Aceptar<T>(IVisitante<T> visitante);
}
```

### 5.3 Nodos hoja y nodos compuestos

```csharp
record Constante(int Valor) : Nodo {
    public override T Aceptar<T>(IVisitante<T> visitante) {
        return visitante.VisitarConstante(this);
    }
}

record Variable(string Nombre) : Nodo {
    public override T Aceptar<T>(IVisitante<T> visitante) {
        return visitante.VisitarVariable(this);
    }
}

abstract record Unario(Nodo Operando) : Nodo {
    public abstract string Operador { get; }
    public abstract int Aplicar(int operando);

    public override T Aceptar<T>(IVisitante<T> visitante) {
        return visitante.VisitarUnario(this);
    }
}

record Positivo(Nodo Operando) : Unario(Operando) {
    public override string Operador => "+";
    public override int Aplicar(int operando) => operando;
}

record Negativo(Nodo Operando) : Unario(Operando) {
    public override string Operador => "-";
    public override int Aplicar(int operando) => -operando;
}

abstract record Binario(Nodo Izquierdo, Nodo Derecho) : Nodo {
    public abstract string Operador { get; }
    public abstract int Aplicar(int izquierdo, int derecho);

    public override T Aceptar<T>(IVisitante<T> visitante) {
        return visitante.VisitarBinario(this);
    }
}

record Suma(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    public override string Operador => "+";
    public override int Aplicar(int izquierdo, int derecho) => izquierdo + derecho;
}

record Resta(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    public override string Operador => "-";
    public override int Aplicar(int izquierdo, int derecho) => izquierdo - derecho;
}

record Multiplicacion(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    public override string Operador => "*";
    public override int Aplicar(int izquierdo, int derecho) => izquierdo * derecho;
}

record Division(Nodo Izquierdo, Nodo Derecho) : Binario(Izquierdo, Derecho) {
    public override string Operador => "/";

    public override int Aplicar(int izquierdo, int derecho) {
        if (derecho == 0) {
            throw new DivideByZeroException("División por cero.");
        }

        return izquierdo / derecho;
    }
}
```

Notá que el compilador prácticamente no cambia: sigue construyendo `Constante`, `Variable`, `Suma`, `Division` y compañía.

Lo que cambia es **dónde vive la lógica**.

### 5.4 Visitante para evaluar

```csharp
class EvaluadorVisitante(Dictionary<string, int> variables) : IVisitante<int> {
    public int VisitarConstante(Constante nodo) => nodo.Valor;

    public int VisitarVariable(Variable nodo) {
        if (!variables.TryGetValue(nodo.Nombre, out int valor)) {
            throw new Exception($"La variable '{nodo.Nombre}' no tiene valor.");
        }

        return valor;
    }

    public int VisitarUnario(Unario nodo) {
        int operando = nodo.Operando.Aceptar(this);
        return nodo.Aplicar(operando);
    }

    public int VisitarBinario(Binario nodo) {
        int izquierdo = nodo.Izquierdo.Aceptar(this);
        int derecho = nodo.Derecho.Aceptar(this);
        return nodo.Aplicar(izquierdo, derecho);
    }
}
```

### 5.5 Visitante para convertir a texto

```csharp
class ImpresorVisitante : IVisitante<string> {
    public string VisitarConstante(Constante nodo) => nodo.Valor.ToString();

    public string VisitarVariable(Variable nodo) => nodo.Nombre;

    public string VisitarUnario(Unario nodo) {
        return $"({nodo.Operador}{nodo.Operando.Aceptar(this)})";
    }

    public string VisitarBinario(Binario nodo) {
        string izquierdo = nodo.Izquierdo.Aceptar(this);
        string derecho = nodo.Derecho.Aceptar(this);
        return $"({izquierdo} {nodo.Operador} {derecho})";
    }
}
```

### 5.6 Uso con visitantes

```csharp
var variables = new Dictionary<string, int> {
    ["x"] = 4
};

Nodo arbol = Compilar("2 * x + 3");

string texto = arbol.Aceptar(new ImpresorVisitante());
int valor = arbol.Aceptar(new EvaluadorVisitante(variables));

Console.WriteLine(texto); // ((2 * x) + 3)
Console.WriteLine(valor); // 11
```

Y si después queremos agregar otra operación, por ejemplo contar nodos, no tocamos ninguna clase del AST:

```csharp
class ContadorVisitante : IVisitante<int> {
    public int VisitarConstante(Constante nodo) => 1;
    public int VisitarVariable(Variable nodo) => 1;

    public int VisitarUnario(Unario nodo) {
        return 1 + nodo.Operando.Aceptar(this);
    }

    public int VisitarBinario(Binario nodo) {
        return 1 + nodo.Izquierdo.Aceptar(this) + nodo.Derecho.Aceptar(this);
    }
}
```

### 5.7 Visitante para capturar variables

Otro ejemplo útil es un visitante que recorra el árbol y devuelva el conjunto de variables que aparecen en la expresión.

```csharp
class VariablesVisitante : IVisitante<HashSet<string>> {
    public HashSet<string> VisitarConstante(Constante nodo) => [];

    public HashSet<string> VisitarVariable(Variable nodo) => [nodo.Nombre];

    public HashSet<string> VisitarUnario(Unario nodo) {
        return nodo.Operando.Aceptar(this);
    }

    public HashSet<string> VisitarBinario(Binario nodo) {
        HashSet<string> variables = nodo.Izquierdo.Aceptar(this);
        variables.UnionWith(nodo.Derecho.Aceptar(this));
        return variables;
    }
}
```

Uso:

```csharp
Nodo arbol = Compilar("x * (y + 3) - z");
HashSet<string> variables = arbol.Aceptar(new VariablesVisitante());

Console.WriteLine(string.Join(", ", variables)); // x, y, z
```

Este visitante no evalúa ni imprime: simplemente inspecciona la estructura para extraer información.

### 5.8 Visitante para optimizar expresiones

También podemos crear un visitante que devuelva un **nuevo árbol optimizado**. El caso más simple es el de las subexpresiones constantes.

Por ejemplo:

```text
(2 + 3) * x
```

puede transformarse en:

```text
5 * x
```

Eso se conoce como **constant folding**.

```csharp
class OptimizadorVisitante : IVisitante<Nodo> {
    public Nodo VisitarConstante(Constante nodo) => nodo;

    public Nodo VisitarVariable(Variable nodo) => nodo;

    public Nodo VisitarUnario(Unario nodo) {
        Nodo operando = nodo.Operando.Aceptar(this);

        if (operando is Constante constante) {
            return new Constante(nodo.Aplicar(constante.Valor));
        }

        return nodo switch {
            Positivo => new Positivo(operando),
            Negativo => new Negativo(operando),
            _ => throw new InvalidOperationException("Operador unario desconocido.")
        };
    }

    public Nodo VisitarBinario(Binario nodo) {
        Nodo izquierdo = nodo.Izquierdo.Aceptar(this);
        Nodo derecho = nodo.Derecho.Aceptar(this);

        if (izquierdo is Constante izq && derecho is Constante der) {
            return new Constante(nodo.Aplicar(izq.Valor, der.Valor));
        }

        return nodo switch {
            Suma => new Suma(izquierdo, derecho),
            Resta => new Resta(izquierdo, derecho),
            Multiplicacion => new Multiplicacion(izquierdo, derecho),
            Division => new Division(izquierdo, derecho),
            _ => throw new InvalidOperationException("Operador binario desconocido.")
        };
    }
}
```

Uso:

```csharp
Nodo original = Compilar("(2 + 3) * (x + 0)");
Nodo optimizado = original.Aceptar(new OptimizadorVisitante());

string antes = original.Aceptar(new ImpresorVisitante());
string despues = optimizado.Aceptar(new ImpresorVisitante());

Console.WriteLine(antes);   // ((2 + 3) * (x + 0))
Console.WriteLine(despues); // (5 * (x + 0))
```

En esta versión el optimizador solo pliega subexpresiones constantes. Más adelante podríamos agregar reglas como:

- `x + 0  -> x`
- `x * 1  -> x`
- `x * 0  -> 0`

Ese es el gran beneficio del patrón: **nuevas operaciones sin tocar los nodos**.

## Comparación entre el paso 4 y el paso 5

### AST simple con `Evaluar` y `ToString`

Ventajas:

- más directo;
- más fácil de entender al principio;
- cada nodo concentra su comportamiento principal.

Desventajas:

- si agregamos muchas operaciones nuevas, la jerarquía empieza a cargarse de métodos.

### AST con Visitantes

Ventajas:

- las operaciones viven separadas del árbol;
- es fácil agregar nuevas operaciones;
- el AST queda más estable.

Desventajas:

- hay más infraestructura;
- al principio cuesta más ver el flujo.

## Resumen final

En este tutorial construimos cinco niveles de diseño:

1. **tokenizamos** el texto;
2. **interpretamos** la expresión con parser descendente recursivo;
3. vimos por qué conviene separar el cálculo del análisis;
4. implementamos un **AST simple** con `Evaluar` y `ToString`;
5. rehicimos ese diseño con el **patrón Visitante**.

La idea importante no es memorizar cada línea, sino entender esta evolución:

**texto → tokens → parser → árbol → operaciones sobre el árbol**

Ese mismo recorrido aparece una y otra vez en compiladores, intérpretes, motores de reglas, plantillas y planillas.

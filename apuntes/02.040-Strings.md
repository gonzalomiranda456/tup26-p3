# Strings en C#

`string` representa texto. Es un tipo de referencia con comportamiento especial: `==` compara contenido (no identidad) y los strings son **inmutables**.

```csharp
string nombre = "Ada";
```

## Formas de escribir un string

```csharp
// Concatenación
string a = "Hola, " + nombre;

// Interpolación: la forma más legible
string b = $"{nombre} tiene {edad} años, el doble es {edad * 2}";

// Verbatim: sin escapar barras, con saltos de línea literales
string c = @"C:\Users\Ada\Documentos";
string d = @"Línea 1
Línea 2";

// Combinada: verbatim + interpolación
string e = $@"C:\Users\Ada\{carpeta}";
```

### Raw string literals (`"""`)

Permiten escribir JSON, código u otro texto con comillas y saltos de línea sin escapar nada:

```csharp
string json = """
    {
        "nombre": "Ada",
        "edad": 20
    }
    """;
```

Con `$$"""` se interpolan valores usando `{{` y `}}`. Las llaves simples quedan como texto literal:

```csharp
string json = $$"""
    {
        "nombre": "{{nombre}}",
        "edad": {{edad}}
    }
    """;
```

## Inmutabilidad

Una vez creado, un string no se modifica. Cuando parece que se cambia, en realidad se crea uno nuevo:

```csharp
string nombre = "Ana";
nombre = nombre + " María"; // crea un string nuevo, "Ana" sigue en memoria
```

### `StringBuilder`

En bucles con muchas concatenaciones, cada `+` genera un objeto. Para esos casos `StringBuilder` trabaja sobre un buffer y solo materializa el string al terminar:

```csharp
var sb = new StringBuilder();

for (int i = 0; i < 1000; i++) {
    sb.Append($"Línea {i}\n");
}

string resultado = sb.ToString();
```

Para uso cotidiano la concatenación e interpolación alcanzan. `StringBuilder` es una optimización para construcción intensiva.

## Conversión automática

Cuando se necesita texto, C# llama a `ToString()` (heredado de `object`):

```csharp
int numero = 42;
string mensaje = "El número es: " + numero; // llama numero.ToString()
```

En clases propias se sobreescribe para definir la representación textual:

```csharp
class Punto {
    public int X { get; set; }
    public int Y { get; set; }

    public override string ToString() => $"({this.X}, {this.Y})";
}

var p = new Punto { X = 3, Y = 7 };
Console.WriteLine($"El punto es {p}"); // El punto es (3, 7)
```

## Comparación

`==` compara contenido. Para ordenar, `CompareTo` devuelve negativo, cero o positivo:

```csharp
"banana".CompareTo("manzana"); // negativo: "banana" va antes
```

`string.Compare` y `Equals` aceptan un `StringComparison` para controlar la comparación:

```csharp
frase.Equals("hola", StringComparison.OrdinalIgnoreCase);
frase.Contains("ESTO", StringComparison.OrdinalIgnoreCase);
frase.StartsWith("ho",  StringComparison.OrdinalIgnoreCase);
```

Las dos opciones más usadas:

- `OrdinalIgnoreCase`: para comparaciones técnicas (nombres de archivo, claves, comandos).
- `CurrentCultureIgnoreCase`: para texto visible al usuario, donde importan las reglas del idioma.

```csharp
var config = new Dictionary<string, string>(StringComparer.OrdinalIgnoreCase);
config["Nombre"] = "Ada";
config["nombre"]; // "Ada"
```

## Null, vacío y blanco

```csharp
string? nulo  = null;
string  vacio = "";
string  blanco = "   ";

string.IsNullOrEmpty(nulo);        // true
string.IsNullOrEmpty(vacio);       // true
string.IsNullOrWhiteSpace(blanco); // true
```

`IsNullOrWhiteSpace` suele ser más útil: un string de solo espacios casi siempre representa "sin contenido".

## Métodos comunes

| Método | Qué hace |
|---|---|
| `Length` | Cantidad de unidades `char` |
| `ToUpper()` / `ToLower()` | Cambia mayúsculas/minúsculas |
| `Trim()` / `TrimStart()` / `TrimEnd()` | Elimina espacios en los extremos |
| `Contains(sub)` | Verifica si contiene un fragmento |
| `StartsWith(sub)` / `EndsWith(sub)` | Verifica inicio o final |
| `IndexOf(sub)` | Posición de la primera aparición; `-1` si no existe |
| `Replace(viejo, nuevo)` | Reemplaza todas las apariciones |
| `Split(sep)` | Divide en array usando un separador |
| `string.Join(sep, array)` | Une un array en un string |
| `Substring(inicio, largo)` o `s[i..j]` | Extrae una parte |
| `PadLeft(n, c)` / `PadRight(n, c)` | Rellena hasta longitud `n` |
| `string.IsNullOrEmpty(s)` | `true` si es `null` o `""` |
| `string.IsNullOrWhiteSpace(s)` | `true` si es `null`, `""` o solo espacios |

```csharp
"informe_final.pdf".EndsWith(".pdf");       // true
"hola mundo".Replace("mundo", "Ada");       // "hola Ada"
"Ana,Luis,Pedro".Split(',');                // { "Ana", "Luis", "Pedro" }
string.Join(", ", new[] {"a","b","c"});     // "a, b, c"
"42".PadLeft(6, '0');                       // "000042"
```

## Unicode y emojis

`char` en C# es 16 bits. Los emojis y símbolos fuera del rango básico usan dos `char` (par sustituto):

```csharp
string emoji = "😀";
emoji.Length; // 2 — no 1
```

Para iterar correctamente usar `Rune`:

```csharp
foreach (Rune r in "A😀B".EnumerateRunes())
    Console.Write($"[{r}] ");
// [A] [😀] [B]
```

Para contar caracteres visibles usar `StringInfo`:

```csharp
new StringInfo("Hola 😀").LengthInTextElements; // 6
```

## Resumen

| Necesidad | Herramienta |
|---|---|
| Texto con valores intercalados | interpolación `$"..."` |
| Multilínea o rutas Windows | verbatim `@"..."` |
| JSON / código con comillas | raw literal `"""..."""` |
| Concatenar mucho en bucle | `StringBuilder` |
| Igualdad ignorando mayúsculas | `StringComparison.OrdinalIgnoreCase` |
| Comparar respetando el idioma | `StringComparison.CurrentCultureIgnoreCase` |
| Diccionario con claves insensibles | `StringComparer.OrdinalIgnoreCase` |
| Detectar "sin contenido real" | `string.IsNullOrWhiteSpace` |
| Iterar emojis correctamente | `Rune` / `EnumerateRunes()` |
| Contar caracteres visibles | `StringInfo.LengthInTextElements` |

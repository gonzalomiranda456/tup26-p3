# Tutorial de CLI en C#: `sortx` en un solo archivo

## ¿Qué vamos a construir?

Una herramienta llamada **`sortx`** que lee un archivo de texto delimitado (CSV, TSV, PSV, etc.), ordena sus filas según uno o más criterios, y escribe el resultado.

```
sortx [input [output]] [-b|--by campo[:tipo[:orden]]]...
      [-i|--input input] [-o|--output output]
      [-d|--delimiter delimitador]
      [-nh|--no-header] [-h|--help]
```

**Ejemplos:**

```bash
# Ordenar por apellido (alfabético, ascendente)
sortx empleados.csv -b apellido

# Ordenar por salario descendente y guardar en archivo
sortx empleados.csv resultado.csv -b salario:num:desc

# Múltiples criterios: por departamento, dentro por salario desc
sortx empleados.csv -b departamento -b salario:num:desc

# TSV sin encabezado, ordenar por la segunda columna
sortx datos.tsv -d "\t" -nh -b 1:alpha:asc

# Encadenando con redirección
cat empleados.csv | sortx -b apellido > ordenado.csv

# Ayuda
sortx --help
```

**Resultado esperado** (con `-b departamento -b salario:num:desc`):

```text
nombre,apellido,edad,salario,departamento
Ana,Álvarez,28,72000,Diseño
Beatriz,Blanco,25,65000,Diseño
Carlos,Castro,38,95000,Gerencia
Diana,Díaz,42,120000,Gerencia
Eduardo,Espinosa,31,88000,Ingeniería
Fernanda,Fernández,35,85000,Ingeniería
```

---

## La hoja de ruta del programa

Cada paso recibe datos, los transforma y los pasa al siguiente:

```
1. ParseArgs       → leer la configuración desde los argumentos
2. ReadInput       → leer el texto desde el archivo o stdin
3. ParseDelimited  → convertir el texto en filas (lista de diccionarios)
4. SortRows        → ordenar las filas según los criterios
5. Serialize       → convertir las filas ordenadas de vuelta a texto
6. WriteOutput     → escribir en el archivo de salida o stdout
```

Cada paso es independiente: recibe su entrada, produce su salida, no sabe nada de los demás. El punto de entrada se limita a invocarlos en orden, dentro de un `try/catch`.

---

## Paso 0: el archivo

A diferencia de un proyecto tradicional con `.csproj`, en C# 14 / .NET 10 se pueden ejecutar archivos `.cs` sueltos como **file-based programs**. Toda la herramienta vive en un único `sortx.cs`.

```bash
# Crear el archivo
touch sortx.cs
code sortx.cs
```

Para ejecutar mientras se desarrolla:

```bash
dotnet run sortx.cs -- empleados.csv -b apellido
```

El `--` separa los argumentos para `dotnet run` de los argumentos para `sortx`. Sin él, `dotnet` los interpretaría como propios.

---

## Paso 1: el modelo de datos — dos `record`

Antes de parsear los argumentos, definimos *qué* vamos a parsear. En C# tenemos los **records**: tipos inmutables, compactos, perfectos para representar datos que no cambian una vez construidos.

```csharp
record SortField(string Name, bool Numeric, bool Descending);

record AppConfig(
    string?         InputFile,
    string?         OutputFile,
    string          Delimiter,
    bool            NoHeader,
    List<SortField> SortFields
);
```

`SortField` representa **un** criterio de ordenación: el nombre del campo (o el índice como string si no hay encabezado), si la comparación es numérica, y si es descendente. `AppConfig` agrupa toda la configuración de una corrida: archivos, delimitador, flag de encabezado, y la lista de criterios.

> **Dos records, no uno:** `AppConfig` contiene una `List<SortField>` porque un solo comando puede pedir ordenar por varios campos a la vez (`-b departamento -b salario:num:desc`). Modelar cada criterio como su propio tipo hace que el código que los procesa quede mucho más legible.

> **`record` vs `class`:** Un `record` es inmutable por defecto. Sus propiedades se asignan al construirlo y no cambian después. Para representar configuración, es la herramienta exacta.

## Paso 2: parsear los argumentos

La sintaxis que debemos soportar:

| Opción larga    | Corta | Descripción                                         |
|-----------------|-------|-----------------------------------------------------|
| `--help`        | `-h`  | Muestra la ayuda y sale                             |
| `--by`          | `-b`  | Criterio de ordenación, repetible                   |
| `--input`       | `-i`  | Archivo de entrada                                  |
| `--output`      | `-o`  | Archivo de salida                                   |
| `--delimiter`   | `-d`  | Carácter delimitador (default: `,`)                 |
| `--no-header`   | `-nh` | El archivo no tiene fila de encabezado              |

La estrategia: recorremos `args` con un índice `i` y consumimos uno o dos tokens según el caso.

```csharp
AppConfig ParseArgs(string[] args) {
    string?      inputFile  = null;
    string?      outputFile = null;
    string       delimiter  = ",";
    bool         noHeader   = false;
    var          sortFields = new List<SortField>();
    var          positional = new List<string>();

    int i = 0;
    while (i < args.Length) {
        string arg = args[i];
        switch (arg) {
            case "--help" or "-h":
                PrintHelp();
                Environment.Exit(0);
                break;

            case "--input" or "-i":
                inputFile = Next(args, ref i, arg);
                break;

            case "--output" or "-o":
                outputFile = Next(args, ref i, arg);
                break;

            case "--by" or "-b":
                sortFields.Add(ParseSortField(Next(args, ref i, arg)));
                break;

            case "--delimiter" or "-d":
                delimiter = NormalizeDelimiter(Next(args, ref i, arg));
                break;

            case "--no-header" or "-nh":
                noHeader = true;
                i++;
                break;

            case var option when option.StartsWith('-'):
                throw new Exception($"Opción desconocida: '{option}'. Use --help.");

            case var value:
                positional.Add(value);
                i++;
                break;
        }
    }

    if (positional.Count >= 1 && inputFile  == null) inputFile  = positional[0];
    if (positional.Count >= 2 && outputFile == null) outputFile = positional[1];
    if (positional.Count >  2)
        throw new Exception("Demasiados argumentos. Máximo: [input] [output]");
    if (sortFields.Count == 0)
        throw new Exception("Debe indicar al menos un criterio con -b|--by.");

    return new AppConfig(inputFile, outputFile, delimiter, noHeader, sortFields);
}
```

`Next` es una función auxiliar que avanza el índice y retorna el siguiente token:

```csharp
string Next(string[] args, ref int i, string option) {
    i++;
    if (i >= args.Length)
        throw new Exception($"'{option}' requiere un valor.");
    return args[i++];
}
```

> **`ref int i`:** Pasamos el índice por referencia para que `Next` pueda avanzarlo. Cuando retorna, `i` ya apunta al token siguiente al valor consumido. Así evitamos manejar el índice en dos lugares.

> **`-nh` no consume valor:** A diferencia de las otras opciones, `-nh` es un *flag* booleano: no toma argumento. Por eso solo hace `i++` para pasar al siguiente token, sin llamar a `Next`.

### Paso 2a: parsear cada `--by`

Cada valor de `--by` tiene la forma `campo[:tipo[:orden]]`:

| Expresión          | Significado                                   |
|--------------------|-----------------------------------------------|
| `apellido`         | por `apellido`, alfabético, ascendente        |
| `salario:num`      | por `salario`, numérico, ascendente           |
| `salario:num:desc` | por `salario`, numérico, descendente          |
| `2:num:asc`        | por columna 2, numérico, ascendente (sin enc) |

Lo separamos por `:` y vamos asignando defaults para las partes ausentes:

```csharp
SortField ParseSortField(string spec) {
    string[] parts = spec.Split(':');

    string name       = parts[0];
    bool   numeric    = false;
    bool   descending = false;

    if (parts.Length >= 2) {
        switch (parts[1]) {
            case "num":   numeric = true;  break;
            case "alpha": numeric = false; break;
            default:
                throw new Exception($"Tipo inválido en '{spec}': '{parts[1]}'. Use 'num' o 'alpha'.");
        }
    }

    if (parts.Length >= 3) {
        switch (parts[2]) {
            case "asc":  descending = false; break;
            case "desc": descending = true;  break;
            default:
                throw new Exception($"Orden inválido en '{spec}': '{parts[2]}'. Use 'asc' o 'desc'.");
        }
    }

    if (parts.Length > 3)
        throw new Exception($"Especificación inválida: '{spec}'.");

    if (string.IsNullOrEmpty(name))
        throw new Exception($"Falta el nombre del campo en '{spec}'.");

    return new SortField(name, numeric, descending);
}
```

> **Validar al parsear, no al usar:** Si el usuario escribe `salario:numerico:descendente`, queremos que el error se vea acá, junto al argumento que lo causó. Si lo dejáramos pasar y fallara más tarde durante la ordenación, el mensaje sería confuso.

### Paso 2b: normalizar el delimitador

El usuario puede pasar `-d "\t"` para tabulación. Pero según la shell, `"\t"` puede llegar como dos caracteres literales (`\` y `t`) en lugar del carácter tab real. Lo convertimos uniformemente:

```csharp
string NormalizeDelimiter(string raw) {
    return raw switch {
        "\\t" => "\t",
        "\\n" => "\n",
        "\\r" => "\r",
        ""    => throw new Exception("El delimitador no puede estar vacío."),
        _     => raw
    };
}
```

> **`switch` como expresión:** En C# moderno, `switch` puede devolver un valor directamente sin `break` ni asignación intermedia. Es ideal para mapeos cortos como este.

## Paso 3: leer y escribir

Dos funciones simétricas. Una lee desde archivo o `stdin`; la otra escribe hacia archivo o `stdout`. No cambian respecto a cualquier otra herramienta CLI.

```csharp
string ReadInput(string? filePath) {
    if (filePath == null)
        return Console.In.ReadToEnd();

    if (!File.Exists(filePath))
        throw new FileNotFoundException($"Archivo no encontrado: '{filePath}'");

    return File.ReadAllText(filePath);
}

void WriteOutput(string? filePath, string content) {
    if (filePath == null) {
        Console.Write(content);
        return;
    }

    File.WriteAllText(filePath, content);
}
```

> **Redirección de consola:** `Console.In.ReadToEnd()` y `Console.Write()` funcionan tanto en terminal como con redirección (`|`, `<`, `>`). El sistema operativo se encarga; el código no cambia.

## Paso 4: parsear el archivo delimitado

Convertimos el texto en una lista de diccionarios. Cada diccionario representa una fila: `campo → valor`. El delimitador y la presencia o no de encabezado se reciben como parámetros.

```csharp
(string[] headers, List<Dictionary<string, string>> rows) ParseDelimited(
    string content,
    string delimiter,
    bool   noHeader)
{
    string[] split(string line) {
        string[] parts = line.Split(delimiter);
        for (int j = 0; j < parts.Length; j++)
            parts[j] = parts[j].Trim();
        return parts;
    }

    string[] lines = content.Split('\n');

    // Eliminar líneas vacías al final
    int last = lines.Length - 1;
    while (last >= 0 && lines[last].Trim() == "") last--;

    if (last < 0)
        return ([], []);

    // Determinar headers según haya o no encabezado
    string[] headers;
    int      firstDataLine;

    if (noHeader) {
        // Sin encabezado: usamos índices "0", "1", "2"... como nombres
        string[] firstRow = split(lines[0]);
        headers = new string[firstRow.Length];
        for (int j = 0; j < firstRow.Length; j++)
            headers[j] = j.ToString();
        firstDataLine = 0;
    } else {
        headers       = split(lines[0]);
        firstDataLine = 1;
    }

    // Parsear cada fila como diccionario campo → valor
    var rows = new List<Dictionary<string, string>>();
    for (int i = firstDataLine; i <= last; i++) {
        string line = lines[i];
        if (line == "") continue;

        string[] values = split(line);
        var row = new Dictionary<string, string>();
        for (int col = 0; col < headers.Length; col++)
            row[headers[col]] = col < values.Length ? values[col] : "";

        rows.Add(row);
    }

    return (headers, rows);
}
```

> **Sin encabezado → headers numéricos:** El truco está en que, cuando no hay encabezado, generamos un array `["0", "1", "2", ...]` que cumple el mismo rol. Así, todo el resto del programa puede seguir usando `row["apellido"]` o `row["2"]` sin distinguir el caso. La diferencia se concentra en este único punto.

> **Tuple return `(T1, T2)`:** `ParseDelimited` produce dos cosas: los encabezados y las filas. El tipo de retorno tupla expresa eso directamente, sin necesidad de crear una clase contenedora. Lo desestructuramos con `var (headers, rows) = ParseDelimited(...)`.

> **Función local `split`:** Es una función adentro de otra. Se define una sola vez y se reutiliza en este `ParseDelimited` sin contaminar el resto del archivo. Aplica también el delimitador correcto sin tener que pasarlo cada vez.

## Paso 5: ordenar las filas

Acá vive la lógica del programa. Recibe las filas y la lista de criterios; devuelve las filas reordenadas. El truco es que cuando hay varios criterios, el segundo solo desempata cuando el primero da igual, y así sucesivamente.

```csharp
List<Dictionary<string, string>> SortRows(
    List<Dictionary<string, string>> rows,
    string[]                         headers,
    List<SortField>                  sortFields)
{
    // 1. Validar que todos los campos pedidos existan
    foreach (var field in sortFields) {
        bool found = false;
        for (int i = 0; i < headers.Length; i++)
            if (headers[i] == field.Name) { found = true; break; }

        if (!found)
            throw new Exception($"Campo no encontrado: '{field.Name}'");
    }

    if (rows.Count == 0 || sortFields.Count == 0)
        return rows;

    // 2. Construir el ordenamiento encadenando criterios
    IOrderedEnumerable<Dictionary<string, string>>? ordered = null;

    foreach (var field in sortFields) {
        IComparable Key(Dictionary<string, string> row) {
            string value = row[field.Name];
            if (field.Numeric)
                return double.TryParse(value, out double n) ? n : double.NaN;
            return value;
        }

        if (ordered == null)
            ordered = field.Descending
                ? rows.OrderByDescending(Key)
                : rows.OrderBy(Key);
        else
            ordered = field.Descending
                ? ordered.ThenByDescending(Key)
                : ordered.ThenBy(Key);
    }

    return ordered!.ToList();
}
```

> **Validar antes de ordenar:** Si un campo del `--by` no existe entre los encabezados, queremos enterarnos *antes* de empezar a procesar. Validamos los campos primero, después ordenamos. Es el mismo principio que en `sumx`: primero verificar, después trabajar.

> **`OrderBy` vs `ThenBy`:** El primer criterio usa `OrderBy`/`OrderByDescending` y arranca el ordenamiento desde cero. Los criterios siguientes usan `ThenBy`/`ThenByDescending`, que solo se aplican como desempate. Por eso el primer criterio se trata distinto: si lo tratáramos igual que los siguientes, perdería su rol de criterio principal.

> **`IComparable` como tipo de la clave:** `OrderBy` puede ordenar por cualquier tipo que sepa compararse consigo mismo. `string` y `double` ambos implementan `IComparable`. Devolver `IComparable` desde la función local nos permite alternar entre comparación numérica y alfabética sin duplicar el código de ordenación.

> **Función local que captura `field`:** `Key` está definida adentro del `foreach` y usa `field`. C# captura cada `field` correctamente para cada iteración (desde C# 5), así que cada ordenamiento usa su propio criterio sin interferir con los demás.

> **`double.NaN` para celdas no numéricas:** Si una celda esperada como número no se puede parsear, devolvemos `NaN`. .NET lo ordena consistentemente al final y evita que el programa explote por una celda mal formada.

## Paso 6: serializar de vuelta a texto

Lo opuesto exacto a `ParseDelimited`: tomamos las filas (ya ordenadas) y las convertimos a texto delimitado, respetando si hay encabezado y cuál es el delimitador.

```csharp
string Serialize(
    string[]                         headers,
    List<Dictionary<string, string>> rows,
    string                           delimiter,
    bool                             noHeader)
{
    var sb = new StringBuilder();

    if (!noHeader)
        sb.AppendLine(string.Join(delimiter, headers));

    foreach (var row in rows) {
        string[] values = new string[headers.Length];
        for (int i = 0; i < headers.Length; i++)
            values[i] = row.TryGetValue(headers[i], out string? v) ? v : "";

        sb.AppendLine(string.Join(delimiter, values));
    }

    return sb.ToString();
}
```

> **`StringBuilder` para construir texto:** Concatenar muchas veces con `+` crea un string nuevo cada vez (los strings son inmutables). Para archivos con miles de filas, eso degrada el rendimiento. `StringBuilder` mantiene un buffer interno y lo va completando, mucho más eficiente.

> **El orden de las columnas se respeta gracias a `headers`:** Aunque las filas son diccionarios (sin orden definido), `headers` es un array y conserva el orden original de las columnas. Por eso recorremos `headers` para reconstruir cada fila.

> **Si no había encabezado, no lo hay en la salida:** Cuando `noHeader` es `true`, los `headers` que generamos (`"0", "1", ...`) eran un detalle interno, no parte de los datos. La salida arranca directo con la primera fila.

## Paso 8: el punto de entrada

Conectamos las piezas. En C# *file-based programs*, las instrucciones al nivel raíz del archivo son el punto de entrada — no hace falta clase ni método `Main`.

```csharp
try {
    AppConfig config    = ParseArgs(args);                                         // Paso 1
    string    text      = ReadInput(config.InputFile);                             // Paso 2
    var (headers, rows) = ParseDelimited(text, config.Delimiter, config.NoHeader); // Paso 3
    var       sorted    = SortRows(rows, headers, config.SortFields);              // Paso 4
    string    output    = Serialize(headers, sorted, config.Delimiter, config.NoHeader); // Paso 5
    WriteOutput(config.OutputFile, output);                                        // Paso 6
} catch (Exception ex) {
    Console.Error.WriteLine($"Error: {ex.Message}");
    Console.Error.WriteLine("Use --help para ver las opciones.");
    Environment.Exit(1);
}
```

Seis líneas, seis pasos. El punto de entrada no toma decisiones: solo conecta funciones.

> **`Console.Error` para errores:** Si el usuario redirige la salida (`sortx ... > ordenado.csv`), los mensajes de error van a la terminal y no contaminan el archivo. Siempre usar `Console.Error` para mensajes que no son el output del programa.

> **`Environment.Exit(1)`:** Convención universal de Unix: código `0` significa éxito, cualquier otro valor significa error. Permite encadenar el programa en pipelines y scripts (`sortx ... && echo OK`).

## El archivo completo

```csharp
// ┌─────────────────────────────────────────────────────────────────────────┐
// │  sortx — ordena las filas de un archivo de texto delimitado             │
// │  Uso: sortx [input [output]] [-b|--by campo[:tipo[:orden]]]...          │
// │             [-i|--input input] [-o|--output output]                     │
// │             [-d|--delimiter delim] [-nh|--no-header] [-h|--help]        │
// └─────────────────────────────────────────────────────────────────────────┘

using System.Text;

record SortField(string Name, bool Numeric, bool Descending);

record AppConfig(
    string?         InputFile,
    string?         OutputFile,
    string          Delimiter,
    bool            NoHeader,
    List<SortField> SortFields
);

// ── Punto de entrada ──────────────────────────────────────────────────────────

try {
    AppConfig config    = ParseArgs(args);
    string    text      = ReadInput(config.InputFile);
    var (headers, rows) = ParseDelimited(text, config.Delimiter, config.NoHeader);
    var       sorted    = SortRows(rows, headers, config.SortFields);
    string    output    = Serialize(headers, sorted, config.Delimiter, config.NoHeader);
    WriteOutput(config.OutputFile, output);
} catch (Exception ex) {
    Console.Error.WriteLine($"Error: {ex.Message}");
    Console.Error.WriteLine("Use --help para ver las opciones.");
    Environment.Exit(1);
}

// ── Paso 1: Parsear argumentos ────────────────────────────────────────────────

AppConfig ParseArgs(string[] args) {
    string? inputFile  = null;
    string? outputFile = null;
    string  delimiter  = ",";
    bool    noHeader   = false;
    var     sortFields = new List<SortField>();
    var     positional = new List<string>();

    int i = 0;
    while (i < args.Length) {
        string arg = args[i];
        switch (arg) {
            case "--help" or "-h":
                PrintHelp();
                Environment.Exit(0);
                break;

            case "--input" or "-i":
                inputFile = Next(args, ref i, arg);
                break;

            case "--output" or "-o":
                outputFile = Next(args, ref i, arg);
                break;

            case "--by" or "-b":
                sortFields.Add(ParseSortField(Next(args, ref i, arg)));
                break;

            case "--delimiter" or "-d":
                delimiter = NormalizeDelimiter(Next(args, ref i, arg));
                break;

            case "--no-header" or "-nh":
                noHeader = true;
                i++;
                break;

            case var option when option.StartsWith('-'):
                throw new Exception($"Opción desconocida: '{option}'. Use --help.");

            case var value:
                positional.Add(value);
                i++;
                break;
        }
    }

    if (positional.Count >= 1 && inputFile  == null) inputFile  = positional[0];
    if (positional.Count >= 2 && outputFile == null) outputFile = positional[1];
    if (positional.Count >  2)
        throw new Exception("Demasiados argumentos. Máximo: [input] [output]");

    if (sortFields.Count == 0)
        throw new Exception("Debe indicar al menos un criterio con -b|--by.");

    return new AppConfig(inputFile, outputFile, delimiter, noHeader, sortFields);
}

string Next(string[] args, ref int i, string option) {
    i++;
    if (i >= args.Length)
        throw new Exception($"'{option}' requiere un valor.");
    return args[i++];
}

SortField ParseSortField(string spec) {
    string[] parts = spec.Split(':');

    string name       = parts[0];
    bool   numeric    = false;
    bool   descending = false;

    if (parts.Length >= 2) {
        switch (parts[1]) {
            case "num":   numeric = true;  break;
            case "alpha": numeric = false; break;
            default:
                throw new Exception($"Tipo inválido en '{spec}': '{parts[1]}'. Use 'num' o 'alpha'.");
        }
    }

    if (parts.Length >= 3) {
        switch (parts[2]) {
            case "asc":  descending = false; break;
            case "desc": descending = true;  break;
            default:
                throw new Exception($"Orden inválido en '{spec}': '{parts[2]}'. Use 'asc' o 'desc'.");
        }
    }

    if (parts.Length > 3)
        throw new Exception($"Especificación inválida: '{spec}'.");

    if (string.IsNullOrEmpty(name))
        throw new Exception($"Falta el nombre del campo en '{spec}'.");

    return new SortField(name, numeric, descending);
}

string NormalizeDelimiter(string raw) {
    return raw switch {
        "\\t" => "\t",
        "\\n" => "\n",
        "\\r" => "\r",
        ""    => throw new Exception("El delimitador no puede estar vacío."),
        _     => raw
    };
}

// ── Pasos 2 y 6: I/O ─────────────────────────────────────────────────────────

string ReadInput(string? filePath) {
    if (filePath == null)
        return Console.In.ReadToEnd();

    if (!File.Exists(filePath))
        throw new FileNotFoundException($"Archivo no encontrado: '{filePath}'");

    return File.ReadAllText(filePath);
}

void WriteOutput(string? filePath, string content) {
    if (filePath == null) {
        Console.Write(content);
        return;
    }

    File.WriteAllText(filePath, content);
}

// ── Paso 3: Parsear archivo delimitado ───────────────────────────────────────

(string[] headers, List<Dictionary<string, string>> rows) ParseDelimited(
    string content,
    string delimiter,
    bool   noHeader)
{
    string[] split(string line) {
        string[] parts = line.Split(delimiter);
        for (int j = 0; j < parts.Length; j++)
            parts[j] = parts[j].Trim();
        return parts;
    }

    string[] lines = content.Split('\n');

    int last = lines.Length - 1;
    while (last >= 0 && lines[last].Trim() == "") last--;

    if (last < 0)
        return ([], []);

    string[] headers;
    int      firstDataLine;

    if (noHeader) {
        string[] firstRow = split(lines[0]);
        headers = new string[firstRow.Length];
        for (int j = 0; j < firstRow.Length; j++)
            headers[j] = j.ToString();
        firstDataLine = 0;
    } else {
        headers       = split(lines[0]);
        firstDataLine = 1;
    }

    var rows = new List<Dictionary<string, string>>();
    for (int i = firstDataLine; i <= last; i++) {
        string line = lines[i];
        if (line == "") continue;

        string[] values = split(line);
        var row = new Dictionary<string, string>();
        for (int col = 0; col < headers.Length; col++)
            row[headers[col]] = col < values.Length ? values[col] : "";

        rows.Add(row);
    }

    return (headers, rows);
}

// ── Paso 4: Ordenar filas ─────────────────────────────────────────────────────

List<Dictionary<string, string>> SortRows(
    List<Dictionary<string, string>> rows,
    string[]                         headers,
    List<SortField>                  sortFields)
{
    foreach (var field in sortFields) {
        bool found = false;
        for (int i = 0; i < headers.Length; i++)
            if (headers[i] == field.Name) { found = true; break; }

        if (!found)
            throw new Exception($"Campo no encontrado: '{field.Name}'");
    }

    if (rows.Count == 0 || sortFields.Count == 0)
        return rows;

    IOrderedEnumerable<Dictionary<string, string>>? ordered = null;

    foreach (var field in sortFields) {
        IComparable Key(Dictionary<string, string> row) {
            string value = row[field.Name];
            if (field.Numeric)
                return double.TryParse(value, out double n) ? n : double.NaN;
            return value;
        }

        if (ordered == null)
            ordered = field.Descending
                ? rows.OrderByDescending(Key)
                : rows.OrderBy(Key);
        else
            ordered = field.Descending
                ? ordered.ThenByDescending(Key)
                : ordered.ThenBy(Key);
    }

    return ordered!.ToList();
}

// ── Paso 5: Serializar de vuelta a texto ──────────────────────────────────────

string Serialize(
    string[]                         headers,
    List<Dictionary<string, string>> rows,
    string                           delimiter,
    bool                             noHeader)
{
    var sb = new StringBuilder();

    if (!noHeader)
        sb.AppendLine(string.Join(delimiter, headers));

    foreach (var row in rows) {
        string[] values = new string[headers.Length];
        for (int i = 0; i < headers.Length; i++)
            values[i] = row.TryGetValue(headers[i], out string? v) ? v : "";

        sb.AppendLine(string.Join(delimiter, values));
    }

    return sb.ToString();
}
```

## Probando

Creá `empleados.csv`:

```csv
nombre,apellido,edad,salario,departamento
Fernanda,Fernández,35,85000,Ingeniería
Ana,Álvarez,28,72000,Diseño
Diana,Díaz,42,120000,Gerencia
Eduardo,Espinosa,31,88000,Ingeniería
Beatriz,Blanco,25,65000,Diseño
Carlos,Castro,38,95000,Gerencia
```

```bash
# Ordenar por apellido (alfabético, ascendente)
dotnet run sortx.cs -- empleados.csv -b apellido

# Ordenar por salario descendente
dotnet run sortx.cs -- empleados.csv -b salario:num:desc

# Ordenar por departamento, dentro por salario descendente
dotnet run sortx.cs -- empleados.csv -b departamento -b salario:num:desc

# Guardar en archivo
dotnet run sortx.cs -- empleados.csv resultado.csv -b apellido

# Leyendo desde stdin
cat empleados.csv | dotnet run sortx.cs -- -b apellido > ordenado.csv

# Sin encabezado, ordenar por columna 3 numérica descendente
tail -n +2 empleados.csv | dotnet run sortx.cs -- -nh -b 3:num:desc

# Campo inexistente → error claro
dotnet run sortx.cs -- empleados.csv -b columnaInexistente
# Error: Campo no encontrado: 'columnaInexistente'

# Sin criterios → error claro
dotnet run sortx.cs -- empleados.csv
# Error: Debe indicar al menos un criterio con -b|--by.

# Especificación inválida → error claro
dotnet run sortx.cs -- empleados.csv -b salario:numerico
# Error: Tipo inválido en 'salario:numerico': 'numerico'. Use 'num' o 'alpha'.

# Ver ayuda
dotnet run sortx.cs -- --help
```

> **¿Por qué `--` antes de los argumentos?** Le dice a `dotnet run` que todo lo que sigue son argumentos para *tu* programa, no para `dotnet`. Sin esa marca, opciones como `-h` o `--help` las consumiría `dotnet` antes de llegar a `sortx`.

## Lo que aprendimos

| Concepto                       | Dónde lo usamos                                                      |
|--------------------------------|----------------------------------------------------------------------|
| *File-based program*           | Todo el programa en `sortx.cs`, sin `.csproj` ni clase ni `Main`     |
| `record` con varios campos     | `AppConfig`: 5 campos inmutables construidos de una vez              |
| `record` anidado en colección  | `List<SortField>` dentro de `AppConfig`                              |
| Funciones locales              | Cada paso del pipeline es una función independiente                  |
| `switch` con patrones `or`     | `"--by" or "-b"` en un solo `case`                                   |
| `switch` como expresión        | `NormalizeDelimiter` mapea valores con una expresión `switch`        |
| `ref int i` en `Next`          | El índice se avanza dentro de la función auxiliar                    |
| Tuple return `(T1, T2)`        | `ParseDelimited` retorna headers y rows juntos                       |
| Función local anidada          | `split` dentro de `ParseDelimited` reutiliza el delimitador          |
| Headers numéricos sintéticos   | Cuando no hay encabezado, generamos `"0", "1", ...` como nombres     |
| `OrderBy` + `ThenBy`           | El primer criterio inicia el orden; los siguientes desempatan        |
| `IComparable` como key         | Permite alternar entre orden numérico y alfabético sin duplicar código |
| Captura en función local       | `Key` captura `field` correctamente en cada iteración del `foreach`  |
| `double.NaN` como caída suave  | Una celda no numérica no rompe el ordenamiento                       |
| `StringBuilder`                | `Serialize` evita concatenar strings inmutables uno por uno          |
| Validar antes de operar        | `SortRows` valida los campos antes de ordenar                        |
| Validar al parsear             | `ParseSortField` valida tipo y orden donde el error tiene sentido    |
| Raw string literal `"""..."""` | `PrintHelp` con bloque multilínea sin escapes                        |
| `Console.Error` para errores   | Los errores no contaminan la salida redirigida                       |
| `Environment.Exit(código)`     | `0` para éxito, distinto de `0` para fallo: convención Unix          |

## Resumen

```
¿Necesito construir una CLI con varios criterios combinables?
    └─► pipeline de funciones + lista de records

¿Necesito modelar configuración inmutable con sub-elementos?
    └─► `record` que contiene `List<otroRecord>`

¿Necesito parsear opciones con sub-sintaxis (campo:tipo:orden)?
    └─► función dedicada `ParseSortField` con `Split(':')` y validación temprana

¿Necesito un flag booleano (sin valor)?
    └─► en el `switch` solo `i++`, sin llamar a `Next`

¿Necesito que el delimitador `\t` funcione siempre?
    └─► normalizar el string antes de usarlo

¿Necesito leer desde archivo o desde stdin?
    └─► `File.ReadAllText(...)` o `Console.In.ReadToEnd()`

¿Necesito procesar un archivo delimitado con o sin encabezado?
    └─► generar headers numéricos sintéticos cuando no hay encabezado

¿Necesito ordenar por varios criterios encadenados?
    └─► `OrderBy(...)` para el primero, `ThenBy(...)` para los siguientes

¿Necesito alternar comparación numérica y alfabética?
    └─► función local que devuelve `IComparable`, eligiendo `double` o `string`

¿Necesito construir texto eficientemente?
    └─► `StringBuilder` y `string.Join(delim, ...)`

¿Necesito mostrar ayuda clara y ejemplos?
    └─► `PrintHelp()` con raw string literal `"""..."""`

¿Necesito que el programa devuelva código de error a la shell?
    └─► `Environment.Exit(1)` desde el `catch`

Ejemplos clave:
    `record SortField`     → un criterio inmutable de ordenación
    `record AppConfig`     → toda la configuración de una corrida
    `switch + or`          → opciones cortas y largas en un solo case
    `Split(':')`           → sub-sintaxis de campo:tipo:orden
    `Dictionary<,>`        → filas con acceso por nombre o por índice
    `OrderBy / ThenBy`     → ordenamiento primario y desempate
    `IComparable`          → tipo unificado de clave de ordenación
    `IOrderedEnumerable`   → resultado intermedio de `OrderBy`
    `StringBuilder`        → construcción eficiente del texto de salida
    `Console.Error`        → mensajes de error separados del output
```

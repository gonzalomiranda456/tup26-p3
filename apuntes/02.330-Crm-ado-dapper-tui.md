
# Mini CRM con TUI, ADO.NET y Dapper — Diseño por capas

Un mini CRM (contactos + oportunidades + actividades) que combina Terminal.Gui, SQLite y Dapper.

## Arquitectura en cuatro capas

```
Presentación (TUI)       → CrmWindow, diálogos, listas
Aplicación               → CrmWindow coordina acciones del usuario
Persistencia             → CrmDatabase (SQL con Dapper)
Dominio                  → Contact, Opportunity, Activity
```

## Dominio — las entidades

## Persistencia — CrmDatabase con Dapper

```csharp
public class CrmDatabase {
    private readonly string connStr;

    public CrmDatabase(string dbPath) {
        connStr = $"Data Source={dbPath}";
        Initialize();
    }

    private IDbConnection Conn() => new SqliteConnection(connStr);

    private void Initialize() {
        using var conn = Conn();
        conn.Execute("""
            CREATE TABLE IF NOT EXISTS Contacts (
                Id INTEGER PRIMARY KEY AUTOINCREMENT,
                Name TEXT NOT NULL, Company TEXT, Email TEXT,
                Phone TEXT, Status TEXT, Notes TEXT,
                CreatedAt TEXT DEFAULT (datetime('now'))
            );
            CREATE TABLE IF NOT EXISTS Opportunities (
                Id INTEGER PRIMARY KEY AUTOINCREMENT,
                ContactId INTEGER REFERENCES Contacts(Id) ON DELETE CASCADE,
                Title TEXT, Stage TEXT, Value REAL,
                ExpectedCloseDate TEXT, Notes TEXT
            );
            CREATE TABLE IF NOT EXISTS Activities (
                Id INTEGER PRIMARY KEY AUTOINCREMENT,
                ContactId INTEGER REFERENCES Contacts(Id) ON DELETE CASCADE,
                Type TEXT, Subject TEXT, DueDate TEXT,
                IsDone INTEGER DEFAULT 0, Notes TEXT
            );
            """);
    }

    // Búsqueda
    public List<Contact> SearchContacts(string q) {
        using var conn = Conn();
        return conn.Query<Contact>(
            "SELECT * FROM Contacts WHERE Name LIKE @q OR Email LIKE @q OR Company LIKE @q",
            new { q = $"%{q}%" }).ToList();
    }

    // CRUD contactos
    public void AddContact(Contact c) {
        using var conn = Conn();
        conn.Execute("INSERT INTO Contacts (Name, Company, Email, Phone, Status, Notes) " +
                     "VALUES (@Name, @Company, @Email, @Phone, @Status, @Notes)", c);
    }
    public void UpdateContact(Contact c) {
        using var conn = Conn();
        conn.Execute("UPDATE Contacts SET Name=@Name, Company=@Company, Email=@Email, " +
                     "Phone=@Phone, Status=@Status, Notes=@Notes WHERE Id=@Id", c);
    }
    public void DeleteContact(int id) {
        using var conn = Conn();
        conn.Execute("DELETE FROM Contacts WHERE Id=@id", new { id });
    }

    // Oportunidades y actividades por contacto
    public List<Opportunity> GetOpportunities(int contactId) {
        using var conn = Conn();
        return conn.Query<Opportunity>(
            "SELECT * FROM Opportunities WHERE ContactId=@contactId", new { contactId }).ToList();
    }
    public List<Activity> GetActivities(int contactId) {
        using var conn = Conn();
        return conn.Query<Activity>(
            "SELECT * FROM Activities WHERE ContactId=@contactId ORDER BY DueDate", new { contactId }).ToList();
    }

    public void CompleteActivity(int id) {
        using var conn = Conn();
        conn.Execute("UPDATE Activities SET IsDone=1 WHERE Id=@id", new { id });
    }
}
```

**Dapper vs ADO.NET crudo:** Dapper mapea automáticamente filas a objetos C#. Sin Dapper habría que leer columna por columna con `DbDataReader`. El SQL sigue siendo explícito — no hay "magia" de ORM.

## Coordinación de estado en CrmWindow

El patrón maestro-detalle: cuando cambia la selección en la lista de contactos, todo lo demás se actualiza.

```csharp
public class CrmWindow : Window {
    private Contact? selectedContact;
    private List<Contact> contacts = [];
    private readonly CrmDatabase db;

    // Al cambiar selección en la lista:
    private void SelectContact(int index) {
        if (index < 0 || index >= contacts.Count) return;
        selectedContact = contacts[index];
        RefreshDetail();
        RefreshOpportunities();
        RefreshActivities();
    }

    private void RefreshContacts(string q = "") {
        contacts = db.SearchContacts(q);
        contactsList.SetSource(contacts.Select(c => $"{c.Name,-25} {c.Company,-20} {c.Status}").ToList());
    }
    private void RefreshDetail() { /* actualiza labels con datos de selectedContact */ }
    private void RefreshOpportunities() {
        if (selectedContact == null) return;
        var opps = db.GetOpportunities(selectedContact.Id);
        opportunitiesList.SetSource(opps.Select(o => $"{o.Title} — {o.Stage} — ${o.Value:N0}").ToList());
    }
    private void RefreshActivities() {
        if (selectedContact == null) return;
        var acts = db.GetActivities(selectedContact.Id);
        activitiesList.SetSource(acts.Select(a =>
            $"[{(a.IsDone ? "✓" : " ")}] {a.Subject,-30} {a.DueDate:dd/MM}").ToList());
    }
}
```

## Flujo de datos al editar

```
Usuario presiona F2
    ↓
CrmWindow abre ContactDialog (modal)
    ↓
Application.Run(dialog) — bloquea hasta que cierra
    ↓
si !dialog.Cancelado → db.UpdateContact(dialog.Resultado)
    ↓
RefreshContacts()  — la lista refleja el cambio
```

## Decisiones de diseño

| Decisión                       | Razón                                                              |
|--------------------------------|--------------------------------------------------------------------|
| `ON DELETE CASCADE` en BD      | Al borrar un contacto, sus datos hijos se eliminan automáticamente |
| Datos semilla al inicio        | Permite probar la app inmediatamente                               |
| Un archivo, capas conceptuales | Simplicidad didáctica sin perder separación de responsabilidades   |
| SQL explícito con Dapper       | Se ve qué hace la app sin abstracciones de ORM                     |
| Diálogos modales para edición  | Aíslan la tarea; el usuario no pierde contexto                     |
| `RefreshX()` centralizado      | Evita duplicar lógica de actualización en cada evento              |

# Repository y Unit of Work en C#

```
Repository = interfaz uniforme para acceder a datos (oculta SQL/JSON/etc.)
Unit of Work = agrupa cambios y los persiste todos juntos al final
```

## Repository — la interfaz

```csharp
public interface IRepository<T, TId> {
    T?        GetById(TId id);
    List<T>   GetAll();
    void      Add(T entity);
    void      Update(T entity);
    void      Delete(TId id);
}
```

El código de negocio solo habla con `IRepository`. No sabe si los datos vienen de un archivo, SQLite o una API.

## Unit of Work — la interfaz

```csharp
public interface IUnitOfWork : IDisposable {
    IRepository<Contacto, int> Contactos { get; }
    void SaveChanges();
}
```

`SaveChanges()` persiste todos los cambios acumulados. Garantiza atomicidad: o todo se guarda, o nada.

## Change tracking — EntityState

Para saber qué operación realizar al guardar, se rastrea el estado de cada entidad:

```csharp
public enum EntityState { Unchanged, Added, Modified, Deleted }

public class TrackedEntry<T> {
    public T Entity    { get; }
    public EntityState State { get; set; }
    public TrackedEntry(T entity, EntityState state) { Entity = entity; State = state; }
}
```

## Implementación JSON

```csharp
public class JsonUnitOfWork : IUnitOfWork {
    private readonly List<TrackedEntry<Contacto>> tracked = [];
    private readonly string path;

    public JsonUnitOfWork(string path) {
        this.path = path;
        Contactos = new TrackedContactoRepository(tracked, path);
    }

    public IRepository<Contacto, int> Contactos { get; }

    public void SaveChanges() {
        var todos = JsonSerializer.Deserialize<List<Contacto>>(File.ReadAllText(path)) ?? [];
        foreach (var entry in tracked) {
            switch (entry.State) {
                case EntityState.Added:    todos.Add(entry.Entity); break;
                case EntityState.Modified: var i = todos.FindIndex(c => c.Id == entry.Entity.Id);
                                           if (i >= 0) todos[i] = entry.Entity; break;
                case EntityState.Deleted:  todos.RemoveAll(c => c.Id == entry.Entity.Id); break;
            }
        }
        File.WriteAllText(path, JsonSerializer.Serialize(todos, new JsonSerializerOptions { WriteIndented = true }));
        tracked.Clear();
    }

    public void Dispose() { }
}
```

## Implementación SQLite

```csharp
public class SqliteUnitOfWork : IUnitOfWork {
    private readonly SqliteConnection conn;
    private SqliteTransaction? tx;

    public SqliteUnitOfWork(string connStr) {
        conn = new SqliteConnection(connStr);
        conn.Open();
        tx = conn.BeginTransaction();
        Contactos = new SqliteContactoRepository(conn, tx);
    }

    public IRepository<Contacto, int> Contactos { get; }

    public void SaveChanges() { tx?.Commit(); tx = conn.BeginTransaction(); }

    public void Dispose() { tx?.Rollback(); conn.Dispose(); }
}
```

Si `Dispose()` se llama sin `SaveChanges()`, el `Rollback()` deshace todo — comportamiento seguro por defecto.

## Uso

```csharp
// En el composition root (Program.cs)
using IUnitOfWork uow = new SqliteUnitOfWork("Data Source=app.db");

uow.Contactos.Add(new Contacto { Nombre = "Ada", Email = "ada@example.com" });
uow.Contactos.Add(new Contacto { Nombre = "Grace", Email = "grace@example.com" });
uow.SaveChanges(); // ambas se persisten juntas
```

## Resumen

| Concepto              | Rol                                                   |
|-----------------------|-------------------------------------------------------|
| `IRepository<T,TId>`  | CRUD para un tipo de entidad                          |
| `IUnitOfWork`         | Agrupa repositorios y persiste con `SaveChanges()`    |
| `EntityState`         | Unchanged / Added / Modified / Deleted                |
| `TrackedEntry<T>`     | Entidad + su estado pendiente                         |
| `SaveChanges()`       | Aplica todos los cambios — commit o escritura a disco |

Cambiar de JSON a SQLite = cambiar `new JsonUnitOfWork(...)` por `new SqliteUnitOfWork(...)`. El código de negocio no cambia.

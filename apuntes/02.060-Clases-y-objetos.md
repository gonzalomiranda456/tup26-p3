# Clases y objetos en C#

Una **clase** define el estado (datos) y el comportamiento (métodos) de un tipo. Un **objeto** es una instancia de esa clase creada con `new`.

## Sintaxis básica

```csharp
public class Persona {
    private string nombre;
    private int    edad;

    public Persona(string nombre, int edad) {
        this.nombre = nombre;
        this.edad   = edad;
    }

    public string Nombre => this.nombre;
    public string Saludar() => $"Hola, soy {this.nombre} y tengo {this.edad} años.";
}

var p = new Persona("Ana", 25);
Console.WriteLine(p.Saludar());
```

## Constructores

```csharp
// Constructor delegado: un constructor llama a otro con :this(...)
public class Conexion {
    public string Host   { get; }
    public int    Puerto { get; }
    public bool   Segura { get; }

    public Conexion(string host, int puerto, bool segura) {
        Host = host; Puerto = puerto; Segura = segura;
    }
    public Conexion(string host) : this(host, 443, true) { }
    public Conexion() : this("localhost") { }
}

// Primary constructor (C# 12+) — forma más compacta
public class Punto(double x, double y) {
    public double X => x;
    public double Y => y;
    public double Distancia => Math.Sqrt(x * x + y * y);
}

// Inicializador de objeto — asigna propiedades post-construcción
var e = new Empleado { Nombre = "Carlos", Departamento = "IT", Salario = 95_000m };
```

## Propiedades

| Sintaxis                  | Acceso                  | Cuándo usarla                 |
| :------------------------ | :---------------------- | :---------------------------- |
| `{ get; set; }`           | Libre                   | Estado mutable                |
| `{ get; private set; }`   | Escritura interna       | Estado con control            |
| `{ get; }`                | Solo lectura            | Se asigna en constructor      |
| `{ get; init; }`          | Solo en `new {}`        | DTOs, inicializadores         |
| `=> expresion`            | Solo lectura calculada  | Propiedades derivadas         |
| `required { get; init; }` | Obligatoria en `new {}` | Campos que no pueden omitirse |

```csharp
public class Articulo {
    public required string  Nombre { get; init; }
    public decimal Precio { get; set; }
    public int     Stock  { get; private set; }
    public bool    Disponible => Stock > 0;   // calculada
}
```

## Métodos y sobrecarga

```csharp
public class Logger {
    public void Log(string mensaje)               => Console.WriteLine($"[INFO] {mensaje}");
    public void Log(string mensaje, Exception ex) => Console.WriteLine($"[ERROR] {mensaje}: {ex.Message}");
    public void Log(string mensaje, string nivel) => Console.WriteLine($"[{nivel}] {mensaje}");
}
```

La sobrecarga se resuelve en **compilación**. Es distinta del `override`, que se resuelve en ejecución.

## Miembros estáticos

Pertenecen a la clase, no a las instancias:

```csharp
public class Contador {
    private static int total = 0;
    public int Id { get; }

    public Contador() { total++; Id = total; }

    public static int Total => total;
}

Console.WriteLine(Contador.Total); // sin instanciar
```

Clases completamente estáticas (no se instancian):

```csharp
public static class MathEx {
    public static bool EsPrimo(int n) {
        if (n < 2) return false;
        for (int i = 2; i * i <= n; i++)
            if (n % i == 0) return false;
        return true;
    }
}
```

## Herencia

Una clase derivada adquiere los miembros de su clase base (relación "es un"):

```csharp
public class Animal {
    public string Nombre { get; }
    public Animal(string nombre) { Nombre = nombre; }
    public virtual string HacerSonido() => "...";
}

public class Perro : Animal {
    public string Raza { get; }
    public Perro(string nombre, string raza) : base(nombre) { Raza = raza; }
    public override string HacerSonido() => "Guau!";
}
```

`base(...)` encadena con el constructor de la clase padre. `protected` hace que los miembros sean visibles en las subclases pero no fuera.

## Polimorfismo

`virtual` + `override` = despacho dinámico: el runtime decide qué implementación ejecutar según el tipo real del objeto, no el tipo declarado de la variable:

```csharp
List<Animal> animales = [new Perro("Rex", "Labrador"), new Gato("Mia")];
foreach (var a in animales)
    Console.WriteLine(a.HacerSonido()); // cada uno ejecuta su override
```

`new` oculta el miembro base pero **no es polimórfico** — preferir siempre `override`.

Para verificar el tipo en runtime:

```csharp
if (animal is Perro perro) Console.WriteLine(perro.Raza);
var gato = animal as Gato; // null si falla, no excepción
```

## Clases abstractas

No se pueden instanciar. Definen estructura compartida y obligan a implementar ciertos métodos:

```csharp
public abstract class Figura {
    public abstract double Area();
    public abstract double Perimetro();
    public virtual string Describir() => $"{GetType().Name}: área={Area():F2}";
}

public class Circulo(double radio) : Figura {
    public override double Area()      => Math.PI * radio * radio;
    public override double Perimetro() => 2 * Math.PI * radio;
}
```

|                    | Clase abstracta | Interfaz          |
| :----------------- | :-------------- | :---------------- |
| Puede tener estado | Sí              | No                |
| Herencia múltiple  | No              | Sí                |
| Expresa "es un"    | Sí              | No necesariamente |

## Inmutabilidad

Un tipo inmutable no permite modificar su estado después de construirse. Las "mutaciones" devuelven un nuevo objeto:

```csharp
public sealed class Dinero {
    public decimal Monto  { get; }
    public string  Moneda { get; }

    public Dinero(decimal monto, string moneda) { Monto = monto; Moneda = moneda; }
    public Dinero Sumar(Dinero otro) => new(Monto + otro.Monto, Moneda);
    public static Dinero operator +(Dinero a, Dinero b) => a.Sumar(b);
}
```

`sealed` impide que se herede la clase.

## class vs. record vs. struct

|            | `class`                      | `record`               | `struct`                 |
| :--------- | :--------------------------- | :--------------------- | :----------------------- |
| Tipo       | Referencia                   | Referencia             | Valor                    |
| Igualdad   | Por referencia               | Por valor              | Por valor                |
| Ideal para | Entidades con estado mutable | DTOs, datos inmutables | Datos pequeños y simples |

```csharp
record Persona(string Nombre, int Edad);  // igualdad por valor, ToString, deconstrucción

var p1 = new Persona("Ana", 25);
var p2 = p1 with { Edad = 26 };          // copia con cambio
Console.WriteLine(p1 == p2);            // false
```

## Métodos de extensión

Agregan funcionalidad a tipos existentes sin heredar:

```csharp
public static class StringExtensions {
    public static bool EsEmail(this string s) => s.Contains('@') && s.Contains('.');
}

"ana@ejemplo.com".EsEmail(); // true
```

## Resumen

| Necesidad                       | Herramienta                         |
| :------------------------------ | :---------------------------------- |
| Estado + comportamiento propios | `class`                             |
| Inicialización compacta         | primary constructor                 |
| Propiedad obligatoria           | `required { get; init; }`           |
| Comportamiento sin instancia    | `static`                            |
| Especializar un tipo            | herencia + `override`               |
| Variar comportamiento por tipo  | polimorfismo (`virtual`/`override`) |
| Forzar implementación           | `abstract`                          |
| Datos inmutables entre capas    | `record`                            |
| Datos pequeños por valor        | `struct`                            |
| Extender tipos externos         | métodos de extensión                |

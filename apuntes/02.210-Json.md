# JSON — El formato de intercambio de datos

---

## 1. El problema: compartir datos entre programas

Imaginá que tenés dos programas. Uno está escrito en C# y el otro en cualquier otro lenguaje. Quieren pasarse datos.

El primer programa tiene esto en memoria:

```csharp
var producto = new Producto(1, "Teclado mecánico", 85000.00m, true);
```

Ese objeto existe como bits en la RAM. Tiene una forma específica que depende de C#, de .NET, de la arquitectura del procesador. Otro programa no puede leer esos bits directamente — no sabe cómo interpretarlos.

Para que dos programas se entiendan, los datos tienen que salir de la memoria y convertirse en algo **universal**: texto. El texto es texto en todos lados. No importa el lenguaje, el sistema operativo, ni la arquitectura.

El problema ahora es: ¿qué texto? Si mandás:

```
1|Teclado mecánico|85000.00|true
```

¿Cómo sabe el otro programa que `1` es el id, `Teclado mecánico` es el nombre, y que el separador es `|`? ¿Qué pasa si el nombre tiene un `|` adentro?

Necesitás un **formato estándar**: reglas acordadas de antemano sobre cómo se escribe la información, de forma que cualquier programa que conozca esas reglas pueda leerla correctamente.

Ese formato es **JSON**.

---

## 2. La solución: un formato de texto

JSON (*JavaScript Object Notation*) nació en los años 2000 como una forma simple de representar datos estructurados como texto. Hoy es el formato estándar para intercambiar datos en la web.

Tiene tres características que lo hicieron ganar:

**Es legible por humanos.** Podés abrir un archivo JSON en cualquier editor y entender qué dice. Eso hace que sea muy fácil de debuggear.

**Es simple.** Tiene solo seis tipos de valores. No hay nada más.

**Es universal.** Todos los lenguajes modernos tienen soporte nativo para leer y escribir JSON.

El mismo producto de antes, en JSON:

```json
{
  "id": 1,
  "nombre": "Teclado mecánico",
  "precio": 85000.00,
  "disponible": true
}
```

Cualquier programa en cualquier lenguaje puede leer eso. Las reglas son públicas y simples.

---

## 3. Los valores primitivos

JSON tiene cuatro tipos de valores simples:

### String

Texto entre **comillas dobles**. Siempre dobles, nunca simples.

```json
"Hola mundo"
"ada@history.dev"
"2024-11-15"
""
```

Si el texto tiene comillas dobles adentro, se escapan con `\`:

```json
"Ella dijo \"hola\""
```

Otros caracteres especiales:

| Secuencia | Carácter         |
|-----------|------------------|
| `\n`      | salto de línea   |
| `\t`      | tabulación       |
| `\\`      | barra invertida  |

### Number

Números sin comillas. Enteros o decimales, positivos o negativos.

```json
42
-7
3.14
0.001
1000000
```

> JSON no distingue entre entero y decimal — es solo "number". En C#, el tipo al que se mapea (int, double, decimal) depende de cómo declaremos la propiedad.

### Boolean

`true` o `false`. En minúsculas, sin comillas.

```json
true
false
```

### Null

Ausencia de valor. En minúsculas, sin comillas.

```json
null
```

---

## 4. Los objetos

Un objeto agrupa varios valores bajo **claves con nombre**. Se escribe con llaves:

```json
{ "clave": valor, "otraClave": otroValor }
```

Las reglas:

- Las **claves** son siempre strings: entre comillas dobles.
- Los **valores** pueden ser cualquiera de los seis tipos de JSON.
- Los pares clave-valor se separan con comas.
- El último par **no lleva coma** (a diferencia de C#).

```json
{
  "nombre": "Ada",
  "apellido": "Lovelace",
  "edad": 36,
  "activa": true,
  "apodo": null
}
```

El orden de las claves no importa. Estos dos objetos son equivalentes:

```json
{ "nombre": "Ada", "edad": 36 }
{ "edad": 36, "nombre": "Ada" }
```

---

## 5. Los arrays

Un array es una lista ordenada de valores. Se escribe con corchetes:

```json
[ valor, valor, valor ]
```

Los valores pueden ser de cualquier tipo, y pueden mezclarse (aunque en la práctica, los arrays de una API suelen tener todos el mismo tipo):

```json
[1, 2, 3, 4, 5]

["Ada", "Alan", "Grace"]

[true, false, true]

[]
```

El orden importa. `[1, 2, 3]` y `[3, 2, 1]` son arrays distintos.

---

## 6. Anidamiento

La potencia real de JSON viene de que los valores de un objeto o array pueden ser **otros objetos o arrays**. Se pueden anidar sin límite.

### Objeto con un array adentro

```json
{
  "nombre":    "Grace",
  "apellido":  "Hopper",
  "telefonos": ["+54 381 111-1111", "+54 381 222-2222"]
}
```

### Array de objetos

La estructura más común en las APIs: el servidor devuelve una lista de recursos.

```json
[
  { "id": 1, "texto": "Estudiar HTTP",         "completada": false },
  { "id": 2, "texto": "Practicar Minimal API", "completada": true  },
  { "id": 3, "texto": "Hacer el parcial",      "completada": false }
]
```

### Objeto con objetos anidados

```json
{
  "id": 1,
  "nombre": "Teclado mecánico",
  "precio": 85000.00,
  "categoria": {
     "id": 3,
     "nombre": "Periféricos"
  },
  "dimensiones": {
     "ancho": 44.5,
     "alto": 14.0,
     "profundidad": 3.2
  }
}
```

### Lectura de JSON complejo

Cuando ves un JSON largo, la forma de leerlo es identificar primero la estructura exterior y luego ir hacia adentro:

```json
{
  "pagina": 1,
  "total": 47,
  "resultados": [
    {
      "id": 1,
      "nombre": "Ada Lovelace",
      "lenguajes": ["Analytical Engine", "Assembly"]
    },
    {
      "id": 2,
      "nombre": "Alan Turing",
      "lenguajes": ["Turing machine", "TypeScript"]
    }
  ]
}
```

Preguntás: ¿qué es esto? Un **objeto** con tres claves: `pagina` (number), `total` (number) y `resultados` (array). ¿Qué hay en `resultados`? Un **array de objetos**, cada uno con `id`, `nombre` y `lenguajes`. ¿Qué es `lenguajes`? Un **array de strings**.

## 7. JSON en C#: System.Text.Json

.NET incluye `System.Text.Json` como la librería oficial para trabajar con JSON. Está en el namespace `System.Text.Json` y viene incluida en el SDK — no necesitás instalar nada.

```csharp
using System.Text.Json;
```

Dos operaciones centrales:

- **Serializar**:   convertir un objeto C# → string JSON.
- **Deserializar**: convertir un string JSON → objeto C#.

---

## 8. Serializar: de objeto C# a JSON

### El caso básico

```csharp
using System.Text.Json;

var tarea = new Tarea(1, "Estudiar HTTP", false);
string json = JsonSerializer.Serialize(tarea);

Console.WriteLine(json);
// {"id":1,"texto":"Estudiar HTTP","completada":false}

record Tarea(int Id, string Texto, bool Completada);
```

Por defecto, el JSON sale sin espacios (minified). Para que sea legible:

```csharp
var opciones = new JsonSerializerOptions { WriteIndented = true };
string json  = JsonSerializer.Serialize(tarea, opciones);

Console.WriteLine(json);
// {
//   "id": 1,
//   "texto": "Estudiar HTTP",
//   "completada": false
// }
```

### Serializar colecciones

```csharp
var tareas = new List<Tarea> {
    new(1, "Estudiar HTTP",         false),
    new(2, "Practicar Minimal API", true),
};

string json  = JsonSerializer.Serialize(tareas, new JsonSerializerOptions { WriteIndented = true });
```

```json
[
  { "id": 1, "texto": "Estudiar HTTP",         "completada": false },
  { "id": 2, "texto": "Practicar Minimal API", "completada": true  }
]
```

### Nombres de las claves

Por defecto, `System.Text.Json` usa los nombres tal cual están en C#. Un `record Tarea(int Id, ...)` produce `"Id"` con mayúscula, no `"id"`.

La convención en JSON es `camelCase` (primera letra minúscula). Para aplicarla:

```csharp
var opciones = new JsonSerializerOptions {
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
    WriteIndented = true
};

string json  = JsonSerializer.Serialize(tarea, opciones);
// { "id": 1, "texto": "...", "completada": false }
```

> En Minimal API esto ya está configurado automáticamente — las claves salen en `camelCase` sin que hagas nada.

## 9. Deserializar: de JSON a objeto C#

### El caso básico

```csharp
string json = """{ "id": 1, "texto": "Estudiar HTTP", "completada": false }""";

Tarea? tarea = JsonSerializer.Deserialize<Tarea>(json);

Console.WriteLine(tarea?.Texto);  // Estudiar HTTP
```

`Deserialize<T>` puede retornar `null` si el JSON es el literal `"null"`, por eso el tipo de retorno es `T?`.

### Deserializar una lista

```csharp
string json = """
[
  { "id": 1, "texto": "Estudiar HTTP",         "completada": false },
  { "id": 2, "texto": "Practicar Minimal API", "completada": true  }
]
""";

List<Tarea>? tareas = JsonSerializer.Deserialize<List<Tarea>>(json);

foreach (var t in tareas ?? [])
    Console.WriteLine($"{t.Id}: {t.Texto}");
```

### El mapeo entre tipos C# y tipos JSON

| JSON                 | C#                                          |
|----------------------|---------------------------------------------|
| `string`             | `string`                                    |
| `number` (entero)    | `int`, `long`                               |
| `number` (decimal)   | `double`, `float`, `decimal`                |
| `boolean`            | `bool`                                      |
| `null`               | `null` (tipo nullable: `string?`, `int?`)   |
| `object { }`         | clase o record                              |
| `array [ ]`          | `List<T>`, `T[]`, `IEnumerable<T>`          |

### Cuando los nombres no coinciden

Si el JSON usa `camelCase` (`"nombreProducto"`) pero tu clase usa `PascalCase` (`NombreProducto`), la deserialización falla por defecto porque los nombres no coinciden.

Solución: configurar la política de nombres:

```csharp
var opciones = new JsonSerializerOptions {
    PropertyNameCaseInsensitive = true  // ignora mayúsculas/minúsculas al deserializar
};

string json  = JsonSerializer.Serialize(tarea, opciones);
```

O con la convención de camelCase explícita:

```csharp
var opciones = new JsonSerializerOptions {
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase
};
```

## 10. Casos frecuentes y sus soluciones

### Propiedades opcionales con valor por defecto

El JSON puede no incluir todas las propiedades. Si una propiedad no está en el JSON, la propiedad en C# queda con el valor por defecto del tipo (`0` para `int`, `false` para `bool`, `null` para `string?`).

```csharp
string json = """{ "id": 5, "texto": "Nueva tarea" }""";
// "completada" no está en el JSON

var tarea = JsonSerializer.Deserialize<Tarea>(json);
Console.WriteLine(tarea?.Completada);  // false  ← valor por defecto de bool
```

Para propiedades opcionales con valor distinto a `false`/`0`, usá un record con parámetros con default:

```csharp
record Tarea(int Id, string Texto, bool Completada = false, int Prioridad = 1);
```

### Propiedades que pueden ser null

Si un campo puede estar ausente o ser `null`, declaralo como nullable (`?`):

```json
{ "id": 1, "texto": "Tarea", "descripcion": null }
```

```csharp
record Tarea(int Id, string Texto, string? Descripcion);
//                                         ^ puede ser null
```

### Ignorar propiedades extras

Si el JSON tiene propiedades que no existen en tu clase, `System.Text.Json` las ignora por defecto. No tira error.

```json
{ "id": 1, "texto": "Tarea", "campoQueNoConozco": "xyz" }
```

```csharp
var tarea = JsonSerializer.Deserialize<Tarea>(json);
// "campoQueNoConozco" se ignora silenciosamente
```

### Propiedades con nombres distintos: `[JsonPropertyName]`

Si no podés cambiar el nombre de la propiedad en C# pero el JSON usa otro nombre:

```json
{ "task_id": 1, "task_text": "Estudiar" }
```

```csharp
record Tarea(
    [property: JsonPropertyName("task_id")]   
    int Id,
    
    [property: JsonPropertyName("task_text")] 
    string Texto
);
```

### Enums como strings

Por defecto, los enums se serializan como números. Para que salgan como strings:

```csharp
var opciones = new JsonSerializerOptions {
    Converters = { new JsonStringEnumConverter() }
};

var tarea = new Tarea(1, "Estudiar", Prioridad.Alta);
string json = JsonSerializer.Serialize(tarea, opciones);
// { "id": 1, "texto": "Estudiar", "prioridad": "Alta" }

enum Prioridad { Baja, Normal, Alta }
record Tarea(int Id, string Texto, Prioridad Prioridad);
```

## 11. Ejemplo integrador: una agenda en JSON

Para cerrar, un ejemplo completo que combina todo lo visto: serializar y deserializar una agenda de contactos desde y hacia un archivo JSON.

### El modelo

```csharp
record Contacto(int Id, string Nombre, string Apellido, List<string> Telefonos);
```

Una lista de `Contacto` en JSON se ve así:

```json
[
  {
    "id": 1,
    "nombre": "Ada",
    "apellido": "Lovelace",
    "telefonos": ["+54 381 111-1111"]
  },
  {
    "id": 2,
    "nombre": "Alan",
    "apellido": "Turing",
    "telefonos": ["+54 381 222-2222", "+54 381 333-3333"]
  },
  {
    "id": 3,
    "nombre": "Grace",
    "apellido": "Hopper",
    "telefonos": []
  }
]
```

Observaciones sobre la estructura:

- La raíz es un **array** de objetos — no un objeto con una clave "contactos".
- `telefonos` es un **array de strings** — puede tener cero, uno o varios elementos.
- Los nombres de clave están en `camelCase` — convención estándar de JSON.

### Las opciones compartidas

Definimos las opciones una sola vez para usarlas tanto al leer como al escribir:

```csharp
var opciones = new JsonSerializerOptions {
    PropertyNamingPolicy        = JsonNamingPolicy.CamelCase,
    PropertyNameCaseInsensitive = true,
    WriteIndented               = true
};
```

- `CamelCase` — las propiedades `Nombre`, `Apellido` salen como `"nombre"`, `"apellido"`.
- `CaseInsensitive` — al leer, `"nombre"` se mapea a `Nombre` sin importar las mayúsculas.
- `WriteIndented` — el JSON escrito es legible por humanos.

### Escribir la agenda a un archivo

```csharp
void GuardarAgenda(List<Contacto> contactos, string ruta) {
    string json = JsonSerializer.Serialize(contactos, opciones);
    File.WriteAllText(ruta, json);
}
```

`File.WriteAllText` crea el archivo si no existe, o lo sobreescribe si ya existe. Simple.

### Leer la agenda desde un archivo

```csharp
List<Contacto> CargarAgenda(string ruta) {
    if (!File.Exists(ruta))
        return [];                        // archivo nuevo → lista vacía

    string json = File.ReadAllText(ruta);
    return JsonSerializer.Deserialize<List<Contacto>>(json, opciones) ?? [];
}
```

Dos casos de borde que hay que manejar:

- El archivo no existe todavía (primera ejecución) → devolvemos lista vacía.
- El JSON es el literal `null` → `Deserialize` retorna `null`, el `?? []` lo convierte en lista vacía.

### El programa completo

```csharp
using System.Text.Json;

const string archivo = "agenda.json";

var opciones = new JsonSerializerOptions {
    PropertyNamingPolicy        = JsonNamingPolicy.CamelCase,
    PropertyNameCaseInsensitive = true,
    WriteIndented               = true
};

// --- Cargar ---
var agenda = CargarAgenda(archivo);
Console.WriteLine($"Contactos cargados: {agenda.Count}");

// --- Agregar un contacto nuevo ---
int nuevoId = agenda.Count == 0 ? 1 : agenda.Max(c => c.Id) + 1;
agenda.Add(new Contacto(nuevoId, "Grace", "Hopper", ["+54 381 555-5555"]));

// --- Guardar ---
GuardarAgenda(agenda, archivo);
Console.WriteLine($"Agenda guardada con {agenda.Count} contacto(s).");

// --- Recargar y mostrar ---
var recargada = CargarAgenda(archivo);
foreach (var c in recargada) {
    var tels = c.Telefonos.Count == 0 ? "sin teléfonos" : string.Join(", ", c.Telefonos);
    Console.WriteLine($"[{c.Id}] {c.Apellido}, {c.Nombre} — {tels}");
}

// ---- Funciones ----

List<Contacto> CargarAgenda(string ruta) {
    if (!File.Exists(ruta)) return [];
    string json = File.ReadAllText(ruta);
    return JsonSerializer.Deserialize<List<Contacto>>(json, opciones) ?? [];
}

void GuardarAgenda(List<Contacto> contactos, string ruta) {
    string json = JsonSerializer.Serialize(contactos, opciones);
    File.WriteAllText(ruta, json);
}

record Contacto(int Id, string Nombre, string Apellido, List<string> Telefonos);
```

Ejecutá el programa dos veces. La primera crea el archivo y agrega a Grace Hopper. La segunda carga la agenda del archivo (con Grace ya ahí) y agrega un segundo contacto. El archivo crece entre ejecuciones.

### El archivo generado

```json
[
  {
    "id": 1,
    "nombre": "Grace",
    "apellido": "Hopper",
    "telefonos": [
      "+54 381 555-5555"
    ]
  }
]
```

## 12. Resumen

```
JSON tiene seis tipos de valores:
  string    → "texto entre comillas dobles"
  number    → 42 / 3.14 / -7
  boolean   → true / false
  null      → null
  object    → { "clave": valor, ... }
  array     → [ valor, valor, ... ]

Las reglas de sintaxis que más olvidan:
  ✓  Claves de objetos siempre entre comillas dobles
  ✓  Sin coma después del último elemento
  ✓  true, false, null en minúsculas
  ✗  Comillas simples → inválido
  ✗  Comentarios     → JSON no tiene comentarios

En C# con System.Text.Json:
  Serializar   → JsonSerializer.Serialize(objeto)
  Deserializar → JsonSerializer.Deserialize<Tipo>(json)

  Opciones útiles:
    WriteIndented         = true   → JSON con sangría
    PropertyNamingPolicy  = CamelCase  → claves en camelCase
    PropertyNameCaseInsensitive = true → deserializar sin importar mayúsculas
```

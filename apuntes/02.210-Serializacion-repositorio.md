# Serialización y Repositorio en C#

Persistencia progresiva: texto plano → JSON/XML → base de datos. El patrón Repository oculta el medio de almacenamiento detrás de una interfaz.

## IEntity y Repository genérico

```csharp
public interface IEntity { int Id { get; set; } }

public class Repository<T> where T : class, IEntity {
    private readonly List<T> items;
    private readonly IFileSerializer<T> serializer;
    private readonly string path;

    public Repository(IFileSerializer<T> serializer, string path) {
        this.serializer = serializer;
        this.path = path;
        items = File.Exists(path) ? serializer.Load(path) : [];
    }

    public List<T> GetAll() => items;
    public T? GetById(int id) => items.FirstOrDefault(i => i.Id == id);

    public void Add(T item) {
        item.Id = items.Count == 0 ? 1 : items.Max(i => i.Id) + 1;
        items.Add(item);
        serializer.Save(items, path);
    }

    public void Update(T item) {
        var idx = items.FindIndex(i => i.Id == item.Id);
        if (idx >= 0) { items[idx] = item; serializer.Save(items, path); }
    }

    public void Delete(int id) {
        items.RemoveAll(i => i.Id == id);
        serializer.Save(items, path);
    }
}
```

## IFileSerializer — abstrae JSON vs XML

```csharp
public interface IFileSerializer<T> {
    List<T> Load(string path);
    void Save(List<T> items, string path);
}
```

**JSON:**
```csharp
public class JsonFileSerializer<T> : IFileSerializer<T> {
    private readonly JsonSerializerOptions opts = new() {
        WriteIndented = true, PropertyNamingPolicy = JsonNamingPolicy.CamelCase
    };
    public List<T> Load(string path) =>
        JsonSerializer.Deserialize<List<T>>(File.ReadAllText(path), opts) ?? [];
    public void Save(List<T> items, string path) =>
        File.WriteAllText(path, JsonSerializer.Serialize(items, opts));
}
```

**XML:**
```csharp
public class XmlFileSerializer<T> : IFileSerializer<T> {
    private readonly XmlSerializer xml = new(typeof(List<T>));
    public List<T> Load(string path) {
        using var f = File.OpenRead(path);
        return (List<T>?)xml.Deserialize(f) ?? [];
    }
    public void Save(List<T> items, string path) {
        using var f = File.Create(path);
        xml.Serialize(f, items);
    }
}
```

## FileSerializerFactory

```csharp
public static class FileSerializerFactory {
    public static IFileSerializer<T> Create<T>(string extension) => extension switch {
        ".json" => new JsonFileSerializer<T>(),
        ".xml"  => new XmlFileSerializer<T>(),
        _       => throw new NotSupportedException($"Formato no soportado: {extension}")
    };
}
```

**Uso:**
```csharp
var serializer = FileSerializerFactory.Create<Contacto>(".json");
var repo = new Repository<Contacto>(serializer, "agenda.json");
repo.Add(new Contacto { Nombre = "Grace", Apellido = "Hopper" });
```

## Limitaciones del almacenamiento en archivos

| Problema | Archivos | Base de datos |
|---|---|---|
| Búsquedas complejas | Filtrar todo en memoria | Consultas SQL indexadas |
| Concurrencia | Sin control | Transacciones y locks |
| Relaciones | Manual | Foreign keys |
| Rendimiento (escala) | Carga todo | Solo lo necesario |

Los archivos JSON/XML son útiles para aplicaciones pequeñas o configuración. Para datos que crecen o requieren consultas complejas: SQLite o una base de datos real.

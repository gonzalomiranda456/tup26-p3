# Manejo de errores — `try / catch / finally`

## Estructura básica

```csharp
try {
    string contenido = File.ReadAllText("datos.txt");
    Console.WriteLine(contenido);
} catch (FileNotFoundException ex) {
    Console.WriteLine($"Archivo no encontrado: {ex.FileName}");
} catch (IOException ex) {
    Console.WriteLine($"Error de I/O: {ex.Message}");
} catch (Exception ex) {
    Console.WriteLine($"Error inesperado: {ex.Message}");
    throw; // relanzar sin perder el stack trace
} finally {
    // siempre se ejecuta, haya error o no
    Console.WriteLine("Operación finalizada.");
}
```

Las excepciones más específicas van **primero**; `Exception` al final.

## `finally` y `using`

`finally` es para liberar recursos. `using` lo reemplaza automáticamente para `IDisposable`:

```csharp
// Equivalentes — preferir using
try {
    using var lector = new StreamReader("datos.txt");
    Console.WriteLine(lector.ReadToEnd());
} catch (FileNotFoundException) {
    Console.WriteLine("No existe.");
}
// Dispose() se llama automáticamente
```

## Lanzar excepciones

```csharp
throw new ArgumentOutOfRangeException(nameof(n), "No puede ser negativo");
throw new InvalidOperationException("Estado incorrecto");

// Helpers de C# 10+
ArgumentNullException.ThrowIfNull(nombre);
ArgumentException.ThrowIfNullOrWhiteSpace(nombre);
ArgumentOutOfRangeException.ThrowIfNegative(valor);

// throw como expresión
string nombre = ObtenerNombre() ?? throw new InvalidOperationException("Requerido");

static double Dividir(double a, double b) =>
    b == 0 ? throw new DivideByZeroException() : a / b;
```

## Filtros con `when`

```csharp
catch (HttpRequestException ex) when (ex.StatusCode == HttpStatusCode.NotFound) {
    Console.WriteLine("No encontrado (404)");
} catch (HttpRequestException ex) when (ex.StatusCode == HttpStatusCode.Unauthorized) {
    Console.WriteLine("Sin autorización (401)");
}
```

Si la condición `when` es falsa, la excepción sigue propagándose al siguiente `catch`.

## Excepciones personalizadas

```csharp
public class SaldoInsuficienteException : Exception {
    public decimal SaldoActual     { get; }
    public decimal MontoSolicitado { get; }

    public SaldoInsuficienteException(decimal saldo, decimal monto)
        : base($"Disponible: {saldo:N2}, solicitado: {monto:N2}") {
        SaldoActual = saldo; MontoSolicitado = monto;
    }
}

throw new SaldoInsuficienteException(cuenta.Saldo, montoSolicitado);
```

## Buenas prácticas

Capturar excepciones específicas, no `Exception` genérica. No silenciar errores con `catch (Exception) { }`. Usar `TryParse` / `TryGetValue` para situaciones esperadas (más eficiente que try/catch):

```csharp
// MAL: excepciones para control de flujo
try { var v = diccionario["clave"]; } catch (KeyNotFoundException) { }

// BIEN
if (diccionario.TryGetValue("clave", out var v)) Console.WriteLine(v);
```

## Jerarquía de excepciones comunes

```
Exception
├── ArgumentException
│   ├── ArgumentNullException
│   └── ArgumentOutOfRangeException
├── InvalidOperationException
├── NullReferenceException   ← señal de null no manejado
├── IOException
│   ├── FileNotFoundException
│   └── UnauthorizedAccessException
└── FormatException
```

## Resumen

| Necesidad                         | Herramienta                   |
| :-------------------------------- | :---------------------------- |
| Código protegido ante fallos      | `try/catch`                   |
| Limpieza garantizada              | `finally`                     |
| Liberar recursos                  | `using`                       |
| Señalar estado inválido           | `throw new XxxException(...)` |
| Propagar error original           | `throw;` (sin argumentos)     |
| Filtrar condición dentro de catch | `catch (...) when (...)`      |
| Error del dominio propio          | excepción personalizada       |
| Conversión/búsqueda segura        | `TryParse` / `TryGetValue`    |

# Enumeraciones en C#

Un **enum** define un conjunto finito de constantes con nombre. Reemplaza números o cadenas mágicas por valores con significado explícito:

```csharp
// Sin enum — ¿qué significa el 2?
if (orden.Estado == 2) { ... }

// Con enum — el código se explica solo
if (orden.Estado == EstadoOrden.Enviado) { ... }
```

Los enums son **tipos por valor** basados en un entero (`int` por defecto).

## Declaración

```csharp
// Valores asignados automáticamente: 0, 1, 2...
public enum DiaSemana { Lunes, Martes, Miercoles, Jueves, Viernes, Sabado, Domingo }

// Valores explícitos (no necesitan ser consecutivos)
public enum EstadoOrden {
    Pendiente  = 0,
    Confirmado = 1,
    Enviado    = 2,
    Cancelado  = 99
}

// Tipo subyacente distinto de int
public enum CodigoError : short { Ninguno = 0, NoEncontrado = 1001 }
```

## Uso básico

```csharp
DiaSemana hoy = DiaSemana.Miercoles;

if (hoy == DiaSemana.Sabado || hoy == DiaSemana.Domingo)
    Console.WriteLine("Fin de semana");

string tipo = hoy switch {
    DiaSemana.Sabado or DiaSemana.Domingo => "Fin de semana",
    DiaSemana.Viernes                     => "¡Viernes!",
    _                                     => "Día hábil"
};
```

## Conversión

```csharp
int valor  = (int)EstadoOrden.Enviado;       // enum → int: 2
var estado = (EstadoOrden)2;                 // int → enum: Enviado

string nombre = DiaSemana.Lunes.ToString();  // "Lunes"

// string → enum seguro
if (Enum.TryParse<DiaSemana>("Viernes", ignoreCase: true, out DiaSemana d))
    Console.WriteLine(d);

// Verificar si un valor es válido (útil para datos externos)
bool valido = Enum.IsDefined<EstadoOrden>((EstadoOrden)50); // false
```

## Reflexión

```csharp
DiaSemana[] todos  = Enum.GetValues<DiaSemana>();
string[]    nombres = Enum.GetNames<DiaSemana>();

foreach (DiaSemana dia in Enum.GetValues<DiaSemana>())
    Console.WriteLine($"{(int)dia}: {dia}");
```

## Flags — combinaciones de bits

Cuando un valor puede ser **combinación** de varios miembros, se usa `[Flags]`. Cada miembro debe ser potencia de 2:

```csharp
[Flags]
public enum Permiso {
    Ninguno  = 0b0000,
    Leer     = 0b0001,
    Escribir = 0b0010,
    Ejecutar = 0b0100,
    Eliminar = 0b1000,
    Completo = Leer | Escribir | Ejecutar | Eliminar
}

Permiso editor = Permiso.Leer | Permiso.Escribir;

editor.HasFlag(Permiso.Escribir); // true
editor.HasFlag(Permiso.Eliminar); // false

editor |= Permiso.Ejecutar;   // agregar
editor &= ~Permiso.Escribir;  // quitar

Console.WriteLine(editor); // "Leer, Ejecutar"
```

## Serialización JSON

```csharp
// Por defecto: número
// {"Documento":1}

// Como string (más legible):
var opciones = new JsonSerializerOptions {
    Converters = { new JsonStringEnumConverter() }
};
// {"Documento":"Pasaporte"}
```

## Buenas prácticas

Siempre definir un valor `0` con nombre (`Ninguno` o equivalente), ya que es el valor por defecto al hacer `default`:

```csharp
// MAL
public enum Color { Rojo = 1, Verde = 2 }
Color c = default; // 0 — no está definido

// BIEN
public enum Color { Ninguno = 0, Rojo = 1, Verde = 2 }
Color c = default; // Color.Ninguno
```

Validar siempre datos externos antes de hacer cast:

```csharp
if (!Enum.IsDefined<EstadoOrden>((EstadoOrden)valorExterno))
    throw new ArgumentOutOfRangeException(...);
```

No usar enums para conjuntos que crecen con el tiempo (países, monedas): preferir constantes de string.

## Resumen

| Necesidad | Herramienta |
|---|---|
| Conjunto finito de opciones con nombre | `enum` |
| Combinación de varias opciones | `[Flags]` + potencias de 2 |
| Texto amigable por valor | `[Description]` + método de extensión |
| Iterar todos los valores | `Enum.GetValues<T>()` |
| Parsear string de forma segura | `Enum.TryParse<T>()` + `Enum.IsDefined()` |
| Serializar como texto en JSON | `JsonStringEnumConverter` |

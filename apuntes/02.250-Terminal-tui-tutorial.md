# Interfaces de usuario en la terminal con Terminal.Gui v2

## 1. ¿Qué es Terminal.Gui?

Terminal.Gui es una librería que dibuja interfaces gráficas **dentro de la terminal**: campos editables, tablas, menús, foco, atajos y diálogos, todo sin abrir ventanas del sistema operativo. El resultado es una TUI (*Text User Interface*).

En este apunte usamos **Terminal.Gui v2**, cuya API cambió en varios puntos respecto de v1: eventos, manejo de teclado, tipos de algunos widgets y ejemplos más orientados a `Toplevel`, `Window`, `Dialog` y `TableView`.

**Instalación** con la directiva `#:package` (programas file-based / dotnet-script):

```csharp
#:package Terminal.Gui@2.*
```

En un proyecto `.csproj` estándar:

```text
dotnet add package Terminal.Gui --version 2.*
```

## 2. Estructura de una aplicación

En v2 conviene pensar la app en tres piezas:

1. `Application.Init()` inicializa el entorno TUI.
2. Un `Toplevel` actúa como raíz visual.
3. `Application.Run(...)` entra al loop de eventos hasta que alguien llama `Application.RequestStop()`.

```csharp
using Terminal.Gui;

Application.Init();

try {
    var top = new Toplevel();

    var win = new Window {
        Title = "Mi App",
        X = 0, Y = 0,
        Width = Dim.Fill(), Height = Dim.Fill(),
    };

    win.Add(new Label { Text = "¡Hola, Terminal.Gui v2!", X = 2, Y = 1 });
    top.Add(win);

    Application.Run(top);
} finally {
    Application.Shutdown();
}
```

El patrón `Init → Run → Shutdown` sigue siendo obligatorio. Sí, la terminal también aprecia que la dejen ordenada al salir.

Toplever es el elemento raiz de la aplicacion dentro podemos poner ventajas, menues etc. y gestionar eventos globales. Es el equivalente a la ventana principal de una app gráfica tradicional, pero sin bordes ni título.

## 3. Layout: `Pos` y `Dim`

El layout usa **columnas y filas de caracteres**, no píxeles.

**`Pos` — dónde empieza un widget:**

| Expresión          | Significado                                |
| ------------------ | ------------------------------------------ |
| `1`                | columna/fila fija                          |
| `Pos.Center()`     | centrado en el padre                       |
| `Pos.Right(otro)`  | a la derecha del widget `otro`             |
| `Pos.Bottom(otro)` | debajo del widget `otro`                   |
| `Pos.Top(otro)`    | alineado con el borde superior de `otro`   |
| `Pos.AnchorEnd(1)` | pegado al borde final, dejando margen 1    |

**`Dim` — qué tamaño tiene un widget:**

| Expresión     | Significado                                     |
| ------------- | ----------------------------------------------- |
| `20`          | tamaño fijo                                     |
| `Dim.Fill()`  | todo el espacio disponible                      |
| `Dim.Fill(n)` | todo el espacio disponible menos `n` caracteres |

**Ejemplo — tres botones alineados:**

```csharp
var btnNuevo  = new Button { Text = "_Nuevo",  X = 1, Y = Pos.Bottom(tabla) + 1 };
var btnEditar = new Button { Text = "_Editar", X = Pos.Right(btnNuevo) + 2, Y = Pos.Top(btnNuevo) };
var btnSalir  = new Button { Text = "_Salir",  X = Pos.Right(btnEditar) + 2, Y = Pos.Top(btnNuevo) };
```

`Pos.Right(btnNuevo)` calcula en tiempo de ejecución dónde termina el botón. Si el texto cambia, el layout se ajusta solo.

> La `_` en el texto de un botón marca el carácter acelerador (por ejemplo, `Alt+N`).

## 4. Widgets fundamentales en v2

### `Toplevel`

Es la raíz de la interfaz que se pasa a `Application.Run(...)`.

```csharp
var top = new Toplevel();
top.Add(win);
```

### `Window`

Contenedor principal con borde y título.

```csharp
var win = new Window {
    Title = "Agenda",
    X = 0, Y = 1,
    Width = Dim.Fill(), Height = Dim.Fill(1),
};

win.Add(widget1, widget2, widget3);
```

### `Label`

Texto estático. No suele aceptar foco.

```csharp
new Label { Text = "Nombre:", X = 1, Y = 0 }
```

### `TextField`

Campo editable de una sola línea.

```csharp
var campo = new TextField {
    Text = "valor inicial",
    X = 12, Y = 0,
    Width = 30,
};

string valor = (campo.Text ?? "").Trim();
```

En v2, `Text` se usa normalmente como `string`, sin la conversión a `ustring` que era típica en v1.

### `Button`

Botón accionable. En v2 es común manejarlo con el evento `Accepting`.

```csharp
var btnGuardar = new Button {
    Text = "_Guardar",
    IsDefault = true,
    X = 1, Y = 4,
};

btnGuardar.Accepting += (_, _) => {
    Guardar();
};
```

### `MenuBar`

Barra de menús superior para comandos globales.

```csharp
var menu = new MenuBar {
    Menus = new[] {
        new MenuBarItem("_Archivo", new[] {
            new MenuItem("_Guardar", "Guarda los cambios",    () => Guardar()),
            new MenuItem("_Salir",    "Cierra la aplicación", () => Application.RequestStop()),
        })
    }
};
```

### `TableView`

Tabla navegable por teclado. En v2 es una opción muy útil cuando la UI muestra filas y columnas.

```csharp
using System.Data;

var dt = new DataTable();
dt.Columns.Add("Nombre");
dt.Columns.Add("Apellido");
dt.Rows.Add("Ada", "Lovelace");
dt.Rows.Add("Alan", "Turing");

var tabla = new TableView {
    X = 0, Y = 2,
    Width = Dim.Fill(), Height = Dim.Fill(1),
    Table = new DataTableSource(dt),
    FullRowSelect = true,
};

int filaSeleccionada = tabla.SelectedRow;
```

Si cambia el contenido del `DataTable`, se puede pedir redibujado con:

```csharp
tabla.SetNeedsDraw();
```

## 5. Eventos y teclado en v2

En v2 varios ejemplos cambian respecto de v1:

| Widget / contenedor | Evento          | Uso típico                               |
| ------------------- | --------------- | ---------------------------------------- |
| `Button`            | `Accepting`     | Ejecutar la acción principal             |
| `TextField`         | `TextChanged`   | Reaccionar a cambios de texto            |
| `View` / `Toplevel` | `KeyDown`       | Atajos de teclado                        |
| `Toplevel`          | `Closing`       | Confirmar salida o cancelar el cierre    |

**`TextChanged` — búsqueda en tiempo real:**

```csharp
txtBusqueda.TextChanged += (_, _) => {
    var termino = (txtBusqueda.Text ?? "").Trim();
    AplicarFiltro(termino);
};
```

**`KeyDown` — atajos globales:**

```csharp
top.KeyDown += (_, key) => {
    if (key.KeyCode == KeyCode.F2) {
        AgregarContacto();
        key.Handled = true;
    } else if (key == Key.S.WithCtrl) {
        Guardar();
        key.Handled = true;
    }
};
```

En v2 es habitual distinguir:

- `key.KeyCode == KeyCode.Enter` para teclas especiales.
- `key == Key.S.WithCtrl` para combinaciones con modificadores.

**`Closing` — evitar salir si hay cambios sin guardar:**

```csharp
top.Closing += (_, e) => {
    if (HayCambiosSinGuardar()) {
        e.Cancel = true;
    }
};
```

**Foco:**

```csharp
txtBusqueda.SetFocus();
```

## 6. Diálogos modales

Un `Dialog` bloquea la interacción con el resto de la interfaz hasta que se cierra.

```csharp
string? resultado = null;

var dlg = new Dialog {
    Title = "Nuevo contacto",
    Width = 60, Height = 10,
};

var txtNombre = new TextField {
    X = 12, Y = 0,
    Width = Dim.Fill(2),
};

dlg.Add( 
    new Label { Text = "Nombre:", X = 1, Y = 0 },
    txtNombre
);

var btnOk = new Button { Text = "_Aceptar", IsDefault = true };
btnOk.Accepting += (_, _) => {
    resultado = (txtNombre.Text ?? "").Trim();
    Application.RequestStop();
};

var btnCancel = new Button { Text = "_Cancelar" };
btnCancel.Accepting += (_, _) => {
    resultado = null;
    Application.RequestStop();
};

dlg.AddButton(btnOk);
dlg.AddButton(btnCancel);

Application.Run(dlg);
dlg.Dispose();
```

La técnica para “devolver” datos desde un diálogo sigue siendo la misma: guardar el resultado en una variable antes de cerrar.

**Diálogos rápidos con `MessageBox`:**

```csharp
int r = MessageBox.Query("Confirmar", "¿Eliminar contacto?", "Sí", "No");
if (r == 0) {
    BorrarContacto();
}

MessageBox.ErrorQuery("Error", "No se pudo guardar.", "Aceptar");
```

## 7. Resumen

```text
¿Qué aporta Terminal.Gui v2?
    └─► Permite construir interfaces interactivas dentro de la terminal
    └─► Usa ventanas, tablas, menús, atajos y diálogos sin GUI tradicional

Estructura básica:
    ├── `Application.Init()` inicializa la TUI
    ├── `Toplevel` actúa como raíz
    ├── `Application.Run(top)` entra al loop de eventos
    └── `Application.Shutdown()` restaura la terminal al salir

Layout:
    ├── `Pos` define posición
    │   ├── valores fijos
    │   ├── referencias a otros widgets (`Right`, `Bottom`, `Top`)
    │   └── `AnchorEnd` para pegar al borde final
    └── `Dim` define tamaño
        ├── valores fijos
        └── `Dim.Fill()` / `Dim.Fill(n)` para ocupar el espacio restante

Widgets frecuentes:
    ├── `Toplevel`  raíz visual de la aplicación
    ├── `Window`    contenedor principal
    ├── `Label`     texto estático
    ├── `TextField` entrada editable; `.Text` se usa como string
    ├── `Button`    acción disparada típicamente por `Accepting`
    ├── `MenuBar`   comandos globales
    └── `TableView` tabla navegable por filas y columnas

Eventos:
    ├── `TextChanged` reacciona a cambios de texto
    ├── `KeyDown`     permite atajos globales o locales
    ├── `Closing`     permite cancelar el cierre
    └── `Handled`     evita que una tecla siga propagándose

Diálogos:
    ├── `Dialog` + `Application.Run(dlg)` crean una ventana modal
    ├── `AddButton(...)` agrega acciones estándar al diálogo
    └── `Application.RequestStop()` cierra el diálogo o la app actual

Reglas prácticas:
    ├── Mantener el orden `Init → Run → Shutdown`
    ├── Para datos tabulares, `TableView` + `DataTableSource` simplifica mucho
    ├── Usar `KeyCode` para teclas especiales y `WithCtrl` para combinaciones
    └── Terminal.Gui sigue trabajando con caracteres, no con píxeles
```

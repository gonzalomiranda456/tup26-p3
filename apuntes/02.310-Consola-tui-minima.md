# Cómo funciona la mini TUI...

Este tutorial explica el programa [23.6-consola-nano-tui.cs](..\clases\23.6-consola-nano-tui.cs), una TUI mínima hecha solo con funciones de `Console`.

La meta del programa es simple:
- mantener una agenda en memoria
- mostrar un menú principal
- navegar con teclado
- listar contactos
- cargar un contacto nuevo con validación

No usa librerías externas. Toda la interfaz se arma con:
- `Clear()`
- `Write()` y `WriteLine()`
- `ReadKey(true)`
- `ReadLine()`
- `SetCursorPosition()`
- `ForegroundColor` y `ResetColor()`
- `CursorTop`, `CursorLeft` y `WindowWidth`

## 1. Estructura general

El programa define primero una agenda de contactos:

```csharp
List<Contacto> agenda = [
    new("Analia",   "Altamira",  5551234, 36),
    new("Beatriz",  "Benitez",   5555678, 41),
    new("Carlos",   "Córdoba",   5559012, 16),
    new("Diego",    "Dominguez", 5553456, 28),
    new("Elena",    "Espósito",  5557890, 22),
    new("Federico", "Fernandez", 5552345, 31),
    new("Gabriela", "Garcia",    5556789, 27),
    new("Hector",   "Hernandez", 5550123, 45),
];
```

Luego entra en el bucle principal:

```csharp
while(true) {
    Clear();
    Titulo("Menú Principal");
    int opcion = Menu("Listar Contactos", "Agregar Contacto", "Editar Contacto", "Borrar Contacto", "Salir");
    switch(opcion) {
        case 0: ListarContactos(); break;
        case 1: AgregarContacto(); break;
        case 2: EditarContacto();  break;
        case 3: BorrarContacto();  break;
        case 4:
        case -1:
            WriteLine("\nChau!");
            return;
    }
}
```

Ese bucle organiza toda la interfaz:
- limpia la pantalla actual
- dibuja el título
- muestra el menú
- ejecuta la opción elegida

## 2. Preparación de la consola

Antes del bucle principal, el programa configura la salida Unicode:

```csharp
Console.OutputEncoding = System.Text.Encoding.UTF8;
```

Eso permite que caracteres como `Córdoba`, `Espósito` o `Menú` se vean bien en pantalla.

## 3. Título de cada pantalla

Cada pantalla usa la misma función:

```csharp
void Titulo(string mensaje) {
    ForegroundColor = ConsoleColor.DarkBlue;
    WriteLine($"\n=== {mensaje} ===");
    ResetColor();
}
```

Acá se usan dos funciones importantes de `Console`:
- `ForegroundColor` para cambiar el color del título
- `WriteLine()` para imprimirlo

## 4. Menú con teclado

La función `Menu` dibuja las opciones y espera la tecla del usuario:

```csharp
int Menu(params string[] opciones) {
    int y = CursorTop;
    int pos = 0;
    while(true) {
        SetCursorPosition(0, y);
        for(int i = 0; i < opciones.Length; i++) {
            MostrarCursor($"{i + 1}. {opciones[i]}", i == pos);
        }

        var key = ReadKey(true).Key;
        if(key == ConsoleKey.Escape) return -1;
        if(key == ConsoleKey.Enter)  return pos;

        MoverCursor(key, opciones.Length, ref pos);
    }
}
```

Lo importante acá es:
- `CursorTop` guarda la fila inicial del menú
- `SetCursorPosition(0, y)` vuelve siempre al mismo bloque de líneas
- `ReadKey(true)` lee teclas sin mostrarlas

## 5. Cómo se marca la opción actual

Cada línea del menú se imprime con `MostrarCursor`:

```csharp
void MostrarCursor(string texto, bool seleccionada) {
    ForegroundColor = seleccionada ? ConsoleColor.Green : ConsoleColor.Gray;
    WriteLine($" {(seleccionada ? ">" : " ")} {texto}");
    ResetColor();
}
```

Esta función define el aspecto de la selección:
- verde para la opción activa
- gris para las demás
- un prefijo `>` como cursor visual

## 6. Cómo se mueve la selección

La lógica de navegación está separada en `MoverCursor`:

```csharp
void MoverCursor(ConsoleKey key, int cantidad, ref int actual) {
    actual = key switch {
        ConsoleKey.UpArrow   => actual - 1,
        ConsoleKey.DownArrow => actual + 1,
        >= ConsoleKey.D0 and <= ConsoleKey.D9 => key - ConsoleKey.D1,
        _ => 0
    };
    actual = Math.Clamp(actual, 0, cantidad - 1);
}
```

Eso permite dos modos de uso:
- flechas arriba y abajo
- teclas numéricas

## 7. Pantalla de listado

La función `ListarContactos` muestra una lista navegable:

```csharp
void ListarContactos() {
    Clear();
    Titulo("LISTA DE CONTACTOS");

    var y = CursorTop;
    var pos = 0;
    while(true){
        Console.SetCursorPosition(0, y);
        for(int i = 0; i < agenda.Count; i++) {
            var c = agenda[i];
            MostrarCursor($"{i + 1}. {c.NombreCompleto.PadRight(30)} | {c.Telefono.ToString("000-0000").PadRight(15)} | {c.Edad.ToString().PadLeft(3)} años", i == pos);
        }

        var key = ReadKey(true).Key;
        if(key == ConsoleKey.Escape || key == ConsoleKey.Enter) { return; }
        MoverCursor(key, agenda.Count, ref pos);
    }
}
```

Acá se combinan varias técnicas:
- `Clear()` para entrar a otra pantalla
- `CursorTop` para recordar desde qué fila redibujar
- `SetCursorPosition()` para volver siempre al inicio de la lista
- `PadRight()` y `PadLeft()` para alinear columnas

## 8. Alta de contacto

La función `AgregarContacto` usa un lector genérico para pedir los datos:

```csharp
void AgregarContacto() {
    Clear();
    Titulo("Agregar Contacto");

    Contacto nuevo = new(
        Nombre:   Leer<string>("Nombre",   TryParseNombre),
        Apellido: Leer<string>("Apellido", TryParseNombre),
        Telefono: Leer<int>("Teléfono",    TryParseNumero),
        Edad:     Leer<int>("Edad",        TryParseEdad)
    );

    agenda.Add(nuevo);
    WriteLine();
    MostrarContacto(nuevo);
    Pausa("Contacto agregado. \n\nPresione una tecla para continuar...");
}
```

Lo interesante es que el mismo lector sirve para distintos tipos (`string` e `int`).

## 9. Lector genérico

La función `Leer<T>` arma el formulario campo por campo:

```csharp
T Leer<T>(string mensaje, Convertidor<T> convertir) {
    Console.ForegroundColor = ConsoleColor.Cyan;
    Write($"{mensaje.PadLeft(12)}: ");
    ResetColor();

    int y = CursorTop;

    while (true) {
        Console.SetCursorPosition(14, y);
        LimpiarLinea();

        var linea = ReadLine() ?? "";
        if (convertir(linea, out var valor)) {
            LimpiarLinea();
            return valor;
        }
    }
}
```

Paso a paso:
1. Escribe la etiqueta del campo.
2. Guarda la fila actual con `CursorTop`.
3. Mueve el cursor a la columna 14, donde empieza la entrada.
4. Limpia la línea actual.
5. Lee con `ReadLine()`.
6. Intenta convertir el valor.
7. Si la conversión es válida, devuelve el resultado.

## 10. Limpiar una línea

Para reutilizar la misma zona de la pantalla, el programa usa `LimpiarLinea`:

```csharp
void LimpiarLinea() {
    int x = CursorLeft, y = CursorTop;
    Write(new string(' ', WindowWidth - 1));
    Console.SetCursorPosition(x, y);
}
```

Esto borra la línea actual sin perder la posición del cursor.

## 11. Mensajes de error

Los errores se escriben debajo del campo actual:

```csharp
void MostrarError(string mensaje = "") {
    Console.SetCursorPosition(14, CursorTop);
    LimpiarLinea();
    ForegroundColor = ConsoleColor.Red;
    Write(mensaje);
    ResetColor();
}
```

La idea es:
- mover el cursor a la zona del mensaje
- limpiar el contenido anterior
- escribir el error en rojo

## 12. Validadores

Cada tipo de dato tiene su propia validación.

Para nombre:

```csharp
bool TryParseNombre(string texto, out string valor) {
    valor = texto.Trim().ToTitle();
    if( !string.IsNullOrWhiteSpace(valor)) { return true; }
    MostrarError("El nombre no puede estar vacío.");
    return false;
}
```

Para teléfono:

```csharp
bool TryParseNumero(string texto, out int valor) {
    if (int.TryParse(texto, out valor)) { return true; }
    MostrarError("Por favor, ingrese un número válido.");
    return false;
}
```

Para edad:

```csharp
bool TryParseEdad(string texto, out int valor) {
    if (int.TryParse(texto, out valor) && valor >= 18 && valor <= 65) { return true; }
    MostrarError("Por favor, ingrese una edad válida (entre 18 y 65).");
    return false;
}
```

## 13. Mostrar un contacto

Después de dar de alta un contacto, el programa lo vuelve a mostrar:

```csharp
void MostrarContacto(Contacto contacto) {
    WriteLine($"      Nombre: {contacto.Nombre}");
    WriteLine($"    Apellido: {contacto.Apellido}");
    WriteLine($"    Teléfono: {contacto.Telefono.ToTelefono()}");
    WriteLine($"        Edad: {contacto.Edad}");
}
```

## 14. Pausa entre pantallas

La función `Pausa` detiene el flujo hasta que el usuario presione una tecla:

```csharp
void Pausa(string mensaje = "Presione una tecla para continuar...") {
    WriteLine();
    ForegroundColor = ConsoleColor.DarkGray;
    Write(mensaje);
    ResetColor();
    ReadKey(true);
    LimpiarLinea();
}
```

Esto se usa después de mostrar resultados o mensajes temporales.

## 15. Extensiones de formato

El programa usa dos extensiones para normalizar la salida:

```csharp
static class Formatos {
    public static string ToTitle(this string texto) {
        CultureInfo cultura = new("es-AR");
        return cultura.TextInfo.ToTitleCase(texto.ToLower(cultura));
    }

    public static string ToTelefono(this int valor) {
        return valor.ToString("000-0000");
    }
}
```

## 16. Delegado y modelo

El lector genérico se apoya en este delegado:

```csharp
delegate bool Convertidor<T>(string texto, out T valor);
```

Y los datos se representan con este `record`:

```csharp
record Contacto(string Nombre, string Apellido, int Telefono, int Edad) {
    public string NombreCompleto => $"{Apellido}, {Nombre}";
    public override string ToString() => $"{Nombre} {Apellido} - {Telefono.ToTelefono()} - {Edad} años";
}
```

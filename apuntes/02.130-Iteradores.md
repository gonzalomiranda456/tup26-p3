# Iteradores en C#: `IEnumerable`, `IEnumerator` y `yield`

## El problema

`foreach` funciona sobre cualquier colección porque estas implementan `IEnumerable<T>`. El `foreach` se traduce internamente a:

```csharp
IEnumerator<T> e = coleccion.GetEnumerator();
try {
    while (e.MoveNext()) {
        var x = e.Current;
        // cuerpo del foreach
    }
} finally {
    e.Dispose();
}
```

## Las interfaces

```csharp
public interface IEnumerable<out T> {
    IEnumerator<T> GetEnumerator(); // devuelve un iterador nuevo
}

public interface IEnumerator<out T> : IDisposable {
    T    Current  { get; }  // elemento actual
    bool MoveNext();        // avanza; false si no hay más
}
```

- `IEnumerable` = "se me puede recorrer" (la colección)
- `IEnumerator` = "llevo la cuenta del recorrido" (el iterador)

`GetEnumerator()` devuelve un objeto **nuevo** cada vez → se pueden tener varios recorridos simultáneos sin interferencia.

## Implementación manual

```csharp
public class RangoEntero : IEnumerable<int> {
    readonly int inicio, fin;
    public RangoEntero(int inicio, int fin) { this.inicio = inicio; this.fin = fin; }

    public IEnumerator<int> GetEnumerator() =>
        new RangoEnumerador(inicio, fin);
    IEnumerator IEnumerable.GetEnumerator() => GetEnumerator();
}

public class RangoEnumerador : IEnumerator<int> {
    readonly int inicio, fin;
    int actual; bool iniciado;

    public RangoEnumerador(int i, int f) { inicio = i; fin = f; }
    public int  Current => actual;
    object IEnumerator.Current => Current;
    public void Reset()   { iniciado = false; }
    public void Dispose() { }

    public bool MoveNext() {
        if (iniciado) actual++;
        else { iniciado = true; actual = inicio; }
        return actual <= fin;
    }
}
```

Esto es mucho código. `yield return` lo simplifica enormemente.

## `yield return` — la forma moderna

El compilador genera la clase enumerador por vos:

```csharp
public class RangoEntero : IEnumerable<int> {
    readonly int inicio, fin;
    public RangoEntero(int i, int f) { inicio = i; fin = f; }

    public IEnumerator<int> GetEnumerator() {
        for (int i = inicio; i <= fin; i++)
            yield return i;   // "pausar aquí, devolver i, continuar en el próximo MoveNext()"
    }

    IEnumerator IEnumerable.GetEnumerator() => GetEnumerator();
}

foreach (int n in new RangoEntero(1, 5))
    Console.WriteLine(n); // 1 2 3 4 5
```

También como método estático que devuelve `IEnumerable<T>`:

```csharp
static IEnumerable<int> Rango(int inicio, int fin) {
    for (int i = inicio; i <= fin; i++)
        yield return i;
}

// Secuencia infinita — solo consume lo que se necesita
static IEnumerable<int> NumerosNaturales() {
    int n = 1;
    while (true) yield return n++;
}

// Tomar los primeros 5 pares
var pares = NumerosNaturales().Where(n => n % 2 == 0).Take(5);
```

`yield break` detiene el iterador antes de terminar:

```csharp
static IEnumerable<int> HastaMaximo(IEnumerable<int> fuente, int max) {
    foreach (int n in fuente) {
        if (n > max) yield break; // detener
        yield return n;
    }
}
```

## Evaluación diferida (lazy)

Los métodos con `yield` **no ejecutan nada al llamarlos**. Solo ejecutan cuando se itera el resultado. Esto permite trabajar con secuencias potencialmente infinitas y encadenar transformaciones sin materializar resultados intermedios:

```csharp
var resultado = NumerosNaturales()
    .Where(n => n % 2 == 0)  // no ejecuta aún
    .Select(n => n * n)       // no ejecuta aún
    .Take(5)                  // no ejecuta aún
    .ToList();                // acá sí ejecuta todo
```

## Cuándo usar `yield`

- Secuencias que se calculan sobre la marcha (no están guardadas)
- Secuencias potencialmente infinitas
- Evitar materializar toda la colección en memoria cuando solo se necesita una parte
- Recorrer fuentes externas (archivos, bases de datos) línea por línea

## Resumen

|                      | `IEnumerable<T>`  | `IEnumerator<T>`         |
| :------------------- | :---------------- | :----------------------- |
| Rol                  | La colección      | El iterador              |
| Método principal     | `GetEnumerator()` | `MoveNext()` / `Current` |
| Una instancia por... | Colección         | Recorrido activo         |

```
¿Quiero que mi clase sea recorrible con foreach?  → implementar IEnumerable<T>
¿Quiero implementar el recorrido sin boilerplate? → usar yield return
¿Quiero generar una secuencia perezosa?           → método que devuelve IEnumerable<T> con yield
¿Quiero detener el iterador antes?                → yield break
```

# Interfaces y contratos en C#

Una **interfaz** define un **contrato**: dice qué puede hacer un tipo, sin decir cómo. Expresa "puede hacer…" en lugar de "es un…" (que es la herencia).

## Sintaxis básica

```csharp
public interface ILogger {
    void Log(string mensaje);
    void LogError(string mensaje, Exception? ex = null);
    bool EstaActivo { get; }
}

public class ConsoleLogger : ILogger {
    public bool EstaActivo => true;
    public void Log(string mensaje)                   => Console.WriteLine($"[INFO] {mensaje}");
    public void LogError(string mensaje, Exception? ex = null) => Console.WriteLine($"[ERROR] {mensaje}");
}

// Uso polimórfico — el código trabaja con la interfaz
ILogger log = new ConsoleLogger();
log.Log("Iniciando");
```

Una interfaz puede tener métodos, propiedades y eventos — sin campos ni constructores.

## Implementación múltiple

Un tipo puede implementar **múltiples interfaces** (ventaja clave sobre clases abstractas):

```csharp
public interface ILegible   { string Leer(); }
public interface IEscribible { void Escribir(string s); }

public class Buffer : ILegible, IEscribible {
    private readonly StringBuilder sb = new();
    public string Leer()                  => sb.ToString();
    public void   Escribir(string s)      => sb.Append(s);
}
```

## Métodos con implementación por defecto

Una interfaz puede proveer implementaciones opcionales que las clases pueden sobreescribir:

```csharp
public interface IFormateador {
    string Formatear(string texto);  // obligatorio

    // opcional — la clase puede sobreescribirlo o usar este
    string FormatearLista(IEnumerable<string> items) =>
        string.Join(", ", items.Select(Formatear));
}
```

Los métodos por defecto solo son accesibles a través del tipo de la interfaz, no de la clase concreta.

## Interfaces vs. clases abstractas

| | Interfaz | Clase abstracta |
|---|---|---|
| Herencia múltiple | Sí | No |
| Puede tener estado (campos) | No | Sí |
| Constructores | No | Sí |
| Expresa "puede hacer..." | Sí | Indirecto |
| Expresa "es un tipo de..." | No | Sí |

```
¿Comparten estado y lógica base?          → clase abstracta
¿Comparten solo un contrato de comportamiento? → interfaz
¿Necesita cumplir múltiples contratos?    → múltiples interfaces
```

## Interfaces genéricas

```csharp
public interface IRepositorio<T> {
    T?       ObtenerPorId(int id);
    IList<T> ObtenerTodos();
    void     Agregar(T entidad);
    void     Eliminar(int id);
}

public class ProductoRepositorio : IRepositorio<Producto> {
    private readonly List<Producto> datos = [];

    public Producto? ObtenerPorId(int id) => datos.FirstOrDefault(p => p.Id == id);
    public IList<Producto> ObtenerTodos() => datos.AsReadOnly();
    public void Agregar(Producto p)        => datos.Add(p);
    public void Eliminar(int id)           => datos.RemoveAll(p => p.Id == id);
}
```

Restricciones en genéricos con `where`:

```csharp
// T debe implementar IComparable<T>
static T Maximo<T>(T a, T b) where T : IComparable<T> =>
    a.CompareTo(b) >= 0 ? a : b;

// T debe ser clase y tener constructor sin parámetros
static T Crear<T>() where T : class, new() => new T();
```

## Interfaces de .NET más usadas

| Interfaz | Para qué |
|---|---|
| `IEnumerable<T>` | Poder recorrer con `foreach` |
| `IComparable<T>` | Comparación para ordenamiento |
| `IComparer<T>` | Criterio de comparación externo |
| `IEquatable<T>` | Igualdad por valor |
| `IDisposable` | Liberar recursos con `using` |
| `IAsyncDisposable` | Liberar recursos async |

```csharp
public class Gestor : IDisposable {
    private bool disposed = false;

    public void Dispose() {
        if (!disposed) {
            // liberar recursos
            disposed = true;
        }
    }
}

using var g = new Gestor(); // Dispose() se llama al salir del bloque
```

## Ejemplo: clases abstracta + interfaces combinadas

```csharp
public abstract class Animal(string nombre) {
    public string Nombre => nombre;
    public abstract void HacerSonido();
}

public interface IEntrenable { void Entrenar(); }
public interface IVacunable  { void Vacunar(); }

// Perro ES un Animal Y PUEDE SER entrenado y vacunado
public class Perro(string nombre) : Animal(nombre), IEntrenable, IVacunable {
    public override void HacerSonido() => Console.WriteLine("Guau!");
    public void Entrenar() => Console.WriteLine($"{Nombre} aprende un truco");
    public void Vacunar()  => Console.WriteLine($"{Nombre} fue vacunado");
}
```

## Resumen

| Necesidad | Herramienta |
|---|---|
| Definir comportamiento sin implementación | interfaz |
| Compartir estado y lógica base | clase abstracta |
| Cumplir múltiples contratos | múltiples interfaces |
| Contrato para un tipo genérico | interfaz genérica |
| Restringir tipo genérico | `where T : IAlgo` |
| Liberar recursos | `IDisposable` + `using` |

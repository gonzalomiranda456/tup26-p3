# REST API vs. MCP: Entendiendo la Revolución de las APIs para Inteligencia Artificial

Tanto **REST** como **MCP** son estándares para comunicar sistemas de software a través de una red o un límite de proceso. Sin embargo, nacieron en épocas distintas y fueron diseñados para **consumidores completamente diferentes**: REST fue hecho para programas escritos por humanos, mientras que MCP fue diseñado para Modelos de Lenguaje (LLMs) y Agentes de IA.

---

## 1. El Primer Principio (La Filosofía Detrás de Cada Uno)

### El Principio de REST (Representational State Transfer)
> **"Determinismo y Contratos Estrictos"**
* **Objetivo:** Permitir que una aplicación cliente ejecute operaciones específicas en un servidor remoto de la forma más predecible y estructurada posible.
* **El Enfoque:** El programador humano del cliente lee la documentación de la API (ej. Swagger/OpenAPI), escribe código explícito indicando cuándo y cómo llamar a cada endpoint (ej. `GET /clientes/5`), y define cómo procesar exactamente la respuesta JSON. Todo el flujo es determinista: si pasa A, el programa llama a B.

### El Principio de MCP (Model Context Protocol)
> **"Descubrimiento Dinámico y Autonomía de la IA"**
* **Objetivo:** Darle a un modelo de IA (LLM) acceso a datos, herramientas y prompts del mundo real de forma dinámica para que el modelo decida por sí mismo qué hacer.
* **El Enfoque:** El cliente MCP (el "cerebro" de la IA) no sabe de antemano qué funciones existen. Al conectarse, el servidor le provee una lista de capacidades descritas en lenguaje natural (ej. *"Esta herramienta lee archivos de texto"*). La IA evalúa la intención del usuario y **decide de forma autónoma** cuál herramienta llamar, en qué orden y qué argumentos pasarle.

---

## 2. Similitudes entre REST y MCP

* **Intercambio de Datos:** Ambos utilizan **JSON** como el formato estándar para enviar y recibir información debido a su ligereza y legibilidad.
* **Desacoplamiento:** Ambos separan la lógica del backend (donde están las bases de datos o el sistema de archivos) del cliente (la interfaz web o el agente inteligente).
* **Independencia Tecnológica:** Puedes implementar tanto servidores REST como MCP en cualquier lenguaje (C#, Python, Node.js, Go) e interactuar con clientes escritos en tecnologías distintas.
* **Transporte Web:** Ambos pueden funcionar sobre HTTP utilizando la infraestructura web tradicional.

---

## 3. Diferencias Clave

| Característica | API REST | MCP (Model Context Protocol) |
| :--- | :--- | :--- |
| **Consumidor Primario** | Un programa informático (flujo rígido codificado por un humano). | Un Modelo de Lenguaje / Agente de IA (flujo dinámico por razonamiento). |
| **Protocolo de Mensajería** | HTTP crudo (métodos GET, POST, PUT, DELETE, códigos de estado 200, 404, 500). | **JSON-RPC 2.0** (mensajes estructurados con campos fijos: `jsonrpc`, `id`, `method`, `params`). |
| **Tipo de Conexión** | **Sin estado (Stateless):** Cada petición es independiente. Se abre y se cierra la conexión HTTP. | **Con estado / Persistente:** Conexión continua bidireccional (vía `stdio` o HTTP con streaming/SSE). |
| **Documentación** | Estática (OpenAPI/Swagger) destinada a que la lea el **programador**. | Semántica (descripciones en lenguaje natural dentro del esquema JSON) para que la lea la **IA**. |
| **Dirección del Flujo** | Unidireccional: el cliente siempre inicia la petición al servidor. | **Bidireccional:** El cliente puede pedir herramientas al servidor, y el servidor puede pedirle al cliente que procese texto (sampling). |

## 4. Un Ejemplo Comparativo Práctico

Imagina que quieres consultar el clima de una ciudad y mostrar una recomendación.

### El Enfoque REST
1. El programador lee la documentación de OpenWeather y ve que el endpoint es `/weather?q={city}`.
2. El programador escribe en C#:
   ```csharp
   // El flujo está "hardcodeado" (cableado) por el desarrollador
   if (usuarioPresionoBoton) {
       var clima = await httpClient.GetFromJsonAsync<Clima>("/weather?q=Rosario");
       if (clima.Temperatura > 30) {
           MostrarMensaje("Lleva agua!");
       }
   }
   ```

### El Enfoque MCP
1. El programador crea un servidor MCP que expone una herramienta llamada `obtener_clima` y la describe: *"Devuelve la temperatura actual de una ciudad"*.
2. El programador conecta la IA a este servidor.
3. El usuario le escribe a la IA: *"Hola, voy a viajar a Rosario hoy, ¿qué me sugieres?"*.
4. La IA (no el código del programa) razona:
   * *La pregunta del usuario requiere saber el clima de Rosario.*
   * *Reviso mis herramientas MCP... ¡Ah! Tengo `obtener_clima`.*
   * *Llamo a `obtener_clima` pasándole `"Rosario"` en el parámetro.*
5. El servidor MCP responde con la temperatura (`32°C`).
6. La IA recibe el dato y razona: *"Hace calor en Rosario. Le sugeriré que use ropa ligera y lleve agua"*.

---

## 5. Conclusión: ¿Cuál elegir?

* Usa una **API REST** cuando quieras automatizar procesos rígidos, de alta velocidad y donde el flujo de control deba ser 100% predecible (ej. pasarelas de pago, registros de usuarios, endpoints móviles).
* Usa **MCP** cuando estés construyendo integraciones de Inteligencia Artificial (copilotos de código, agentes autónomos de soporte, análisis de datos interactivos) y necesites que el modelo interactúe dinámicamente con el entorno del usuario.

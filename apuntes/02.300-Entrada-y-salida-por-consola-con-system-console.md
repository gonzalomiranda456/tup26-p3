# Entrada/salida por consola, archivos de texto y encoding

En C#, la entrada/salida más directa aparece en dos lugares:

- `Console`, para leer y escribir en la terminal;
- `File`, `Directory` y `Path`, para trabajar con archivos, carpetas y rutas.

`Encoding` atraviesa ambos mundos: define cómo se convierten bytes en texto y texto en bytes.

## Console: leer y escribir

`Console` es una clase estática para leer y escribir en la terminal, controlar colores y cursor, y trabajar con los streams estándar.

```csharp
// Escribir
Console.Write("Sin salto");
Console.WriteLine("Con salto");

// Leer
string? linea = Console.ReadLine();         // espera Enter
ConsoleKeyInfo tecla = Console.ReadKey();   // sin Enter, captura la tecla
```

## Colores y cursor

```csharp
Console.ForegroundColor = ConsoleColor.Cyan;
Console.BackgroundColor = ConsoleColor.DarkBlue;
Console.WriteLine("Texto con color");
Console.ResetColor();

Console.Clear();
Console.CursorVisible = false;
Console.SetCursorPosition(col: 10, top: 5);
var (col, row) = Console.GetCursorPosition();
```

## Streams estándar

Un programa de consola no tiene una sola "salida". Tiene tres streams estándar:

| Stream          | Descripción                              |
|-----------------|------------------------------------------|
| `Console.In`    | Entrada estándar: teclado o pipe         |
| `Console.Out`   | Salida estándar: pantalla o redirección  |
| `Console.Error` | Salida de error separada de `stdout`     |

```csharp
// Leer toda la entrada (útil con pipe)
string todo = Console.In.ReadToEnd();

// Escribir errores sin contaminar la salida normal
Console.Error.WriteLine("Algo salió mal");
```

Esta separación importa cuando un programa se usa desde scripts: la salida normal puede procesarse con otro comando y los errores pueden ir por otro camino.

## Redirección desde shell

La terminal puede conectar los streams estándar con archivos u otros programas:

```bash
dotnet run < entrada.txt        # stdin desde archivo
dotnet run > salida.txt         # stdout a archivo
dotnet run 2> error.txt         # stderr a archivo
dotnet run < entrada.txt | grep "x"
```

El programa no necesita saber si `Console.In` viene del teclado o de un archivo. Lee el mismo stream.

## File: leer y escribir archivos de texto

`File` opera sobre archivos. Para texto, los métodos más usados son `ReadAllText`, `WriteAllText` y `AppendAllText`.

```csharp
using System.IO;
using System.Text;

// Leer todo el archivo
string texto = File.ReadAllText("datos.txt");
string textoUtf8 = File.ReadAllText("datos.txt", Encoding.UTF8);

// Escribir (reemplaza el contenido)
File.WriteAllText("salida.txt", "Hola mundo", Encoding.UTF8);

// Agregar al final
File.AppendAllText("log.txt", "Nueva línea\n", Encoding.UTF8);

// Verificar existencia y borrar
if (File.Exists("viejo.txt")) {
    File.Delete("viejo.txt");
}
```

Los paths relativos, como `"datos.txt"` o `"datos/entrada.txt"`, se interpretan desde el directorio de trabajo actual del proceso. En proyectos de consola, ese directorio no siempre coincide con la carpeta donde está el `.cs`.

## Directory: carpetas

`Directory` opera sobre carpetas.

```csharp
using System.IO;

// Crear (no falla si ya existe)
Directory.CreateDirectory("datos/salidas/2026");

// Listar archivos
string[] todos = Directory.GetFiles("mis-archivos");
string[] soloTxt = Directory.GetFiles("mis-archivos", "*.txt");

// Verificar y borrar
if (Directory.Exists("temp")) {
    Directory.Delete("temp", recursive: true);
}
```

`recursive: true` borra también el contenido de la carpeta. Hay que usarlo con cuidado.

## Path: construir rutas

`Path` no lee ni escribe archivos. Construye y analiza rutas de forma portable entre sistemas operativos.

```csharp
using System.IO;

// Usar Combine en vez de concatenar con "/" o "\"
string ruta = Path.Combine("datos", "clientes", "enero.txt");

string archivo = Path.GetFileName(ruta);                  // "enero.txt"
string nombre  = Path.GetFileNameWithoutExtension(ruta);  // "enero"
string ext     = Path.GetExtension(ruta);                 // ".txt"
string? dir    = Path.GetDirectoryName(ruta);             // carpeta contenedora
string abs     = Path.GetFullPath(ruta);                  // ruta absoluta
string bak     = Path.ChangeExtension(ruta, ".bak");      // cambia extensión
string tmp     = Path.GetTempPath();                      // carpeta temporal del sistema
```

La regla práctica es: construir rutas con `Path.Combine`, no con strings pegados a mano.

## Encoding

Un archivo es bytes. El encoding es la regla para convertir bytes en caracteres y caracteres en bytes. Si leés un UTF-8 como Latin1, o al revés, aparecen caracteres rotos: `mañana` puede verse como `maÃ±ana`. Ese fenómeno se llama **mojibake**.

```csharp
using System.Text;

// Regla práctica: mismo encoding al escribir y al leer
File.WriteAllText("archivo.txt", contenido, Encoding.UTF8);
string texto = File.ReadAllText("archivo.txt", Encoding.UTF8);

// Para la consola: poner al inicio del programa
Console.InputEncoding = Encoding.UTF8;
Console.OutputEncoding = Encoding.UTF8;
```

| Encoding           | Cuándo usarlo                                    |
|--------------------|--------------------------------------------------|
| `Encoding.UTF8`    | Por defecto hoy en día; soporta cualquier idioma |
| `Encoding.Latin1`  | Archivos heredados en español/europeo            |
| `Encoding.Unicode` | UTF-16; frecuente en archivos de Windows         |
| `Encoding.ASCII`   | Solo inglés sin acentos                          |

## Ejemplo completo

Este programa lee un archivo, transforma su contenido y escribe otro archivo. Usa `Console.Out` para informar el resultado y `Console.Error` para avisar errores.

```csharp
using System.IO;
using System.Text;

Console.InputEncoding = Encoding.UTF8;
Console.OutputEncoding = Encoding.UTF8;

string carpeta = "datos";
string entrada = Path.Combine(carpeta, "entrada.txt");
string salida = Path.Combine(carpeta, "salida.txt");

Directory.CreateDirectory(carpeta);

if (File.Exists(entrada)) {
    string texto = File.ReadAllText(entrada, Encoding.UTF8);
    File.WriteAllText(salida, texto.ToUpper(), Encoding.UTF8);
    Console.WriteLine("Procesado.");
} else {
    Console.Error.WriteLine("No existe el archivo de entrada.");
}
```

## Resumen

| Necesidad             | Método                                             |
|-----------------------|----------------------------------------------------|
| Leer línea            | `Console.ReadLine()`                               |
| Leer tecla sin Enter  | `Console.ReadKey()`                                |
| Escribir sin salto    | `Console.Write(...)`                               |
| Escribir con salto    | `Console.WriteLine(...)`                           |
| Mensajes de error     | `Console.Error.WriteLine(...)`                     |
| Leer todo stdin       | `Console.In.ReadToEnd()`                           |
| Limpiar pantalla      | `Console.Clear()`                                  |
| Cambiar color         | `Console.ForegroundColor`, `Console.BackgroundColor` |
| Restaurar color       | `Console.ResetColor()`                             |
| Mover cursor          | `Console.SetCursorPosition(col, top)`              |
| Leer archivo          | `File.ReadAllText(path, encoding)`                 |
| Escribir archivo      | `File.WriteAllText(path, texto, encoding)`         |
| Agregar al final      | `File.AppendAllText(path, texto, encoding)`        |
| Existe archivo        | `File.Exists(path)`                                |
| Borrar archivo        | `File.Delete(path)`                                |
| Crear carpeta         | `Directory.CreateDirectory(path)`                  |
| Listar archivos       | `Directory.GetFiles(path, "*.ext")`                |
| Existe carpeta        | `Directory.Exists(path)`                           |
| Borrar carpeta        | `Directory.Delete(path, recursive: true)`          |
| Unir rutas            | `Path.Combine(parte1, parte2, ...)`                |

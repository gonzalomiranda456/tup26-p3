# Parser descendente recursivo y AST

En este tutorial vamos a construir un evaluador pequeño para expresiones aritméticas en C#.

La idea no es solamente calcular un número, sino separar con claridad dos problemas distintos:

1. **analizar** el texto de entrada;
2. **construir** una estructura que represente la expresión;
3. **evaluar** esa estructura tantas veces como queramos.

Ese enfoque aparece una y otra vez en compiladores, intérpretes, motores de reglas y hasta planillas.

La expresión de ejemplo será:

```csharp
1 + 2 * (8 / 4)
```

y el resultado esperado será:

```text
5
```

---

## Paso 1 — Pensar la gramática

Antes de escribir código conviene describir qué expresiones queremos aceptar.

En este caso vamos a soportar:

- números;
- operadores binarios `+`, `-`, `*`, `/`;
- operadores unarios `+` y `-`;
- paréntesis.

Una gramática simple para eso puede escribirse así:

```text
expresion := termino ('+' | '-' termino)*
termino   := factor ('*' | '/' factor)*
factor    := ('+' | '-') factor | '(' expresion ')' | numero
```

La parte importante es que la precedencia ya está codificada en la gramática:

- `*` y `/` se resuelven dentro de `termino`;
- `+` y `-` binarios se resuelven dentro de `expresion`;
- los operadores unarios viven dentro de `factor`.

Eso permite que una expresión como esta:

```text
1 + 2 * 3
```

se interprete como:

```text
1 + (2 * 3)
```

y no como:

```text
(1 + 2) * 3
```

---

## Paso 2 — Parsear con un parser descendente recursivo

Un parser descendente recursivo traduce cada regla de la gramática a una función.

La estructura general queda así:

```csharp
Nodo Parse(string s) {
    var pos = 0;

    char Current() => pos < s.Length ? s[pos] : '\0';
    char Consume() => s[pos++];

    bool Match(char c) {
        while (char.IsWhiteSpace(Current())) {
            Consume();
        }
        return Current() == c;
    }

    Nodo Expresion() {
        var izquierda = Termino();

        while (Match('+') || Match('-')) {
            var operador = Consume();
            var derecha = Termino();

            if (operador == '+') {
                izquierda = new Suma(izquierda, derecha);
            } else {
                izquierda = new Resta(izquierda, derecha);
            }
        }

        return izquierda;
    }

    Nodo Termino() {
        var izquierda = Factor();

        while (Match('*') || Match('/')) {
            var operador = Consume();
            var derecha = Factor();

            if (operador == '*') {
                izquierda = new Multiplicacion(izquierda, derecha);
            } else {
                izquierda = new Division(izquierda, derecha);
            }
        }

        return izquierda;
    }

    Nodo Factor() {
        if (Match('+') || Match('-')) {
            var signo = Consume();
            var nodo = Factor();
            return signo == '+' ? new Positivo(nodo) : new Negativo(nodo);
        }

        if (Match('(')) {
            Consume();
            var nodo = Expresion();
            if (!Match(')')) {
                throw new Exception("Se esperaba ')'");
            }
            Consume();
            return nodo;
        }

        if (char.IsDigit(Current())) {
            var valor = 0f;
            while (char.IsDigit(Current())) {
                valor = valor * 10 + (Consume() - '0');
            }
            return new Numero(valor);
        }

        throw new Exception($"Carácter inesperado: '{Current()}'");
    }

    return Expresion();
}
```

Este parser no evalúa directamente. En lugar de eso, **construye un árbol**.

Ese detalle cambia por completo el diseño.

## Paso 3 — Construir un AST

AST significa *Abstract Syntax Tree*, o árbol de sintaxis abstracta.

Por ejemplo, la expresión:

```text
1 + 2 * 3
```

puede representarse así:

```text
      +
     / \
    1   *
       / \
      2   3
```

Ese árbol ya no es texto: es una estructura de objetos.

En nuestro caso la jerarquía base puede empezar así:

```csharp
abstract class Nodo {
    public abstract float Evaluar();
}
```

Después aparecen dos familias muy naturales:

- nodos **unarios**;
- nodos **binarios**.

Podemos factorizar el comportamiento común en dos clases abstractas.

```csharp
abstract class Unario(string operador, Nodo operando) : Nodo {
    protected string Operador { get; } = operador;
    protected Nodo Operando { get; } = operando;

    public override string ToString() => $"({Operador} {Operando})";
}

abstract class Binario(string operador, Nodo izquierda, Nodo derecha) : Nodo {
    protected string Operador { get; } = operador;
    protected Nodo Izquierda { get; } = izquierda;
    protected Nodo Derecha { get; } = derecha;

    public override string ToString() => $"({Izquierda} {Operador} {Derecha})";
}
```

Y luego definimos los nodos concretos:

```csharp
class Numero(float valor) : Nodo {
    public override float Evaluar() => valor;
    public override string ToString() => valor.ToString();
}

class Positivo(Nodo operando) : Unario("+", operando) {
    public override float Evaluar() => Operando.Evaluar();
}

class Negativo(Nodo operando) : Unario("-", operando) {
    public override float Evaluar() => -Operando.Evaluar();
}

class Suma(Nodo izquierda, Nodo derecha) : Binario("+", izquierda, derecha) {
    public override float Evaluar() => Izquierda.Evaluar() + Derecha.Evaluar();
}

class Resta(Nodo izquierda, Nodo derecha) : Binario("-", izquierda, derecha) {
    public override float Evaluar() => Izquierda.Evaluar() - Derecha.Evaluar();
}

class Multiplicacion(Nodo izquierda, Nodo derecha) : Binario("*", izquierda, derecha) {
    public override float Evaluar() => Izquierda.Evaluar() * Derecha.Evaluar();
}

class Division(Nodo izquierda, Nodo derecha) : Binario("/", izquierda, derecha) {
    public override float Evaluar() => Izquierda.Evaluar() / Derecha.Evaluar();
}
```

## Paso 4 — Evaluar el árbol

Una vez construido el AST, evaluarlo se vuelve simple: cada nodo sabe calcular su propio valor a partir de sus hijos.

Por ejemplo:

```csharp
var expresion = Parse("1 + 2 * (8 / 4)");
Console.WriteLine(expresion.Evaluar());
```

El resultado es:

```text
5
```

Y si además imprimimos el árbol:

```csharp
Console.WriteLine(expresion);
```

obtenemos una forma parentizada que hace explícita la precedencia:

```text
(1 + (2 * (8 / 4)))
```

Eso es útil porque confirma que el parser entendió la expresión como queríamos.

## Paso 5 — Construir el AST manualmente

Una ventaja importante del árbol es que no depende del texto original.

Podemos construir exactamente la misma expresión a mano:

```csharp
var expresion = new Suma(
    new Numero(1),
    new Multiplicacion(
        new Numero(2),
        new Division(
            new Numero(8),
            new Numero(4)
        )
    )
);

Console.WriteLine($"{expresion} = {expresion.Evaluar()}");
```

La salida vuelve a ser:

```text
(1 + (2 * (8 / 4))) = 5
```

Eso demuestra que el AST es una representación independiente de cómo se escribió la expresión originalmente.

## Paso 6 — Agregar azúcar sintáctico

Si el objetivo es trabajar con árboles, escribir siempre `new Suma(...)` o `new Multiplicacion(...)` puede resultar verboso.

Podemos hacer el código más natural agregando operadores a `Nodo`:

```csharp
abstract class Nodo {
    public abstract float Evaluar();

    public static implicit operator Nodo(float valor) => new Numero(valor);
    public static Nodo operator +(Nodo a, Nodo b) => new Suma(a, b);
    public static Nodo operator -(Nodo a, Nodo b) => new Resta(a, b);
    public static Nodo operator *(Nodo a, Nodo b) => new Multiplicacion(a, b);
    public static Nodo operator /(Nodo a, Nodo b) => new Division(a, b);
    public static Nodo operator +(Nodo a) => new Positivo(a);
    public static Nodo operator -(Nodo a) => new Negativo(a);
}
```

Con eso ya podemos escribir:

```csharp
Nodo expresion = 1 + 2 * (8 / 4);
Console.WriteLine($"{expresion} = {expresion.Evaluar()}");
```

La expresión se construye como árbol, no se calcula inmediatamente.

Ese detalle es importante: el resultado no es `5`, sino un `Nodo` que representa la expresión completa.

---

## Paso 7 — Qué ganamos con este diseño

Separar **parser** y **AST** trae varias ventajas:

- el texto se analiza una sola vez;
- la estructura resultante puede reutilizarse;
- `Evaluar()` y `ToString()` recorren el mismo árbol;
- más adelante podemos agregar otras operaciones sin cambiar el parser.

Por ejemplo, sobre ese mismo AST podríamos implementar:

- un optimizador;
- un contador de nodos;
- un visitante;
- un generador de código;
- una impresión con menos paréntesis.

---

## Resumen final

En este tutorial seguimos esta secuencia:

1. definimos una gramática para expresiones aritméticas;
2. escribimos un parser descendente recursivo;
3. hicimos que el parser construya un AST;
4. evaluamos ese árbol;
5. vimos que también puede construirse manualmente;
6. agregamos operadores para escribirlo de forma más natural.

La idea central es esta:

**texto → parser → AST → operaciones sobre el árbol**

Ese modelo es pequeño, pero ya contiene varias ideas fundamentales de diseño de lenguajes.

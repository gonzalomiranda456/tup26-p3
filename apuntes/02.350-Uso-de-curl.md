# Pruebas de APIs con cURL — Guía Práctica

**cURL** (Client URL) es una herramienta de línea de comandos utilizada para transferir datos a través de diversos protocolos de red, principalmente HTTP y HTTPS. Es la herramienta por excelencia para desarrolladores backend al momento de probar, depurar e inspeccionar endpoints de APIs web desde la terminal sin necesidad de usar herramientas visuales como Postman.

---

## 1. Sintaxis Básica y Parámetros Más Comunes

La estructura básica de un comando cURL es:
```bash
curl [opciones] [URL]
```

### Opciones (Flags) Indispensables:

| Flag        | Nombre Completo | Propósito                                                                                  | Ejemplo                                 |
| :---------- | :-------------- | :----------------------------------------------------------------------------------------- | :-------------------------------------- |
| **`-X`**    | `--request`     | Especifica el método HTTP (GET, POST, PUT, DELETE).                                        | `-X POST`                               |
| **`-H`**    | `--header`      | Define cabeceras HTTP (Headers) para enviar metadatos o autenticación.                     | `-H "Content-Type: application/json"`   |
| **`-d`**    | `--data`        | Envía datos en el cuerpo de la petición (Payload).                                         | `-d '{"id": 1}'`                        |
| **`-i`**    | `--include`     | Incluye las cabeceras de la respuesta en la salida (útil para ver códigos de estado).      | `curl -i http://...`                    |
| **`-I`**    | `--head`        | Realiza una petición HEAD (solo descarga las cabeceras de respuesta, no el cuerpo).        | `curl -I http://...`                    |
| **`-v`**    | `--verbose`     | Modo detallado. Muestra todo el flujo (conexión TCP, TLS, cabeceras enviadas y recibidas). | `curl -v http://...`                    |
| **`-L`**    | `--location`    | Sigue redirecciones HTTP automáticamente (si la web redirige de HTTP a HTTPS).             | `curl -L http://...`                    |

## 2. Ejemplos de Peticiones HTTP Básicas

Utilizaremos la API pública de pruebas **JSONPlaceholder** para los siguientes ejemplos.

### A. Petición GET (Obtener Datos)
Por defecto, si no se especifica el flag `-X`, cURL realiza una petición `GET`:
```bash
# Petición GET simple
curl https://jsonplaceholder.typicode.com/posts/1

# Petición GET inspeccionando las cabeceras de respuesta (como Content-Type y Status Code)
curl -i https://jsonplaceholder.typicode.com/posts/1
```

### B. Petición POST (Crear Datos con JSON)
Para enviar datos JSON, debemos configurar el método en `POST`, indicar la cabecera `Content-Type` y pasar la cadena JSON en `-d`:
```bash
curl -X POST https://jsonplaceholder.typicode.com/posts \
  -H "Content-Type: application/json" \
  -d '{"title": "Nuevo Post", "body": "Contenido del post", "userId": 1}'
```

### C. Petición PUT (Actualizar Datos)
El método `PUT` se utiliza para reemplazar completamente un recurso existente:
```bash
curl -X PUT https://jsonplaceholder.typicode.com/posts/1 \
  -H "Content-Type: application/json" \
  -d '{"id": 1, "title": "Post Modificado", "body": "Cuerpo modificado", "userId": 1}'
```

### D. Petición DELETE (Eliminar Datos)
Para borrar un recurso específico:
```bash
curl -X DELETE https://jsonplaceholder.typicode.com/posts/1
```

## 3. Pruebas Avanzadas y Autenticación

### Envío de Tokens de Autenticación (Bearer Token)
Muy común al proteger APIs con JWT:
```bash
curl https://api.miservicio.com/usuarios \
  -H "Authorization: Bearer TU_TOKEN_AQUI"
```

### Descargar el Resultado a un Archivo
Si quieres guardar la respuesta de la API directamente en un archivo físico en lugar de imprimirlo en la terminal:
```bash
# Guarda la respuesta en 'salida.json'
curl -o salida.json https://jsonplaceholder.typicode.com/posts
```

## 4. Ejemplo Práctico: Probando nuestro Servidor MCP HTTP

Podemos probar el servidor MCP minimalista que construimos en la clase anterior directamente usando cURL en nuestra máquina local.

Asegúrate de que tu servidor esté corriendo localmente (por ejemplo, en el puerto `5002`) y ejecuta los siguientes comandos en tu terminal:

### Paso 1: Enviar solicitud de Inicialización (Initialize)
```bash
curl -i -X POST http://localhost:5002/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {
      "protocolVersion": "2025-06-18",
      "capabilities": {},
      "clientInfo": { "name": "curl-client", "version": "1.0" }
    }
  }'
```

### Paso 2: Consultar la lista de Herramientas (List Tools)
```bash
curl -X POST http://localhost:5002/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/list"
  }'
```

### Paso 3: Ejecutar la herramienta `saludar` (Call Tool)
```bash
curl -X POST http://localhost:5002/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 3,
    "method": "tools/call",
    "params": {
      "name": "saludar",
      "arguments": {
        "nombre": "Ada Lovelace"
      }
    }
  }'
```

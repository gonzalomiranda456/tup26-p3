# Null y tipos anulables en C#

`null` representa la **ausencia de valor**. Mal manejado produce el error más frecuente en .NET: `NullReferenceException`.

Desde C# 8 el compilador distingue referencias que pueden ser null (`string?`) de las que no (`string`). En proyectos .NET modernos el modo nullable está activado por defecto.

## Tipos por valor anulables (`T?`)

Los tipos por valor (`int`, `bool`, etc.) no aceptan `null` por defecto. Con `?` sí:

```csharp
int?  edad       = null;   // Nullable<int>
bool? confirmado = null;   // útil para "sí / no / sin responder"

edad.HasValue;              // false si null
edad.Value;                 // lanza excepción si null
edad.GetValueOrDefault();   // 0 si null
edad.GetValueOrDefault(-1); // -1 si null
```

Comparaciones con null siempre dan `false` (no lanza excepción):

```csharp
int? a = null;
Console.WriteLine(a > 5);  // false
Console.WriteLine(a < 5);  // false
```

## Nullable reference types (C# 8+)

```csharp
string  nombre   = "Ana";  // nunca null — el compilador lo verifica
string? apellido = null;   // puede ser null — declarado explícitamente
```

El compilador analiza el flujo y avisa si se accede a algo potencialmente null:

```csharp
string? texto = ObtenerTexto();
texto.Length;   // Warning: posible null

if (texto != null) {
    texto.Length; // OK — el compilador lo sabe
}
```

## Operadores

| Operador | Nombre                     | Uso                                  |
| -------- | -------------------------- | ------------------------------------ |
| `?.`     | Null-conditional           | Acceder a miembro solo si no es null |
| `??`     | Null-coalescing            | Valor de fallback si es null         |
| `??=`    | Null-coalescing assignment | Asignar solo si la variable es null  |
| `!`      | Null-forgiving             | Suprimir warning (usar con cuidado)  |

### `?.` — acceso condicional

```csharp
string? nombre = null;
int? largo = nombre?.Length;  // null, sin excepción

// Encadenable
string? ciudad = pedido?.Cliente?.Direccion?.Ciudad;

// Con métodos e indexadores
string? upper = s?.ToUpper();
char?   c     = s?[0];

// Con delegados
notificar?.Invoke("mensaje");
```

### `??` — valor de fallback

```csharp
string display = nombre ?? "Anónimo";    // "Anónimo" si null
int    segura  = edad   ?? 0;

// Encadenable
string resultado = a ?? b ?? "default"; // usa el primero no-null
```

### `??=` — asignar solo si null

```csharp
string? cache = null;
cache ??= "valor";   // asigna
cache ??= "otro";    // no hace nada — ya tiene valor

// Inicialización perezosa
List<string>? _items = null;
public List<string> Items { get { _items ??= []; return _items; } }
```

### `!` — suprimir warning

```csharp
int largo = texto!.Length;  // el compilador acepta, pero explota si texto es null
```

Usarlo solo cuando hay certeza real de que el valor no es null.

## Pattern matching con null

```csharp
if (nombre is null)     { ... }
if (nombre is not null) { ... }

string respuesta = input switch {
    null             => "Nada ingresado",
    ""               => "Cadena vacía",
    { Length: > 20 } => "Demasiado largo",
    _                => $"Ingresaste: {input}"
};
```

## Buenas prácticas

Preferir `is null` sobre `== null` (no puede ser sobrecargado):

```csharp
if (objeto is null)     { ... }  // siempre compara la referencia real
if (objeto is not null) { ... }
```

Validar parámetros al inicio del método:

```csharp
static void Procesar(string nombre) {
    ArgumentNullException.ThrowIfNull(nombre);  // C# 10+
    // acá el compilador sabe que nombre no es null
}
```

Filtrar nulls de colecciones:

```csharp
List<string?> nombres = ["Ana", null, "Carlos"];
List<string>  validos = nombres.OfType<string>().ToList();
```

## Resumen

```
¿Acceder a miembro que puede ser null?       → obj?.Miembro
¿Valor de fallback si es null?               → valor ?? "default"
¿Asignar solo si es null?                    → variable ??= nuevoValor
¿Invocar delegado de forma segura?           → delegado?.Invoke(args)
¿Verificar si algo es null?                  → valor is null / is not null
¿Suprimir warning (con certeza)?             → valor!
```

## 1. Qué problema resuelve un backend

Una aplicación de consola corre en una computadora. Sus datos viven ahí. Si otra aplicación necesita leer o modificar esos datos, hace falta una forma clara de comunicarse.

Un **backend web** es un programa que:

1. escucha pedidos HTTP;
2. interpreta qué quiere el cliente;
3. ejecuta lógica de aplicación;
4. consulta o modifica datos;
5. devuelve una respuesta HTTP.

Esquema general:

```txt
Cliente HTTP
   ↓ request
Backend ASP.NET Core
   ↓ lógica
Base de datos SQLite
   ↑ datos
Backend ASP.NET Core
   ↑ response
Cliente HTTP
```

El cliente puede ser:

- un navegador;
- una app mobile;
- otro backend;
- un programa de consola;
- un test automatizado;
- una página HTML con JavaScript.

El contrato entre cliente y servidor es **HTTP**.

## 2. HTTP como protocolo de mensajes

HTTP es un protocolo basado en mensajes. En cada interacción hay dos mensajes:

```txt
Request  → pedido que envía el cliente
Response → respuesta que devuelve el servidor
```

### Request

Una request tiene esta estructura conceptual:

```txt
Método + ruta + versión
Headers
Body opcional
```

Ejemplo:

```http
POST /productos HTTP/1.1
Host: localhost:5000
Content-Type: application/json
Accept: application/json

{
  "nombre": "Monitor",
  "precio": 250000
}
```

Se lee así:

```txt
Quiero crear algo dentro de /productos.
Te envío datos en JSON.
Espero una respuesta JSON.
```

### Response

Una response tiene esta estructura conceptual:

```txt
Versión + status code
Headers
Body opcional
```

Ejemplo:

```http
HTTP/1.1 201 Created
Content-Type: application/json
Location: /productos/7

{
  "id": 7,
  "nombre": "Monitor",
  "precio": 250000
}
```

Se lee así:

```txt
El recurso fue creado correctamente.
La representación está en JSON.
El nuevo recurso está en /productos/7.
```

### Métodos HTTP

El método indica la intención del pedido.

| Método     | Intención principal                                   |
| ---------- | ----------------------------------------------------- |
| `GET`      | Leer datos.                                           |
| `POST`     | Crear un recurso o ejecutar una operación de creación.|
| `PUT`      | Reemplazar un recurso completo.                       |
| `PATCH`    | Modificar parcialmente un recurso.                    |
| `DELETE`   | Eliminar un recurso.                                  |

### URL, ruta y query string

Ejemplo:

```txt
http://localhost:5000/productos/5?incluirInactivos=true
│      │              │          │
│      │              │          └── query string
│      │              └──────────── ruta
│      └─────────────────────────── host y puerto
└────────────────────────────────── esquema
```

La **ruta** identifica el recurso dentro del servidor:

```txt
/productos
/productos/5
/clientes/10/facturas
```

La **query string** manda parámetros opcionales:

```txt
/productos?texto=mouse
/productos?pagina=2&tamanio=20
/productos?orden=precio
```

Regla práctica:

```txt
Ruta         → identifica el recurso.
Query string → filtra, ordena o pagina una colección.
```

### Headers

Los headers son metadatos del mensaje.

| Header           | Uso                                      |
| ---------------- | ---------------------------------------- |
| `Content-Type`   | Formato del body que se envía.           |
| `Accept`         | Formato que el cliente prefiere recibir. |
| `Authorization`  | Credenciales o tokens.                   |
| `Content-Length` | Tamaño del body.                         |

### Body

El body contiene datos. En APIs REST modernas, casi siempre usamos JSON.

```json
{
  "nombre": "Teclado",
  "precio": 15000
}
```

`GET` normalmente no lleva body.  
`POST`, `PUT` y `PATCH` normalmente sí.

### Status codes principales

El status code comunica el resultado de la operación.

| Código | Nombre                 | Uso típico                                                                  |
| -----: | ---------------------- | --------------------------------------------------------------------------- |
| 200    | OK                     | La operación salió bien y devuelve contenido.                               |
| 201    | Created                | Se creó un recurso.                                                         |
| 204    | No Content             | La operación salió bien pero no devuelve body.                              |
| 400    | Bad Request            | El cliente mandó datos inválidos.                                           |
| 401    | Unauthorized           | Falta autenticación.                                                        |
| 403    | Forbidden              | Está autenticado, pero no tiene permiso.                                    |
| 404    | Not Found              | El recurso no existe.                                                       |
| 409    | Conflict               | Hay conflicto con el estado actual.                                         |
| 422    | Unprocessable Entity   | El JSON está bien formado, los datos no son válidos para regla de negocio.  |
| 500    | Internal Server Error  | Falló el servidor.                                                          |

Una API no solo devuelve datos. Devuelve datos más significado.

```txt
404 no significa "no hay nada".
404 significa "el recurso solicitado no existe".
```

## 3. JSON

JSON es texto plano para representar datos.

Ejemplo:

```json
{
  "id": 5,
  "nombre": "Mouse",
  "precio": 10000,
  "activo": true
}
```

Tipos comunes:

| Tipo       | Ejemplo                 |
| ---------- | ----------------------- |
| Número     | `42`, `3.14`            |
| Texto      | `"Hola"`                |
| Booleano   | `true`, `false`         |
| Nulo       | `null`                  |
| Objeto     | `{ "clave": "valor" }`  |
| Array      | `[1, 2, 3]`             |

ASP.NET Core convierte objetos C# a JSON automáticamente cuando una API devuelve objetos.

## 4. REST

REST es una forma de diseñar APIs alrededor de **recursos**.

Un recurso es algo con identidad dentro del sistema:

```txt
Producto
Cliente
Factura
Turno
Alumno
Inscripción
```

En REST no diseñamos URLs como acciones:

```txt
/crearProducto
/borrarProducto
/listarProductos
```

Diseñamos URLs como recursos:

```txt
/productos
/productos/5
```

Y usamos el método HTTP para expresar la operación:

| Request                | Significado                         |
| ---------------------- | ----------------------------------- |
| `GET /productos`       | Leer la colección de productos.     |
| `GET /productos/5`     | Leer el producto con id 5.          |
| `POST /productos`      | Crear un producto nuevo.            |
| `PUT /productos/5`     | Reemplazar el producto 5.           |
| `PATCH /productos/5`   | Modificar parte del producto 5.     |
| `DELETE /productos/5`  | Eliminar el producto 5.             |

### Colección y recurso individual

```txt
/productos
```

representa la colección completa.

```txt
/productos/5
```

representa un producto concreto.

### Representación

El recurso real no viaja por la red. Lo que viaja es una representación.

El producto vive en la base de datos. La API devuelve una representación JSON:

```json
{
  "id": 5,
  "nombre": "Mouse",
  "precio": 10000
}
```

Conviene no confundir:

```txt
Tabla de base de datos ≠ entidad C# ≠ DTO ≠ JSON público
```

A veces se parecen. No son lo mismo.

### Métodos seguros e idempotentes

`GET` es seguro: no debería modificar el estado del servidor.

`PUT` y `DELETE` son idempotentes: repetir la misma operación debería dejar el sistema en el mismo estado final.

Ejemplo:

```txt
DELETE /productos/5
DELETE /productos/5
DELETE /productos/5
```

Después de la primera llamada, el producto ya no existe. Repetir la request no debería eliminar productos distintos.

`POST` no es necesariamente idempotente:

```txt
POST /productos
POST /productos
POST /productos
```

puede crear tres productos distintos.

### PUT vs PATCH

`PUT` reemplaza el recurso completo.

```http
PUT /productos/5
Content-Type: application/json

{
  "nombre": "Mouse inalámbrico",
  "precio": 18000
}
```

`PATCH` modifica parcialmente.

```http
PATCH /productos/5
Content-Type: application/json

{
  "precio": 18000
}
```

Para un primer CRUD alcanza con `PUT`. `PATCH` se puede dejar como mejora.


Ejecutar:

```bash
dotnet run app.cs
```

La línea importante:

```csharp
app.MapGet("/saludo", () => "Hola desde el servidor");
```

Se lee así:

```txt
Cuando llegue un GET a /saludo,
ejecutá esta función
y devolvé el resultado como response HTTP.
```

Un **endpoint** es una función disponible por HTTP.

## 6. Rutas y parámetros

```csharp

var builder = WebApplication.CreateBuilder(args);

app.MapGet("/productos/{id:int}", (int id) => {
    return $"Producto solicitado: {id}";
```

```http
GET /productos/10
```


La ruta:

```csharp
"/productos/{id:int}"
```

indica que `id` viene desde la URL y debe poder convertirse a `int`.

Si llega:

```txt
/productos/abc
```

no coincide con esa ruta porque `abc` no es entero.

## 7. CRUD en memoria

Antes de usar base de datos, armamos una API en memoria. Esto permite entender HTTP, REST, DTOs y status codes sin mezclar todavía SQL.

```csharp
#:sdk Microsoft.NET.Sdk.Web

var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

List<Producto> productos =
[
    new Producto(1, "Teclado", 15000),
    new Producto(2, "Mouse", 10000)
];

app.MapGet("/productos", () => {
    return Results.Ok(productos);
});

app.MapGet("/productos/{id:int}", (int id) => {
    Producto? producto = productos.FirstOrDefault(p => p.Id == id);

    if (producto is null) {
        return Results.NotFound();
    }

    return Results.Ok(producto);
});

app.MapPost("/productos", (CrearProductoDto dto) => {
    if (string.IsNullOrWhiteSpace(dto.Nombre)) {
        return Results.BadRequest("El nombre es obligatorio.");
    }

    if (dto.Precio <= 0) {
        return Results.BadRequest("El precio debe ser mayor que cero.");
    }

    int nuevoId = productos.Count == 0
        ? 1
        : productos.Max(p => p.Id) + 1;

    Producto producto = new Producto(nuevoId, dto.Nombre, dto.Precio);

    productos.Add(producto);

    return Results.Created($"/productos/{producto.Id}", producto);
});

app.MapPut("/productos/{id:int}", (int id, ActualizarProductoDto dto) => {
    if (string.IsNullOrWhiteSpace(dto.Nombre)) {
        return Results.BadRequest("El nombre es obligatorio.");
    }

    if (dto.Precio <= 0) {
        return Results.BadRequest("El precio debe ser mayor que cero.");
    }

    int indice = productos.FindIndex(p => p.Id == id);

    if (indice == -1) {
        return Results.NotFound();
    }

    Producto productoActualizado = new Producto(id, dto.Nombre, dto.Precio);
    productos[indice] = productoActualizado;

    return Results.Ok(productoActualizado);
});

app.MapDelete("/productos/{id:int}", (int id) => {
    Producto? producto = productos.FirstOrDefault(p => p.Id == id);

    if (producto is null) {
        return Results.NotFound();
    }

    productos.Remove(producto);

    return Results.NoContent();
});

app.Run();

public record Producto(int Id, string Nombre, decimal Precio);

public record CrearProductoDto(string Nombre, decimal Precio);

public record ActualizarProductoDto(string Nombre, decimal Precio);
```

Este ejemplo ya tiene la estructura de una API real:

```txt
GET colección
GET por id
POST creación
PUT actualización
DELETE eliminación
DTOs de entrada
status codes correctos
JSON automático
```

## 8. DTOs

Un DTO representa un mensaje que entra o sale por la API.

Entidad interna:

```csharp
public class Producto {
    public int Id { get; set; }
    public string Nombre { get; set; } = "";
    public decimal Precio { get; set; }
}
```

DTO para crear:

```csharp
public record CrearProductoDto(string Nombre, decimal Precio);
```

DTO para actualizar:

```csharp
public record ActualizarProductoDto(string Nombre, decimal Precio);
```

El cliente no debería decidir el `Id` al crear. Por eso el DTO de creación no tiene `Id`.

El backend recibe:

```json
{
  "nombre": "Monitor",
  "precio": 250000
}
```

y construye internamente:

```csharp
Producto producto = new Producto {
    Nombre = dto.Nombre,
    Precio = dto.Precio
};
```

## 9. Validación simple

Validar es rechazar datos que no cumplen el contrato.

Ejemplo:

```csharp
if (string.IsNullOrWhiteSpace(dto.Nombre)) {
    return Results.BadRequest("El nombre es obligatorio.");
}

if (dto.Precio <= 0) {
    return Results.BadRequest("El precio debe ser mayor que cero.");
}
```

Regla práctica:

```txt
Dato inválido del cliente → 400 Bad Request
```

No corresponde devolver `500`. Un `500` significa que falló el servidor.

También se puede devolver una lista de errores:

```csharp
static List<string> ValidarProducto(string nombre, decimal precio) {
    List<string> errores = [];

    if (string.IsNullOrWhiteSpace(nombre)) {
        errores.Add("El nombre es obligatorio.");
    }

    if (precio <= 0) {
        errores.Add("El precio debe ser mayor que cero.");
    }

    return errores;
}
```

Uso:

```csharp
List<string> errores = ValidarProducto(dto.Nombre, dto.Precio);

if (errores.Count > 0) {
    return Results.BadRequest(errores);
}
```

## 10. SQLite, ADO.NET, Dapper y Dapper.Contrib

Ahora reemplazamos la lista en memoria por una base SQLite.

Roles de cada herramienta:

| Herramienta      | Para qué sirve                                                   |
| ---------------- | ---------------------------------------------------------------- |
| SQLite           | Base de datos liviana en un archivo local.                       |
| ADO.NET          | Infraestructura base de conexión y comandos SQL.                 |
| Dapper           | Ejecutar SQL y mapear resultados a objetos.                      |
| Dapper.Contrib   | CRUD simple con métodos como `Get`, `Insert`, `Update`, `Delete`.|

### Paquetes

En archivos C# file-based se pueden declarar paquetes arriba del archivo:

```csharp
#:sdk Microsoft.NET.Sdk.Web
#:package Microsoft.Data.Sqlite
#:package Dapper
#:package Dapper.Contrib
```

### Conexión

```csharp
const string connectionString = "Data Source=productos.db";
```

Esto crea o usa un archivo `productos.db` en la carpeta donde se ejecuta la API.

### Crear tabla

```csharp
static void InicializarBaseDeDatos(string connectionString)
{
    using SqliteConnection connection = new SqliteConnection(connectionString);

    string sql = """
        CREATE TABLE IF NOT EXISTS Productos (
            Id INTEGER PRIMARY KEY AUTOINCREMENT,
            Nombre TEXT NOT NULL,
            Precio REAL NOT NULL
        );
        """;

    connection.Execute(sql);
}
```

Aunque `Execute` viene de Dapper, la idea base es ADO.NET:

```txt
crear conexión
abrir conexión cuando haga falta
ejecutar SQL
cerrar conexión
```

La versión sin Dapper sería:

```csharp
static void InicializarBaseDeDatosSoloAdoNet(string connectionString)
{
    using SqliteConnection connection = new SqliteConnection(connectionString);
    connection.Open();

    using SqliteCommand command = connection.CreateCommand();

    command.CommandText = """
        CREATE TABLE IF NOT EXISTS Productos (
            Id INTEGER PRIMARY KEY AUTOINCREMENT,
            Nombre TEXT NOT NULL,
            Precio REAL NOT NULL
        );
        """;

    command.ExecuteNonQuery();
}
```

### Entidad para Dapper.Contrib

```csharp
[Table("Productos")]
public class Producto
{
    [Key]
    public int Id { get; set; }

    public string Nombre { get; set; } = "";

    public decimal Precio { get; set; }
}
```

`[Table("Productos")]` indica la tabla.  
`[Key]` indica la clave primaria.

### Operaciones comunes

```csharp
IEnumerable<Producto> productos = connection.GetAll<Producto>();
```

```csharp
Producto? producto = connection.Get<Producto>(id);
```

```csharp
long id = connection.Insert(producto);
```

```csharp
connection.Update(producto);
```

```csharp
connection.Delete(producto);
```

### Consultas personalizadas con Dapper

Para búsquedas, filtros y consultas más específicas conviene escribir SQL.

```csharp
string sql = """
    SELECT Id, Nombre, Precio
    FROM Productos
    WHERE Nombre LIKE @Patron
    ORDER BY Nombre;
    """;

IEnumerable<Producto> productos = connection.Query<Producto>(
    sql,
    new { Patron = $"%{texto}%" }
);
```

La parte importante:

```csharp
WHERE Nombre LIKE @Patron
```

El valor se pasa como parámetro. No se concatena dentro del SQL.

Evitar:

```csharp
string sql = $"SELECT * FROM Productos WHERE Nombre LIKE '%{texto}%'";
```

Concatenar valores dentro del SQL produce código frágil y abre riesgos de inyección SQL.

## 11. API completa con SQLite

Crear `app.cs`:

```csharp
#:sdk Microsoft.NET.Sdk.Web
#:package Microsoft.Data.Sqlite
#:package Dapper
#:package Dapper.Contrib

using Dapper;
using Dapper.Contrib.Extensions;
using Microsoft.Data.Sqlite;

var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

const string connectionString = "Data Source=productos.db";

InicializarBaseDeDatos(connectionString);

app.MapGet("/", () => "API de productos");

app.MapGet("/productos", () =>
{
    using SqliteConnection connection = new SqliteConnection(connectionString);

    IEnumerable<Producto> productos = connection.GetAll<Producto>();

    return Results.Ok(productos);
});

app.MapGet("/productos/{id:int}", (int id) =>
{
    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto? producto = connection.Get<Producto>(id);

    if (producto is null) {
        return Results.NotFound();
    }

    return Results.Ok(producto);
});

app.MapGet("/productos/buscar", (string texto) =>
{
    using SqliteConnection connection = new SqliteConnection(connectionString);

    string sql = """
        SELECT Id, Nombre, Precio
        FROM Productos
        WHERE Nombre LIKE @Patron
        ORDER BY Nombre;
        """;

    IEnumerable<Producto> productos = connection.Query<Producto>(
        sql,
        new { Patron = $"%{texto}%" }
    );

    return Results.Ok(productos);
});

app.MapPost("/productos", (CrearProductoDto dto) =>
{
    List<string> errores = ValidarProducto(dto.Nombre, dto.Precio);

    if (errores.Count > 0) {
        return Results.BadRequest(errores);
    }

    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto producto = new Producto {
        Nombre = dto.Nombre,
        Precio = dto.Precio
    };

    long id = connection.Insert(producto);
    producto.Id = (int)id;

    return Results.Created($"/productos/{producto.Id}", producto);
});

app.MapPut("/productos/{id:int}", (int id, ActualizarProductoDto dto) =>
{
    List<string> errores = ValidarProducto(dto.Nombre, dto.Precio);

    if (errores.Count > 0) {
        return Results.BadRequest(errores);
    }

    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto? producto = connection.Get<Producto>(id);

    if (producto is null) {
        return Results.NotFound();
    }

    producto.Nombre = dto.Nombre;
    producto.Precio = dto.Precio;

    connection.Update(producto);

    return Results.Ok(producto);
});

app.MapDelete("/productos/{id:int}", (int id) =>
{
    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto? producto = connection.Get<Producto>(id);

    if (producto is null) {
        return Results.NotFound();
    }

    connection.Delete(producto);

    return Results.NoContent();
});

app.Run();

static void InicializarBaseDeDatos(string connectionString) {
    using SqliteConnection connection = new SqliteConnection(connectionString);

    string sql = """
        CREATE TABLE IF NOT EXISTS Productos (
            Id INTEGER PRIMARY KEY AUTOINCREMENT,
            Nombre TEXT NOT NULL,
            Precio REAL NOT NULL
        );
        """;

    connection.Execute(sql);
}

static List<string> ValidarProducto(string nombre, decimal precio) {
    List<string> errores = [];

    if (string.IsNullOrWhiteSpace(nombre)) {
        errores.Add("El nombre es obligatorio.");
    }

    if (precio <= 0) {
        errores.Add("El precio debe ser mayor que cero.");
    }

    return errores;
}

[Table("Productos")]
public class Producto {
    [Key]
    public int Id { get; set; }

    public string Nombre { get; set; } = "";

    public decimal Precio { get; set; }
}

public record CrearProductoDto(string Nombre, decimal Precio);

public record ActualizarProductoDto(string Nombre, decimal Precio);
```

Ejecutar:

```bash
dotnet run app.cs
```

Endpoints disponibles:

| Método     | Ruta                         | Acción                           |
| ---------- | ---------------------------- | -------------------------------- |
| `GET`      | `/`                          | Verificar que la API funciona.   |
| `GET`      | `/productos`                 | Listar productos.                |
| `GET`      | `/productos/{id}`            | Obtener un producto por id.      |
| `GET`      | `/productos/buscar?texto=...`| Buscar productos por nombre.     |
| `POST`     | `/productos`                 | Crear producto.                  |
| `PUT`      | `/productos/{id}`            | Actualizar producto completo.    |
| `DELETE`   | `/productos/{id}`            | Eliminar producto.               |

## 14. Consumir la API con HttpClient

Crear otro archivo: `cliente.cs`.

```csharp
using System.Net.Http.Json;

HttpClient http = new HttpClient
{
    BaseAddress = new Uri("http://localhost:5000")
};

Console.WriteLine("Listado inicial:");

List<Producto>? productos = await http.GetFromJsonAsync<List<Producto>>("/productos");

MostrarProductos(productos);

CrearProductoDto nuevoProducto = new CrearProductoDto("Auriculares", 35000);

HttpResponseMessage respuestaCreacion = await http.PostAsJsonAsync("/productos", nuevoProducto);

Console.WriteLine($"POST /productos -> {(int)respuestaCreacion.StatusCode} {respuestaCreacion.StatusCode}");

Producto? productoCreado = await respuestaCreacion.Content.ReadFromJsonAsync<Producto>();

Console.WriteLine($"Creado: {productoCreado?.Id} - {productoCreado?.Nombre}");

Console.WriteLine("Listado final:");

productos = await http.GetFromJsonAsync<List<Producto>>("/productos");

MostrarProductos(productos);

static void MostrarProductos(List<Producto>? productos)
{
    if (productos is null)
    {
        Console.WriteLine("No se pudo leer la respuesta.");
        return;
    }

    foreach (Producto producto in productos)
    {
        Console.WriteLine($"{producto.Id}: {producto.Nombre} - ${producto.Precio}");
    }
}

public record Producto(int Id, string Nombre, decimal Precio);

public record CrearProductoDto(string Nombre, decimal Precio);
```

Ejecutar la API en una terminal:

```bash
dotnet run app.cs
```

Ejecutar el cliente en otra terminal:

```bash
dotnet run cliente.cs
```

Líneas importantes:

```csharp
List<Producto>? productos = await http.GetFromJsonAsync<List<Producto>>("/productos");
```

hace un `GET`, lee JSON y lo convierte a objetos C#.

```csharp
HttpResponseMessage respuestaCreacion = await http.PostAsJsonAsync("/productos", nuevoProducto);
```

hace un `POST`, serializa el objeto como JSON y lo manda en el body.

## 15. Manejo de errores desde el cliente

Un cliente no debería asumir que todo salió bien.

Ejemplo:

```csharp
HttpResponseMessage response = await http.GetAsync("/productos/9999");

if (response.IsSuccessStatusCode)
{
    Producto? producto = await response.Content.ReadFromJsonAsync<Producto>();
    Console.WriteLine(producto?.Nombre);
}
else if (response.StatusCode == System.Net.HttpStatusCode.NotFound)
{
    Console.WriteLine("El producto no existe.");
}
else
{
    string error = await response.Content.ReadAsStringAsync();
    Console.WriteLine($"Error: {(int)response.StatusCode} {error}");
}
```

Regla práctica:

```txt
La API comunica con status codes.
El cliente decide qué hacer con cada status code.
```

## 16. Consumir la API desde HTML con fetch

ASP.NET Core puede servir archivos estáticos desde una carpeta `wwwroot`.

Crear:

```txt
app.cs
wwwroot/
    index.html
```

Archivo `wwwroot/index.html`:

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Productos</title>
</head>
<body>
    <h1>Productos</h1>

    <form id="formulario">
        <input id="nombre" placeholder="Nombre">
        <input id="precio" placeholder="Precio" type="number">
        <button>Guardar</button>
    </form>

    <ul id="lista"></ul>

    <script>
        const lista = document.getElementById("lista");
        const formulario = document.getElementById("formulario");
        const nombre = document.getElementById("nombre");
        const precio = document.getElementById("precio");

        async function cargarProductos() {
            const respuesta = await fetch("/productos");
            const productos = await respuesta.json();

            lista.innerHTML = "";

            for (const producto of productos) {
                const item = document.createElement("li");
                item.textContent = `${producto.nombre} - $${producto.precio}`;
                lista.appendChild(item);
            }
        }

        formulario.addEventListener("submit", async evento => {
            evento.preventDefault();

            await fetch("/productos", {
                method: "POST",
                headers: {
                    "Content-Type": "application/json"
                },
                body: JSON.stringify({
                    nombre: nombre.value,
                    precio: Number(precio.value)
                })
            });

            nombre.value = "";
            precio.value = "";

            await cargarProductos();
        });

        cargarProductos();
    </script>
</body>
</html>
```

Para servir ese archivo desde ASP.NET Core, agregar antes de mapear endpoints:

```csharp
app.UseDefaultFiles();
app.UseStaticFiles();
```

Ejemplo:

```csharp
var app = builder.Build();

app.UseDefaultFiles();
app.UseStaticFiles();

app.MapGet("/productos", () =>
{
    // ...
});
```

Ahora el navegador puede abrir:

```txt
http://localhost:5000
```

Como el HTML y la API salen del mismo backend, no hace falta CORS.

## 17. CORS

CORS aparece cuando el frontend y el backend están en orígenes distintos.

Ejemplo:

```txt
Frontend: http://localhost:5500
Backend:  http://localhost:5000
```

Para el navegador son orígenes distintos.

Configuración:

```csharp
builder.Services.AddCors(options => {
    options.AddPolicy("Frontend", policy => {
        policy
            .WithOrigins("http://localhost:5500")
            .AllowAnyHeader()
            .AllowAnyMethod();
    });
});
```

Activación:

```csharp
app.UseCors("Frontend");
```

Ubicación típica:

```csharp
var app = builder.Build();

app.UseCors("Frontend");

app.MapGet("/productos", () =>
{
    // ...
});
```

Para clase conviene servir `index.html` desde `wwwroot` usando el mismo backend. Así CORS no aparece hasta que realmente haga falta.

## 18. Middleware

Una request no entra directamente al endpoint. Pasa por una tubería.

```txt
Request
   ↓
Middleware
   ↓
Middleware
   ↓
Endpoint
   ↓
Response
```

Cada middleware puede:

- leer la request;
- modificar la request;
- decidir si la ejecución sigue;
- ejecutar código antes del endpoint;
- ejecutar código después del endpoint;
- modificar la response.

Ejemplo simple para ver el flujo:

```csharp
app.Use(async (context, next) => {
    Console.WriteLine($"Entró {context.Request.Method} {context.Request.Path}");

    await next();

    Console.WriteLine($"Salió {context.Response.StatusCode}");
});
```

Si llega:

```http
GET /productos
```

la consola muestra algo parecido a:

```txt
Entró GET /productos
Salió 200
```

El orden del pipeline importa. En general:

```csharp
var app = builder.Build();

app.UseCors("Frontend");
app.UseDefaultFiles();
app.UseStaticFiles();

app.Use(async (context, next) =>
{
    Console.WriteLine($"{context.Request.Method} {context.Request.Path}");
    await next();
});

app.MapGet("/productos", () =>
{
    // ...
});
```

## 19. Logging con ILogger

ASP.NET Core trae infraestructura de logging.

Se puede pedir un logger en un endpoint:

```csharp
app.MapPost("/productos", (CrearProductoDto dto, ILogger<Program> logger) => {
    logger.LogInformation("Intentando crear producto {Nombre} con precio {Precio}", dto.Nombre, dto.Precio);

    if (string.IsNullOrWhiteSpace(dto.Nombre)) {
        logger.LogWarning("Intento de crear producto con nombre vacío");
        return Results.BadRequest("El nombre es obligatorio.");
    }

    if (dto.Precio <= 0) {
        logger.LogWarning("Intento de crear producto con precio inválido: {Precio}", dto.Precio);
        return Results.BadRequest("El precio debe ser mayor que cero.");
    }

    // insertar producto...

    logger.LogInformation("Producto creado correctamente");

    return Results.Ok();
});
```

Niveles comunes:

| Nivel           | Uso                         |
| --------------- | --------------------------- |
| `Trace`         | Detalle extremo.            |
| `Debug`         | Información de depuración.  |
| `Information`   | Operación normal.           |
| `Warning`       | Algo raro pero controlado.  |
| `Error`         | Falló una operación.        |
| `Critical`      | Fallo grave.                |

Esta forma:

```csharp
logger.LogInformation("Producto creado con id {Id}", producto.Id);
```

es preferible a:

```csharp
logger.LogInformation($"Producto creado con id {producto.Id}");
```

porque conserva el mensaje estructurado.

## 20. Response compression

La compresión reduce el tamaño de respuestas HTTP.

Sirve para:

- JSON grandes;
- HTML;
- CSS;
- JavaScript.

Con `#:sdk Microsoft.NET.Sdk.Web`, el middleware ya está disponible. No hace falta agregar un paquete extra para este caso.

Configurar servicios:

```csharp
builder.Services.AddResponseCompression(options => {
    options.EnableForHttps = true;
});
```

Activar middleware:

```csharp
app.UseResponseCompression();
```

Ubicación típica:

```csharp
var app = builder.Build();

app.UseResponseCompression();
app.UseDefaultFiles();
app.UseStaticFiles();

app.MapGet("/productos", () =>
{
    // ...
});
```

Cuando el cliente manda:

```http
Accept-Encoding: gzip, br
```

el servidor puede responder comprimido.

## 22. Versión final con middleware útil

Esta versión integra:

- API REST;
- SQLite;
- Dapper;
- Dapper.Contrib;
- validación;
- logging;
- archivos estáticos;
- compresión;
- middleware simple.

```csharp
#:sdk Microsoft.NET.Sdk.Web
#:package Microsoft.Data.Sqlite
#:package Dapper
#:package Dapper.Contrib

using Dapper;
using Dapper.Contrib.Extensions;
using Microsoft.Data.Sqlite;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddResponseCompression(options => {
    options.EnableForHttps = true;
});

var app = builder.Build();

app.UseResponseCompression();
app.UseDefaultFiles();
app.UseStaticFiles();

app.Use(async (context, next) => {
    Console.WriteLine($"Entró {context.Request.Method} {context.Request.Path}");

    await next();

    Console.WriteLine($"Salió {context.Response.StatusCode}");
});

const string connectionString = "Data Source=productos.db";

InicializarBaseDeDatos(connectionString);

app.MapGet("/", () => "API de productos");

app.MapGet("/productos", (ILogger<Program> logger) => {
    using SqliteConnection connection = new SqliteConnection(connectionString);

    IEnumerable<Producto> productos = connection.GetAll<Producto>();

    logger.LogInformation("Se consultó el listado de productos");

    return Results.Ok(productos);
});

app.MapGet("/productos/{id:int}", (int id, ILogger<Program> logger) => {
    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto? producto = connection.Get<Producto>(id);

    if (producto is null) {
        logger.LogWarning("Producto {Id} no encontrado", id);
        return Results.NotFound();
    }

    return Results.Ok(producto);
});

app.MapGet("/productos/buscar", (string texto) => {
    using SqliteConnection connection = new SqliteConnection(connectionString);

    string sql = """
        SELECT Id, Nombre, Precio
        FROM Productos
        WHERE Nombre LIKE @Patron
        ORDER BY Nombre;
        """;

    IEnumerable<Producto> productos = connection.Query<Producto>(
        sql,
        new { Patron = $"%{texto}%" }
    );

    return Results.Ok(productos);
});

app.MapPost("/productos", (CrearProductoDto dto, ILogger<Program> logger) => {
    List<string> errores = ValidarProducto(dto.Nombre, dto.Precio);

    if (errores.Count > 0) {
        logger.LogWarning("Producto inválido: {Errores}", string.Join(" | ", errores));
        return Results.BadRequest(errores);
    }

    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto producto = new Producto {
        Nombre = dto.Nombre,
        Precio = dto.Precio
    };

    long id = connection.Insert(producto);
    producto.Id = (int)id;

    logger.LogInformation("Producto creado: {Id} {Nombre}", producto.Id, producto.Nombre);

    return Results.Created($"/productos/{producto.Id}", producto);
});

app.MapPut("/productos/{id:int}", (int id, ActualizarProductoDto dto, ILogger<Program> logger) => {
    List<string> errores = ValidarProducto(dto.Nombre, dto.Precio);

    if (errores.Count > 0) {
        return Results.BadRequest(errores);
    }

    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto? producto = connection.Get<Producto>(id);

    if (producto is null) {
        logger.LogWarning("No se pudo actualizar. Producto {Id} no encontrado", id);
        return Results.NotFound();
    }

    producto.Nombre = dto.Nombre;
    producto.Precio = dto.Precio;

    connection.Update(producto);

    logger.LogInformation("Producto actualizado: {Id}", producto.Id);

    return Results.Ok(producto);
});

app.MapDelete("/productos/{id:int}", (int id, ILogger<Program> logger) => {
    using SqliteConnection connection = new SqliteConnection(connectionString);

    Producto? producto = connection.Get<Producto>(id);

    if (producto is null) {
        logger.LogWarning("No se pudo eliminar. Producto {Id} no encontrado", id);
        return Results.NotFound();
    }

    connection.Delete(producto);

    logger.LogInformation("Producto eliminado: {Id}", id);

    return Results.NoContent();
});

app.Run();

static void InicializarBaseDeDatos(string connectionString) {
    using SqliteConnection connection = new SqliteConnection(connectionString);

    string sql = """
        CREATE TABLE IF NOT EXISTS Productos (
            Id INTEGER PRIMARY KEY AUTOINCREMENT,
            Nombre TEXT NOT NULL,
            Precio REAL NOT NULL
        );
        """;

    connection.Execute(sql);
}

static List<string> ValidarProducto(string nombre, decimal precio) {
    List<string> errores = [];

    if (string.IsNullOrWhiteSpace(nombre)) {
        errores.Add("El nombre es obligatorio.");
    }

    if (precio <= 0) {
        errores.Add("El precio debe ser mayor que cero.");
    }

    return errores;
}

[Table("Productos")]
public class Producto {
    [Key]
    public int Id { get; set; }

    public string Nombre { get; set; } = "";

    public decimal Precio { get; set; }
}

public record CrearProductoDto(string Nombre, decimal Precio);

public record ActualizarProductoDto(string Nombre, decimal Precio);
```

Al terminar el recorrido, deberías poder leer este endpoint:

```csharp
app.MapGet("/productos/{id:int}", (int id) => {
    using SqliteConnection connection = new SqliteConnection(connectionString);
```

Ese es el núcleo de la programación web backend con C#:

```txt
HTTP → endpoint → lógica → datos → response
```

# Dapper en C# — Referencia completa

Dapper extiende `DbConnection` para mapear filas SQL a objetos C# automáticamente. El SQL lo escribís vos; Dapper hace el mapping.

```bash
dotnet add package Dapper
dotnet add package Dapper.Contrib   # CRUD sin SQL manual
dotnet add package Microsoft.Data.Sqlite
```

## Cómo funciona el mapping

Dapper compara nombres de columnas con nombres de propiedades (insensible a mayúsculas):

```sql
-- Tabla: columnas Id, Nombre, Apellido, Email
```
```csharp
class Contacto {
    public int    Id       { get; set; }
    public string Nombre   { get; set; } = "";
    public string Apellido { get; set; } = "";
    public string Email    { get; set; } = "";
}
```

Los nombres coinciden → Dapper conecta automáticamente.

## Métodos principales

```csharp
using DbConnection db = new SqliteConnection("Data Source=app.db");
db.Open();

// Muchos resultados
IEnumerable<Contacto> todos = db.Query<Contacto>("SELECT * FROM Contactos");

// Un resultado o null
Contacto? c = db.QuerySingleOrDefault<Contacto>(
    "SELECT * FROM Contactos WHERE Id = @id", new { id = 5 });

// Un valor escalar
int total = db.QuerySingle<int>("SELECT COUNT(*) FROM Contactos");

// INSERT / UPDATE / DELETE
db.Execute("DELETE FROM Contactos WHERE Id = @id", new { id = 3 });
```

## Parámetros — siempre @, nunca concatenar

```csharp
// ✓ Seguro
db.Query<Contacto>(
    "SELECT * FROM Contactos WHERE Apellido = @apellido",
    new { apellido = "Díaz" });

// ✗ SQL injection
db.Query<Contacto>($"SELECT * FROM Contactos WHERE Apellido = '{apellido}'");
```

Se pueden pasar como objeto anónimo, diccionario, o instancia de la clase:
```csharp
db.Execute("INSERT INTO Contactos (Nombre, Email) VALUES (@Nombre, @Email)", contacto);
```

## Dapper.Contrib — CRUD sin SQL

```csharp
using Dapper.Contrib.Extensions;

// INSERT → devuelve Id generado
long id = db.Insert(new Contacto { Nombre = "Elena", Email = "elena@mail.com" });
db.Insert(new List<Contacto> { ... });   // múltiples a la vez

// SELECT
Contacto? c = db.Get<Contacto>(id);
IEnumerable<Contacto> todos = db.GetAll<Contacto>();

// UPDATE / DELETE
c!.Email = "nuevo@mail.com";
db.Update(c);
db.Delete(c);
db.DeleteAll<Contacto>();
```

**Convenciones:** pluraliza la clase para la tabla (`Contacto` → `Contactos`), usa propiedad `Id` como PK.

**Atributos para sobreescribir convenciones:**
```csharp
[Table("mis_contactos")]
class Contacto {
    [Key]          // PK autogenerada (AUTOINCREMENT) — omitida en INSERT
    public int Id { get; set; }

    // [ExplicitKey]  // PK asignada por nosotros — incluida en INSERT

    [Computed]     // ignorada en INSERT/UPDATE (columna calculada)
    public string NombreCompleto => $"{Nombre} {Apellido}";
}
```

## Filtrar en la base, no en memoria

```csharp
// ✓ Eficiente — solo viajan las filas que coinciden
db.Query<Contacto>(
    "SELECT * FROM Contactos WHERE Nombre > @nombre", new { nombre = "Carlos" });

// ✗ Ineficiente — trae todo a memoria y filtra en C#
db.GetAll<Contacto>().Where(c => c.Nombre > "Carlos");
```

## Ejemplo completo

```csharp
using DbConnection db = new SqliteConnection("Data Source=contactos.db");
db.Open();

// Crear tabla (ADO.NET puro para DDL)
using var cmd = db.CreateCommand();
cmd.CommandText = """
    CREATE TABLE IF NOT EXISTS Contactos (
        Id INTEGER PRIMARY KEY AUTOINCREMENT,
        Nombre TEXT NOT NULL, Apellido TEXT NOT NULL, Email TEXT NOT NULL
    )""";
cmd.ExecuteNonQuery();

// Insertar con Dapper.Contrib
db.Insert(new Contacto { Nombre = "Ana", Apellido = "García", Email = "ana@mail.com" });
db.Insert(new Contacto { Nombre = "Beto", Apellido = "López", Email = "beto@mail.com" });

// Listar
foreach (var c in db.GetAll<Contacto>())
    Console.WriteLine($"{c.Id}: {c.Nombre} {c.Apellido}");

// Consultar con Dapper puro (filtrado en servidor)
var resultado = db.Query<Contacto>(
    "SELECT * FROM Contactos WHERE Apellido = @ap", new { ap = "García" });

// Contar
int total = db.QuerySingle<int>("SELECT COUNT(*) FROM Contactos");
```

## Cuándo usar qué

| Situación | Herramienta |
|---|---|
| CRUD simple | `Dapper.Contrib` |
| JOINs, filtros, consultas complejas | Dapper puro (`Query<T>`) |
| Control total, sin abstracción | ADO.NET |
| Relaciones, tracking, migraciones | EF Core |

Dapper no reemplaza ADO.NET — lo usa internamente. Entender ADO.NET ayuda a entender qué hace Dapper.

# Variables y manejo de memoria en C#

Una variable es un nombre asociado a un valor que puede cambiar durante la ejecución.

```csharp
int contador = 0;
contador = contador + 1;
contador += 1;
```

`x = x + 1` no es una igualdad matemática: es una asignación. Significa "tomar el valor actual de `x`, sumarle `1` y guardar el resultado en `x`".

## `var` e inferencia de tipos

El tipo se puede declarar de forma explícita o dejar que el compilador lo infiera:

```csharp
int edad = 20;
var edad = 20;        // también int
var precio = 19.99;   // double
var alumnos = new List<string>();
```

`var` **no es tipado dinámico**: el tipo sigue siendo estático y se resuelve en compilación. Solo evita repetirlo cuando ya está claro a partir del valor inicial.

Conviene usar `var` cuando el tipo es evidente. Conviene usar tipo explícito cuando el valor inicial no lo deja claro:

```csharp
var dato = ObtenerResultado(); // ¿qué devuelve esto?
```

## `const` y `readonly`

Ambos expresan inmutabilidad, pero en momentos distintos.

`const` es para valores conocidos en tiempo de compilación:

```csharp
const double Pi = 3.14159;
```

`readonly` es para campos que se fijan en la declaración o en el constructor:

```csharp
class Persona {
    public readonly DateTime FechaCreacion;

    public Persona() {
        this.FechaCreacion = DateTime.Now;
    }
}
```

`DateTime.Now` no se conoce en compilación, así que no podría ser `const`.

## Tipos por valor y por referencia

La distinción central: ¿una variable **contiene** el valor, o **referencia** un lugar donde el valor está guardado?

Tipos por valor copian el dato:

```csharp
int a = 10;
int b = a;
b = 20;
// a sigue valiendo 10
```

Tipos por referencia copian la referencia, no el objeto:

```csharp
int[] numeros = { 1, 2, 3 };
int[] copia = numeros;
copia[0] = 99;

Console.WriteLine(numeros[0]); // 99
```

`copia` y `numeros` apuntan al mismo arreglo. Para obtener una copia independiente hay que crearla explícitamente:

```csharp
int[] copia = (int[])numeros.Clone();
```

Esta distinción aparece en todo: paso de parámetros, asignaciones, comparaciones, colecciones.

## Stack y heap

Como primera aproximación:

- la **stack** se asocia con variables locales y llamadas a métodos: vida corta, administración automática.
- el **heap** se asocia con objetos creados dinámicamente: lo administra el recolector de basura.

La realidad del runtime es más matizada que "valores en stack, objetos en heap". Lo importante no es memorizar internals del CLR, sino entender que no toda la memoria se comporta igual.

## Alcance y tiempo de vida

El **alcance** es dónde se puede usar la variable. El **tiempo de vida** es cuánto existe durante la ejecución.

Una variable declarada en un bloque solo es visible dentro de ese bloque:

```csharp
void Metodo() {
    int x = 10;
    if (true) {
        int y = 20;
        Console.WriteLine(x); // ok
    }
    Console.WriteLine(y); // error: y no existe acá
}
```

Una variable local vive mientras se ejecuta su bloque. Una variable estática vive durante toda la ejecución del programa.

### Sombreado

Una variable de alcance menor puede ocultar a otra de alcance mayor:

```csharp
int x = 10;

void Metodo() {
    int x = 20; // oculta a la externa
    Console.WriteLine(x); // 20
}
```

## "Globales" en C#

C# no tiene variables globales, pero un campo estático se puede usar de forma equivalente:

```csharp
static class Global {
    public static int Contador = 0;
}

Global.Contador++;
```

## Namespaces y clases estáticas

Un `namespace` agrupa tipos relacionados y evita conflictos de nombres:

```csharp
namespace MiAplicacion.Utilidades;
```

Una clase estática agrupa miembros utilitarios que no dependen de una instancia:

```csharp
static class ConstantesDelSistema {
    public const int EdadMinima = 18;
    public static readonly string Bienvenida = "Bienvenido";
}
```

Una clase estática gigantesca con todo adentro suele ser mala idea: organizar no es amontonar prolijamente.

## Resumen

| Necesidad | Herramienta |
|---|---|
| Declarar sin repetir el tipo | `var` |
| Valor fijo conocido en compilación | `const` |
| Valor fijado al construir el objeto | `readonly` |
| Copiar el dato | tipo por valor |
| Copiar la referencia al objeto | tipo por referencia |
| Saber dónde se usa una variable | alcance |
| Saber cuánto existe una variable | tiempo de vida |
| Agrupar tipos relacionados | `namespace` |
| Agrupar utilidades sin instancia | `static class` |

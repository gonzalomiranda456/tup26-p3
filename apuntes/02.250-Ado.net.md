# ADO.NET en C# — Referencia rápida

ADO.NET es la capa de bajo nivel para acceder a bases de datos desde .NET. No es un ORM: el SQL lo escribimos nosotros, ADO.NET lo ejecuta.

```
Aplicación C# → ADO.NET → Provider (SqliteConnection, NpgsqlConnection, etc.) → Base de datos
```

## Setup SQLite

```bash
dotnet add package Microsoft.Data.Sqlite
```

```csharp
using Microsoft.Data.Sqlite;
using var conn = new SqliteConnection("Data Source=miapp.db");
conn.Open();
```

`using var` garantiza que la conexión se cierre aunque haya excepciones.

## Los tres métodos de ejecución

```csharp
using var cmd = conn.CreateCommand();
cmd.CommandText = "...";

// Sin filas de resultado (CREATE, INSERT, UPDATE, DELETE)
int filasAfectadas = cmd.ExecuteNonQuery();

// Un único valor (COUNT, SUM, MAX...)
object? valor = cmd.ExecuteScalar();
int count = Convert.ToInt32(cmd.ExecuteScalar());

// Muchas filas (SELECT)
using var reader = cmd.ExecuteReader();
while (reader.Read()) { /* leer columnas */ }
```

## Parámetros — siempre, nunca concatenar

```csharp
// ✗ Vulnerable a SQL injection
cmd.CommandText = $"SELECT * FROM Contactos WHERE Nombre = '{nombre}'";

// ✓ Seguro
cmd.CommandText = "SELECT * FROM Contactos WHERE Nombre = $nombre";
cmd.Parameters.AddWithValue("$nombre", nombre);
```

## CRUD completo

**CREATE TABLE:**
```csharp
cmd.CommandText = """
    CREATE TABLE IF NOT EXISTS Contactos (
        Id      INTEGER PRIMARY KEY AUTOINCREMENT,
        Nombre  TEXT NOT NULL,
        Email   TEXT
    )""";
cmd.ExecuteNonQuery();
```

**INSERT:**
```csharp
cmd.CommandText = "INSERT INTO Contactos (Nombre, Email) VALUES ($n, $e); SELECT last_insert_rowid();";
cmd.Parameters.AddWithValue("$n", contacto.Nombre);
cmd.Parameters.AddWithValue("$e", contacto.Email ?? (object)DBNull.Value);
int nuevoId = Convert.ToInt32(cmd.ExecuteScalar());
```

**SELECT → objetos:**
```csharp
cmd.CommandText = "SELECT Id, Nombre, Email FROM Contactos ORDER BY Nombre";
using var reader = cmd.ExecuteReader();
var lista = new List<Contacto>();
while (reader.Read()) {
    lista.Add(new Contacto {
        Id     = reader.GetInt32(0),
        Nombre = reader.GetString(1),
        Email  = reader.IsDBNull(2) ? null : reader.GetString(2)
    });
}
```

**UPDATE / DELETE:**
```csharp
cmd.CommandText = "UPDATE Contactos SET Nombre=$n WHERE Id=$id";
cmd.Parameters.AddWithValue("$n", contacto.Nombre);
cmd.Parameters.AddWithValue("$id", contacto.Id);
cmd.ExecuteNonQuery();

cmd.CommandText = "DELETE FROM Contactos WHERE Id=$id";
cmd.Parameters.Clear();
cmd.Parameters.AddWithValue("$id", id);
cmd.ExecuteNonQuery();
```

⚠️ Sin `WHERE` en UPDATE/DELETE → se modifican **todas** las filas.

## NULL en ADO.NET

```csharp
// Al leer
string? email = reader.IsDBNull(2) ? null : reader.GetString(2);

// Al escribir
cmd.Parameters.AddWithValue("$email", contacto.Email ?? (object)DBNull.Value);
```

## Transacciones

```csharp
using var tx = conn.BeginTransaction();
try {
    cmd.Transaction = tx;
    // múltiples operaciones
    tx.Commit();
} catch {
    tx.Rollback();
    throw;
}
```

## Cambiar de base de datos

Solo cambia el provider y el connection string. El código ADO.NET (`cmd.ExecuteNonQuery()`, `reader.Read()`, etc.) no cambia:

| Base        | Provider                    | Connection string                           |
|-------------|-----------------------------|---------------------------------------------|
| SQLite      | `Microsoft.Data.Sqlite`     | `Data Source=app.db`                        |
| PostgreSQL  | `Npgsql`                    | `Host=localhost;Database=midb;Username=...` |
| SQL Server  | `Microsoft.Data.SqlClient`  | `Server=...;Database=...;...`               |

## ADO.NET vs Dapper vs EF Core

| Concepto              | ADO.NET                  | Dapper                    | EF Core                          |
|-----------------------|--------------------------|---------------------------|----------------------------------|
| SQL                   | Manual                   | Manual                    | Generado                         |
| Mapeo filas→objetos   | Manual                   | Automático                | Automático                       |
| Tracking              | No                       | No                        | Sí                               |
| Cuándo usarlo         | Aprender / control total | SQL propio + menos código | Proyectos grandes con relaciones |

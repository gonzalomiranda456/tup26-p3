# Motor de Planilla de Cálculo en C#

## Introducción

Una planilla de cálculo es uno de los programas más complejos que existen en términos de la cantidad de problemas que resuelve simultáneamente. Construir uno desde cero requiere aplicar casi todas las herramientas que hemos visto en el curso: tipos algebraicos, pattern matching, parsers recursivos, grafos, algoritmos de propagación y diseño orientado a objetos.

Este tutorial construye un motor completo paso a paso. Al final tendremos:

- Un **modelo de datos** que representa la planilla y sus celdas
- Un **lexer y parser** que convierten texto en un Árbol de Sintaxis Abstracta (AST)
- Un **evaluador** que calcula el valor de cada celda
- Un **motor reactivo** que propaga automáticamente los cambios a través del grafo de dependencias
- Detección de **referencias circulares**
- Soporte para **inserción de filas y columnas** con referencias relativas
- **Serialización** para leer y escribir la planilla en disco

## 1. Arquitectura general

El sistema tiene cuatro capas que se comunican en una sola dirección:

```
┌─────────────────────────────────────────────────────┐
│                    Planilla                         │
│  Matriz de celdas + API pública (Set, Insert, Save) │
└───────────────────┬─────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────────┐
│               Motor Reactivo                        │
│  Grafo de dependencias + propagación + Kahn         │
└───────────────────┬─────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────────┐
│                 Evaluador                           │
│  Recorre el AST y produce un CellValue              │
└───────────────────┬─────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────────────────┐
│              Lexer + Parser (RDP)                   │
│  Texto → Tokens → AST                               │
└─────────────────────────────────────────────────────┘
```

El flujo al actualizar una celda es:

```
Usuario ingresa texto
    → Parser convierte texto en AST
    → AST se guarda en la celda (para recalcular sin re-parsear)
    → Evaluador calcula el valor a partir del AST
    → Motor reactivo propaga el cambio a las celdas dependientes
    → Cada dependiente se re-evalúa en orden topológico
    → Los ciclos se detectan y marcan como #circular
```

## 2. Modelo de datos

### 2.1 Direcciones de celda

Las direcciones tienen dos representaciones. La dirección **absoluta** identifica una celda en la grilla. La referencia **relativa** almacena el desplazamiento desde la celda que contiene la fórmula — esto es lo que se guarda en el AST para que la inserción de filas y columnas funcione correctamente.

```csharp
// Dirección absoluta — columna 0–25 (A–Z), fila 0–999 (1–1000)
record struct CellAddress(int Col, int Row) {
    public static readonly int MaxCol = 26;
    public static readonly int MaxRow = 1000;

    public bool IsValid =>
        Col >= 0 && Col < MaxCol && Row >= 0 && Row < MaxRow;

    // Parsea "A3" → CellAddress(0, 2)
    public static bool TryParse(string s, out CellAddress addr) {
        addr = default;
        if (s.Length < 2) return false;

        char colChar = char.ToUpper(s[0]);
        if (colChar < 'A' || colChar > 'Z') return false;
        if (!int.TryParse(s[1..], out int row) || row < 1 || row > MaxRow) return false;

        addr = new CellAddress(colChar - 'A', row - 1);
        return true;
    }

    public override string ToString() => $"{(char)('A' + Col)}{Row + 1}";
}

// Referencia relativa — desplazamiento desde la celda origen
record struct RelativeRef(int DeltaCol, int DeltaRow) {
    // Calcula la dirección absoluta a partir de la celda origen
    public CellAddress Resolve(CellAddress origin) =>
        new(origin.Col + DeltaCol, origin.Row + DeltaRow);

    // Crea la referencia relativa desde una celda origen hacia un destino
    public static RelativeRef From(CellAddress origin, CellAddress target) =>
        new(target.Col - origin.Col, target.Row - origin.Row);
}
```

> **Por qué referencias relativas:** cuando se inserta una fila, las celdas se desplazan. Si las referencias se guardaran como direcciones absolutas habría que actualizar todas las fórmulas de la planilla. Con referencias relativas, la relación espacial entre la celda y su referencia se mantiene intacta — solo se necesitan ajustes puntuales en los bordes de inserción.

### 2.2 Valores de celda

Una celda puede contener un número, una cadena, o un error. Este es el tipo algebraico central del sistema:

```csharp
abstract record CellValue;

record NumberVal(double Number) : CellValue {
    public override string ToString() => Number.ToString("G");
}

record StringVal(string Text) : CellValue {
    public override string ToString() => Text;
}

record ErrorVal(string Code) : CellValue {
    public override string ToString() => Code;

    public static readonly ErrorVal Error    = new("#error");
    public static readonly ErrorVal Circular = new("#circular");
    public static readonly ErrorVal Ref      = new("#ref");     // referencia fuera de rango
}
```

### 2.3 La celda

Cada celda guarda cuatro cosas: el texto original del usuario, el AST compilado, el valor calculado, y su posición en el grafo de dependencias.

```csharp
class Cell {
    public CellAddress Address { get; }

    // Texto tal como lo ingresó el usuario — nunca se modifica
    public string RawInput { get; private set; } = string.Empty;

    // AST generado por el parser — null si la entrada es un literal simple
    public Expr? CompiledExpr { get; private set; }

    // Resultado de la evaluación
    public CellValue Value { get; private set; } = StringVal.Empty;

    // Celdas de las que depende esta celda (la formula las referencia)
    public HashSet<CellAddress> Dependencies { get; } = [];

    // Celdas que dependen de esta celda (grafo inverso)
    public HashSet<CellAddress> Dependents { get; } = [];

    public Cell(CellAddress address) {
        Address = address;
    }

    public void Update(string rawInput, Expr? compiled, CellValue value, IEnumerable<CellAddress> deps) {
        RawInput     = rawInput;
        CompiledExpr = compiled;
        Value        = value;
        Dependencies.Clear();
        foreach (var dep in deps) Dependencies.Add(dep);
    }

    public void SetValue(CellValue value) {
        Value = value;
    }

    public override string ToString() => $"{Address}: {Value}";
}

// Extensión útil
static class StringValExt {
    public static readonly StringVal Empty = new(string.Empty);
}
```

## 3. El Árbol de Sintaxis Abstracta (AST)

El AST es la representación estructurada de una expresión. Una vez construido el árbol no hace falta volver al texto original para evaluar o propagar cambios.

### 3.1 Los nodos

```csharp
// Nodo base — todos los nodos del árbol son Expr
abstract record Expr;

// ── Literales ──────────────────────────────────────────────────────────────
record NumberLit(double Value) : Expr;

record StringLit(string Value) : Expr;

// ── Referencias ───────────────────────────────────────────────────────────
// La referencia se guarda como desplazamiento relativo desde la celda origen.
// La dirección original ("A3") se guarda solo para mostrar en UI.
record CellRefExpr(RelativeRef Ref, string OriginalText) : Expr;

// ── Operaciones ───────────────────────────────────────────────────────────
record BinaryExpr(BinaryOp Op, Expr Left, Expr Right) : Expr;

record UnaryExpr(UnaryOp Op, Expr Operand) : Expr;

// ── Llamadas a función ────────────────────────────────────────────────────
// Los argumentos son Expr sin evaluar — el evaluador decide cuándo evaluarlos.
// Esto es lo que permite que si() sea lazy.
record FunctionCall(string Name, List<Expr> Args) : Expr;

// ── Enumeraciones de operadores ───────────────────────────────────────────
enum BinaryOp {
    Add, Sub, Mul, Div,           // aritméticos
    Eq, Neq, Lt, Gt, Leq, Geq,   // comparación → producen 0.0 o 1.0
    Concat,                        // & para cadenas
}

enum UnaryOp { Negate }
```

### 3.2 Ejemplo de AST

La fórmula `=A1 * (B2 + 3)` en una celda en `C5` se representa así:

```
BinaryExpr(Mul,
    CellRefExpr(RelativeRef(-2, -4), "A1"),   // A1 relativo a C5 = (-2,-4)
    BinaryExpr(Add,
        CellRefExpr(RelativeRef(-1, -3), "B2"),
        NumberLit(3)
    )
)
```

Cuando se inserta una fila, solo cambian los `RelativeRef`; el árbol en sí permanece igual.

## 4. El analizador léxico (Lexer)

El lexer convierte el texto de entrada en una secuencia de tokens. El parser luego consume esos tokens para construir el AST.

### 4.1 Tokens

```csharp
enum TokenKind {
    Number, String, CellRef, Ident,
    Plus, Minus, Star, Slash, Ampersand,
    LParen, RParen, Comma,
    Eq, Neq, Lt, Gt, Leq, Geq,
    Eof,
}

record Token(TokenKind Kind, string Text, double NumValue = 0) {
    public static readonly Token Eof = new(TokenKind.Eof, string.Empty);
    public override string ToString() => $"{Kind}({Text})";
}
```

### 4.2 El Lexer

```csharp
class Lexer {
    private readonly string _src;
    private int _pos;

    public Lexer(string src) {
        _src = src;
        _pos = 0;
    }

    public List<Token> Tokenize() {
        var tokens = new List<Token>();
        SkipWhitespace();

        while (_pos < _src.Length) {
            tokens.Add(ReadToken());
            SkipWhitespace();
        }

        tokens.Add(Token.Eof);
        return tokens;
    }

    private Token ReadToken() {
        char c = _src[_pos];

        // Número: inicia con dígito o punto decimal
        if (char.IsDigit(c) || (c == '.' && _pos + 1 < _src.Length && char.IsDigit(_src[_pos + 1])))
            return ReadNumber();

        // Cadena: delimitada por comillas dobles
        if (c == '"') return ReadString();

        // Letra: puede ser referencia a celda (A3) o nombre de función (sin, si)
        if (char.IsLetter(c)) return ReadIdentOrCellRef();

        // Operadores y puntuación
        _pos++;
        return c switch {
            '+' => new Token(TokenKind.Plus,      "+"),
            '-' => new Token(TokenKind.Minus,     "-"),
            '*' => new Token(TokenKind.Star,      "*"),
            '/' => new Token(TokenKind.Slash,     "/"),
            '&' => new Token(TokenKind.Ampersand, "&"),
            '(' => new Token(TokenKind.LParen,    "("),
            ')' => new Token(TokenKind.RParen,    ")"),
            ',' => new Token(TokenKind.Comma,     ","),
            '=' => new Token(TokenKind.Eq,        "="),
            '<' => ReadAngle(),
            '>' => ReadAngle2(),
            _   => throw new LexerException($"Carácter inesperado: '{c}' en posición {_pos - 1}"),
        };
    }

    private Token ReadNumber() {
        int start = _pos;
        while (_pos < _src.Length && (char.IsDigit(_src[_pos]) || _src[_pos] == '.'))
            _pos++;

        // Notación científica: 1e10, 1.5E-3
        if (_pos < _src.Length && (_src[_pos] == 'e' || _src[_pos] == 'E')) {
            _pos++;
            if (_pos < _src.Length && (_src[_pos] == '+' || _src[_pos] == '-')) _pos++;
            while (_pos < _src.Length && char.IsDigit(_src[_pos])) _pos++;
        }

        string text = _src[start.._pos];
        if (!double.TryParse(text, System.Globalization.NumberStyles.Float,
                             System.Globalization.CultureInfo.InvariantCulture, out double value))
            throw new LexerException($"Número inválido: {text}");

        return new Token(TokenKind.Number, text, value);
    }

    private Token ReadString() {
        _pos++;  // consumir la comilla de apertura
        int start = _pos;
        while (_pos < _src.Length && _src[_pos] != '"') {
            if (_src[_pos] == '\\') _pos++;  // escape simple
            _pos++;
        }
        string content = _src[start.._pos];
        if (_pos < _src.Length) _pos++;  // consumir comilla de cierre
        return new Token(TokenKind.String, content);
    }

    private Token ReadIdentOrCellRef() {
        int start = _pos;
        while (_pos < _src.Length && char.IsLetter(_src[_pos])) _pos++;

        string letters = _src[start.._pos];

        // Referencia a celda: una o más letras seguidas de dígitos (A3, BC12)
        if (_pos < _src.Length && char.IsDigit(_src[_pos])) {
            int numStart = _pos;
            while (_pos < _src.Length && char.IsDigit(_src[_pos])) _pos++;
            string fullRef = _src[start.._pos];

            // Verificar que es una dirección válida de una sola columna (A–Z)
            if (letters.Length == 1 && CellAddress.TryParse(fullRef, out _))
                return new Token(TokenKind.CellRef, fullRef.ToUpper());

            // Si no es dirección válida, retroceder y tratar como identificador
            _pos = numStart;
        }

        return new Token(TokenKind.Ident, letters.ToLower());
    }

    private Token ReadAngle() {
        // Ya consumimos '<'
        if (_pos < _src.Length && _src[_pos] == '=') { _pos++; return new Token(TokenKind.Leq, "<="); }
        if (_pos < _src.Length && _src[_pos] == '>') { _pos++; return new Token(TokenKind.Neq, "<>"); }
        return new Token(TokenKind.Lt, "<");
    }

    private Token ReadAngle2() {
        // Ya consumimos '>'
        if (_pos < _src.Length && _src[_pos] == '=') { _pos++; return new Token(TokenKind.Geq, ">="); }
        return new Token(TokenKind.Gt, ">");
    }

    private void SkipWhitespace() {
        while (_pos < _src.Length && char.IsWhiteSpace(_src[_pos])) _pos++;
    }
}

class LexerException(string message) : Exception(message);
```

## 5. El analizador sintáctico (Parser)

El parser implementa un **Analizador Descendente Recursivo (RDP)**. Cada regla de la gramática se convierte en un método. Los métodos se llaman recursivamente para construir el AST.

### 5.1 La gramática

```
expr          := comparison
comparison    := concat ( comp_op concat )*
comp_op       := '=' | '<>' | '<' | '>' | '<=' | '>='
concat        := addition ( '&' addition )*
addition      := multiplication ( ('+' | '-') multiplication )*
multiplication := unary ( ('*' | '/') unary )*
unary         := '-' unary | primary
primary       := NUMBER
               | STRING
               | CELLREF
               | IDENT '(' arglist ')'
               | '(' expr ')'
arglist       := ε | expr (',' expr)*
```

La gramática codifica la **precedencia** de operadores: los operadores más profundos en el árbol (multiplicación) tienen mayor precedencia que los más superficiales (comparación).

### 5.2 El Parser

```csharp
class Parser {
    private readonly List<Token> _tokens;
    private int _pos;
    private readonly CellAddress _origin;   // celda donde vive la fórmula

    public Parser(List<Token> tokens, CellAddress origin) {
        _tokens = tokens;
        _pos    = 0;
        _origin = origin;
    }

    // ── API pública ────────────────────────────────────────────────────────

    // Parsea la expresión completa y retorna el AST + referencias encontradas
    public (Expr Ast, List<CellAddress> References) ParseExpr() {
        var refs = new List<CellAddress>();
        var ast  = Expr_(refs);
        Expect(TokenKind.Eof);
        return (ast, refs);
    }

    // ── Métodos de la gramática ────────────────────────────────────────────

    private Expr Expr_(List<CellAddress> refs) => Comparison(refs);

    private Expr Comparison(List<CellAddress> refs) {
        var left = Concat(refs);

        while (Current.Kind is TokenKind.Eq or TokenKind.Neq
                            or TokenKind.Lt or TokenKind.Gt
                            or TokenKind.Leq or TokenKind.Geq) {
            var op = Current.Kind switch {
                TokenKind.Eq  => BinaryOp.Eq,
                TokenKind.Neq => BinaryOp.Neq,
                TokenKind.Lt  => BinaryOp.Lt,
                TokenKind.Gt  => BinaryOp.Gt,
                TokenKind.Leq => BinaryOp.Leq,
                TokenKind.Geq => BinaryOp.Geq,
                _             => throw new InvalidOperationException(),
            };
            Advance();
            var right = Concat(refs);
            left = new BinaryExpr(op, left, right);
        }

        return left;
    }

    private Expr Concat(List<CellAddress> refs) {
        var left = Addition(refs);

        while (Current.Kind == TokenKind.Ampersand) {
            Advance();
            var right = Addition(refs);
            left = new BinaryExpr(BinaryOp.Concat, left, right);
        }

        return left;
    }

    private Expr Addition(List<CellAddress> refs) {
        var left = Multiplication(refs);

        while (Current.Kind is TokenKind.Plus or TokenKind.Minus) {
            var op = Current.Kind == TokenKind.Plus ? BinaryOp.Add : BinaryOp.Sub;
            Advance();
            var right = Multiplication(refs);
            left = new BinaryExpr(op, left, right);
        }

        return left;
    }

    private Expr Multiplication(List<CellAddress> refs) {
        var left = Unary(refs);

        while (Current.Kind is TokenKind.Star or TokenKind.Slash) {
            var op = Current.Kind == TokenKind.Star ? BinaryOp.Mul : BinaryOp.Div;
            Advance();
            var right = Unary(refs);
            left = new BinaryExpr(op, left, right);
        }

        return left;
    }

    private Expr Unary(List<CellAddress> refs) {
        if (Current.Kind == TokenKind.Minus) {
            Advance();
            return new UnaryExpr(UnaryOp.Negate, Unary(refs));
        }
        return Primary(refs);
    }

    private Expr Primary(List<CellAddress> refs) {
        var tok = Current;

        switch (tok.Kind) {
            case TokenKind.Number:
                Advance();
                return new NumberLit(tok.NumValue);

            case TokenKind.String:
                Advance();
                return new StringLit(tok.Text);

            case TokenKind.CellRef: {
                Advance();
                CellAddress.TryParse(tok.Text, out var target);
                refs.Add(target);
                var rel = RelativeRef.From(_origin, target);
                return new CellRefExpr(rel, tok.Text.ToUpper());
            }

            case TokenKind.Ident: {
                string name = tok.Text;
                Advance();
                Expect(TokenKind.LParen);
                var args = ParseArgList(refs);
                Expect(TokenKind.RParen);
                return new FunctionCall(name, args);
            }

            case TokenKind.LParen: {
                Advance();
                var inner = Expr_(refs);
                Expect(TokenKind.RParen);
                return inner;
            }

            default:
                throw new ParseException($"Se esperaba un valor o expresión, se encontró: {tok}");
        }
    }

    private List<Expr> ParseArgList(List<CellAddress> refs) {
        var args = new List<Expr>();
        if (Current.Kind == TokenKind.RParen) return args;   // lista vacía

        args.Add(Expr_(refs));
        while (Current.Kind == TokenKind.Comma) {
            Advance();
            args.Add(Expr_(refs));
        }
        return args;
    }

    // ── Helpers ────────────────────────────────────────────────────────────

    private Token Current => _pos < _tokens.Count ? _tokens[_pos] : Token.Eof;

    private Token Advance() {
        var t = Current;
        _pos++;
        return t;
    }

    private void Expect(TokenKind kind) {
        if (Current.Kind != kind)
            throw new ParseException($"Se esperaba {kind}, se encontró {Current}");
        Advance();
    }
}

class ParseException(string message) : Exception(message);
```

### 5.3 El punto de entrada del compilador

```csharp
static class CellCompiler {
    // Parsea el texto ingresado por el usuario y devuelve el AST + dependencias.
    // Retorna null si la entrada es un literal simple (sin AST necesario).
    public static (Expr? Ast, List<CellAddress> Deps, CellValue? LiteralValue)
        Compile(string input, CellAddress origin) {

        if (string.IsNullOrWhiteSpace(input))
            return (null, [], StringVal.Empty);

        // Cadena literal: "Hola mundo
        if (input.StartsWith('"')) {
            string content = input.Length > 1 ? input[1..] : string.Empty;
            return (null, [], new StringVal(content));
        }

        // Número literal: 42, 3.14, 1e6
        if (char.IsDigit(input[0])) {
            if (double.TryParse(input, System.Globalization.NumberStyles.Float,
                                System.Globalization.CultureInfo.InvariantCulture, out double n))
                return (null, [], new NumberVal(n));
            return (null, [], ErrorVal.Error);
        }

        // Referencia directa a celda: A3
        if (input.Length >= 2 && char.IsLetter(input[0]) && char.IsDigit(input[1])) {
            // Tratar como fórmula "=A3"
            input = "=" + input;
        }

        // Fórmula: =1+A3*sin(B2)
        if (input.StartsWith('=')) {
            try {
                var lexer  = new Lexer(input[1..]);   // saltear el '='
                var tokens = lexer.Tokenize();
                var parser = new Parser(tokens, origin);
                var (ast, refs) = parser.ParseExpr();
                return (ast, refs, null);
            }
            catch (Exception ex) when (ex is LexerException or ParseException) {
                return (null, [], ErrorVal.Error);
            }
        }

        // Texto plano sin prefijo especial
        return (null, [], new StringVal(input));
    }
}
```

## 6. El evaluador

El evaluador recorre el AST con pattern matching y produce un `CellValue`. El parámetro `visiting` es el conjunto de celdas actualmente en la pila de evaluación — permite detectar referencias circulares durante la evaluación directa.

### 6.1 Coerción de tipos

Las fórmulas mezclan tipos libremente. La coerción es automática según el contexto:

```csharp
static class Coerce {
    // Convertir a número — si es cadena, parsear; si es error, propagar
    public static CellValue ToNumber(CellValue v) => v switch {
        NumberVal _                 => v,
        StringVal(var s) when double.TryParse(s,
            System.Globalization.NumberStyles.Float,
            System.Globalization.CultureInfo.InvariantCulture, out double n)
                                    => new NumberVal(n),
        StringVal _                 => ErrorVal.Error,   // cadena no numérica
        ErrorVal  _                 => v,                // propagar error
        _                           => ErrorVal.Error,
    };

    // Convertir a cadena — los números se formatean, los errores se propagan
    public static CellValue ToString_(CellValue v) => v switch {
        StringVal _     => v,
        NumberVal(var n) => new StringVal(n.ToString("G",
                            System.Globalization.CultureInfo.InvariantCulture)),
        ErrorVal  _     => v,
        _               => ErrorVal.Error,
    };

    // Interpretar como booleano para si() y comparaciones
    public static bool IsTrue(CellValue v) => v switch {
        NumberVal(var n) => n != 0.0,
        StringVal(var s) => !string.IsNullOrEmpty(s),
        ErrorVal  _      => false,
        _                => false,
    };
}
```

### 6.2 El evaluador principal

```csharp
class Evaluator {
    private readonly Spreadsheet _sheet;

    public Evaluator(Spreadsheet sheet) {
        _sheet = sheet;
    }

    public CellValue Evaluate(Expr expr, CellAddress origin, HashSet<CellAddress> visiting) {
        return expr switch {
            NumberLit(var v)           => new NumberVal(v),
            StringLit(var s)           => new StringVal(s),
            CellRefExpr(var rel, _)    => EvalRef(rel, origin, visiting),
            UnaryExpr(var op, var e)   => EvalUnary(op, e, origin, visiting),
            BinaryExpr(var op, var l, var r)
                                       => EvalBinary(op, l, r, origin, visiting),
            FunctionCall(var name, var args)
                                       => EvalFunction(name, args, origin, visiting),
            _                          => ErrorVal.Error,
        };
    }

    // ── Referencia a celda ─────────────────────────────────────────────────
    private CellValue EvalRef(RelativeRef rel, CellAddress origin, HashSet<CellAddress> visiting) {
        var target = rel.Resolve(origin);

        if (!target.IsValid) return ErrorVal.Ref;

        // Detección de circularidad directa: si la celda ya está en la pila
        if (visiting.Contains(target)) return ErrorVal.Circular;

        return _sheet.GetValue(target);
    }

    // ── Operador unario ────────────────────────────────────────────────────
    private CellValue EvalUnary(UnaryOp op, Expr e, CellAddress origin, HashSet<CellAddress> visiting) {
        var val = Evaluate(e, origin, visiting);
        if (val is ErrorVal) return val;

        var num = Coerce.ToNumber(val);
        if (num is ErrorVal) return num;

        return op switch {
            UnaryOp.Negate => new NumberVal(-((NumberVal)num).Number),
            _              => ErrorVal.Error,
        };
    }

    // ── Operadores binarios ────────────────────────────────────────────────
    private CellValue EvalBinary(BinaryOp op, Expr left, Expr right,
                                 CellAddress origin, HashSet<CellAddress> visiting) {
        var l = Evaluate(left,  origin, visiting);
        var r = Evaluate(right, origin, visiting);

        // Propagar errores
        if (l is ErrorVal) return l;
        if (r is ErrorVal) return r;

        // Concatenación de cadenas — operador &
        if (op == BinaryOp.Concat) {
            var ls = Coerce.ToString_(l);
            var rs = Coerce.ToString_(r);
            if (ls is ErrorVal) return ls;
            if (rs is ErrorVal) return rs;
            return new StringVal(((StringVal)ls).Text + ((StringVal)rs).Text);
        }

        // Comparación — opera sobre números o cadenas, produce 0.0 / 1.0
        if (op is BinaryOp.Eq or BinaryOp.Neq or BinaryOp.Lt
                or BinaryOp.Gt or BinaryOp.Leq or BinaryOp.Geq) {
            return EvalComparison(op, l, r);
        }

        // Aritmética — convierte a número automáticamente
        var ln = Coerce.ToNumber(l);
        var rn = Coerce.ToNumber(r);
        if (ln is ErrorVal) return ln;
        if (rn is ErrorVal) return rn;

        double lv = ((NumberVal)ln).Number;
        double rv = ((NumberVal)rn).Number;

        return op switch {
            BinaryOp.Add => new NumberVal(lv + rv),
            BinaryOp.Sub => new NumberVal(lv - rv),
            BinaryOp.Mul => new NumberVal(lv * rv),
            BinaryOp.Div => rv == 0.0 ? ErrorVal.Error : new NumberVal(lv / rv),
            _            => ErrorVal.Error,
        };
    }

    private static CellValue EvalComparison(BinaryOp op, CellValue l, CellValue r) {
        // Si ambos son números, comparar numéricamente
        if (l is NumberVal(var lv) && r is NumberVal(var rv)) {
            bool result = op switch {
                BinaryOp.Eq  => lv == rv,
                BinaryOp.Neq => lv != rv,
                BinaryOp.Lt  => lv <  rv,
                BinaryOp.Gt  => lv >  rv,
                BinaryOp.Leq => lv <= rv,
                BinaryOp.Geq => lv >= rv,
                _            => false,
            };
            return new NumberVal(result ? 1.0 : 0.0);
        }

        // Si alguno es cadena, comparar lexicográficamente
        var ls = Coerce.ToString_(l);
        var rs = Coerce.ToString_(r);
        if (ls is ErrorVal || rs is ErrorVal) return ErrorVal.Error;

        int cmp = string.Compare(((StringVal)ls).Text, ((StringVal)rs).Text,
                                 StringComparison.OrdinalIgnoreCase);
        bool res = op switch {
            BinaryOp.Eq  => cmp == 0,
            BinaryOp.Neq => cmp != 0,
            BinaryOp.Lt  => cmp <  0,
            BinaryOp.Gt  => cmp >  0,
            BinaryOp.Leq => cmp <= 0,
            BinaryOp.Geq => cmp >= 0,
            _            => false,
        };
        return new NumberVal(res ? 1.0 : 0.0);
    }

    // ── Funciones ──────────────────────────────────────────────────────────
    private CellValue EvalFunction(string name, List<Expr> args,
                                   CellAddress origin, HashSet<CellAddress> visiting) {
        // si() evalúa los argumentos de forma lazy
        if (name == "si") return EvalSi(args, origin, visiting);

        // El resto de las funciones evalúan todos sus argumentos primero
        var vals = args.Select(a => Evaluate(a, origin, visiting)).ToList();
        var firstError = vals.OfType<ErrorVal>().FirstOrDefault();
        if (firstError is not null) return firstError;

        return name switch {
            "sin"    => NumericFn(vals, Math.Sin),
            "cos"    => NumericFn(vals, Math.Cos),
            "tan"    => NumericFn(vals, Math.Tan),
            "raiz"   => NumericFn(vals, v => v < 0 ? double.NaN : Math.Sqrt(v)),
            "abs"    => NumericFn(vals, Math.Abs),
            "redond" => Round(vals),
            "concat" => ConcatFn(vals),
            "upper"  => StringFn(vals, s => s.ToUpper()),
            "lower"  => StringFn(vals, s => s.ToLower()),
            "largo"  => LargoFn(vals),
            "valor"  => ValorFn(vals),
            "texto"  => TextoFn(vals),
            _        => ErrorVal.Error,
        };
    }

    // si(<condicion>, <valorSi>, <valorNo>) — evaluación lazy
    private CellValue EvalSi(List<Expr> args, CellAddress origin, HashSet<CellAddress> visiting) {
        if (args.Count != 3) return ErrorVal.Error;

        var condition = Evaluate(args[0], origin, visiting);
        if (condition is ErrorVal) return condition;

        // Solo se evalúa la rama elegida
        return Coerce.IsTrue(condition)
            ? Evaluate(args[1], origin, visiting)
            : Evaluate(args[2], origin, visiting);
    }

    // ── Helpers de funciones ───────────────────────────────────────────────

    private static CellValue NumericFn(List<CellValue> args, Func<double, double> fn) {
        if (args.Count != 1) return ErrorVal.Error;
        var n = Coerce.ToNumber(args[0]);
        if (n is ErrorVal) return n;
        double result = fn(((NumberVal)n).Number);
        return double.IsNaN(result) || double.IsInfinity(result)
            ? ErrorVal.Error
            : new NumberVal(result);
    }

    private static CellValue Round(List<CellValue> args) {
        if (args.Count is < 1 or > 2) return ErrorVal.Error;
        var n = Coerce.ToNumber(args[0]);
        if (n is ErrorVal) return n;
        int decimals = args.Count == 2 ? (int)((NumberVal)Coerce.ToNumber(args[1])).Number : 0;
        return new NumberVal(Math.Round(((NumberVal)n).Number, decimals, MidpointRounding.AwayFromZero));
    }

    private static CellValue ConcatFn(List<CellValue> args) {
        var sb = new System.Text.StringBuilder();
        foreach (var v in args) {
            var s = Coerce.ToString_(v);
            if (s is ErrorVal) return s;
            sb.Append(((StringVal)s).Text);
        }
        return new StringVal(sb.ToString());
    }

    private static CellValue StringFn(List<CellValue> args, Func<string, string> fn) {
        if (args.Count != 1) return ErrorVal.Error;
        var s = Coerce.ToString_(args[0]);
        if (s is ErrorVal) return s;
        return new StringVal(fn(((StringVal)s).Text));
    }

    private static CellValue LargoFn(List<CellValue> args) {
        if (args.Count != 1) return ErrorVal.Error;
        var s = Coerce.ToString_(args[0]);
        if (s is ErrorVal) return s;
        return new NumberVal(((StringVal)s).Text.Length);
    }

    private static CellValue ValorFn(List<CellValue> args) {
        if (args.Count != 1) return ErrorVal.Error;
        return Coerce.ToNumber(args[0]);
    }

    private static CellValue TextoFn(List<CellValue> args) {
        if (args.Count != 1) return ErrorVal.Error;
        return Coerce.ToString_(args[0]);
    }
}
```

## 7. El motor reactivo

El motor reactivo es el componente más complejo. Su responsabilidad es garantizar que, cuando una celda cambia, todas las celdas que dependen de ella (directa o transitivamente) se recalculen en el orden correcto, y que los ciclos sean detectados.

### 7.1 El problema del orden

Supongamos que `C1 = A1 + B1` y `D1 = C1 * 2`. Si `A1` cambia:

```
A1 cambió → necesitan recalcularse: C1, D1
¿En qué orden?
    Opción 1: D1 primero, C1 después → D1 usa el valor viejo de C1 ✗
    Opción 2: C1 primero, D1 después → D1 usa el valor nuevo de C1 ✓
```

El orden correcto es el **orden topológico** del grafo de dependencias: calcular siempre antes las celdas de las que otros dependen.

### 7.2 Detección de circularidad con el algoritmo de Kahn

El algoritmo de Kahn es una forma de ordenamiento topológico que también detecta ciclos naturalmente: si al final del proceso quedan nodos sin procesar, esos nodos forman parte de un ciclo.

```
Algoritmo PropagateChange(celda C cambiada):

1. Recopilar todos los descendientes transitivos de C:
   cellsToUpdate = {C} ∪ TransitiveDependents(C)

2. Construir el subgrafo inducido:
   Para cada X en cellsToUpdate:
       inDegree[X] = cantidad de dependencias de X que también están en cellsToUpdate

3. Inicializar la cola con las celdas sin dependencias pendientes:
   queue = { X ∈ cellsToUpdate : inDegree[X] == 0 }

4. Procesar en orden topológico:
   procesadas = {}
   mientras queue no esté vacía:
       X = desencolar(queue)
       Recalcular(X)
       procesadas.Add(X)
       para cada dependiente D de X en cellsToUpdate:
           inDegree[D] -= 1
           si inDegree[D] == 0: encolar(D)

5. Las celdas restantes están en un ciclo:
   para cada X en cellsToUpdate \ procesadas:
       X.Value = #circular
```

### 7.3 Implementación del motor

```csharp
class ReactiveEngine {
    private readonly Cell[,] _grid;   // referencia a la grilla de la planilla
    private readonly Evaluator _eval;

    public ReactiveEngine(Cell[,] grid, Evaluator eval) {
        _grid = grid;
        _eval = eval;
    }

    // Punto de entrada: se llama cada vez que se actualiza una celda
    public void PropagateChange(CellAddress changed) {
        // Paso 1: recopilar todas las celdas afectadas
        var affected = CollectAffected(changed);

        // Paso 2: calcular in-degree dentro del subconjunto afectado
        var inDegree = new Dictionary<CellAddress, int>();
        foreach (var addr in affected) {
            int count = 0;
            foreach (var dep in GetCell(addr).Dependencies) {
                if (affected.Contains(dep)) count++;
            }
            inDegree[addr] = count;
        }

        // Paso 3: inicializar la cola
        var queue = new Queue<CellAddress>();
        foreach (var (addr, degree) in inDegree) {
            if (degree == 0) queue.Enqueue(addr);
        }

        // Paso 4: procesar en orden topológico
        var processed = new HashSet<CellAddress>();
        while (queue.Count > 0) {
            var current = queue.Dequeue();
            processed.Add(current);

            // Recalcular esta celda
            RecalculateCell(current);

            // Actualizar in-degrees de los dependientes
            foreach (var dependent in GetCell(current).Dependents) {
                if (!affected.Contains(dependent)) continue;
                inDegree[dependent]--;
                if (inDegree[dependent] == 0) queue.Enqueue(dependent);
            }
        }

        // Paso 5: las no procesadas están en un ciclo
        foreach (var addr in affected) {
            if (!processed.Contains(addr)) {
                GetCell(addr).SetValue(ErrorVal.Circular);
            }
        }
    }

    // Recalcular el valor de una celda usando su AST guardado
    private void RecalculateCell(CellAddress addr) {
        var cell = GetCell(addr);

        // Celda sin fórmula: su valor ya fue asignado directamente
        if (cell.CompiledExpr is null) return;

        // El conjunto visiting se usa para detectar circularidad directa
        // (ciclos de longitud 1 que Kahn no detecta automáticamente)
        var visiting = new HashSet<CellAddress> { addr };
        var value    = _eval.Evaluate(cell.CompiledExpr, addr, visiting);
        cell.SetValue(value);
    }

    // Recopilar todos los descendientes transitivos de una celda
    private HashSet<CellAddress> CollectAffected(CellAddress start) {
        var visited = new HashSet<CellAddress>();
        var queue   = new Queue<CellAddress>();
        queue.Enqueue(start);

        while (queue.Count > 0) {
            var current = queue.Dequeue();
            if (!visited.Add(current)) continue;

            foreach (var dep in GetCell(current).Dependents) {
                if (!visited.Contains(dep)) queue.Enqueue(dep);
            }
        }

        return visited;
    }

    private Cell GetCell(CellAddress addr) => _grid[addr.Col, addr.Row];
}
```

## 8. La planilla

La clase `Spreadsheet` une todos los componentes y expone la API pública.

```csharp
class Spreadsheet {
    private readonly Cell[,] _grid;
    private readonly Evaluator _eval;
    private readonly ReactiveEngine _engine;

    public Spreadsheet() {
        _grid   = new Cell[CellAddress.MaxCol, CellAddress.MaxRow];
        _eval   = new Evaluator(this);
        _engine = new ReactiveEngine(_grid, _eval);

        // Inicializar todas las celdas
        for (int c = 0; c < CellAddress.MaxCol; c++) {
            for (int r = 0; r < CellAddress.MaxRow; r++) {
                var addr = new CellAddress(c, r);
                _grid[c, r] = new Cell(addr);
            }
        }
    }

    // ── Operaciones de celda ───────────────────────────────────────────────

    public void SetCell(CellAddress addr, string rawInput) {
        if (!addr.IsValid) return;

        var cell = GetCell(addr);

        // Desconectar del grafo de dependencias anterior
        foreach (var dep in cell.Dependencies) {
            GetCell(dep).Dependents.Remove(addr);
        }

        // Compilar el nuevo input
        var (ast, deps, literalValue) = CellCompiler.Compile(rawInput, addr);

        // Calcular el valor inicial
        CellValue value;
        if (ast is not null) {
            var visiting = new HashSet<CellAddress> { addr };
            value = _eval.Evaluate(ast, addr, visiting);
        }
        else {
            value = literalValue ?? StringVal.Empty;
        }

        // Actualizar la celda
        cell.Update(rawInput, ast, value, deps);

        // Conectar al grafo de dependencias nuevo
        foreach (var dep in deps) {
            if (dep.IsValid) GetCell(dep).Dependents.Add(addr);
        }

        // Propagar el cambio a las celdas dependientes
        _engine.PropagateChange(addr);
    }

    public CellValue GetValue(CellAddress addr) =>
        addr.IsValid ? GetCell(addr).Value : ErrorVal.Ref;

    public Cell GetCell(CellAddress addr) => _grid[addr.Col, addr.Row];

    // ── Inserción de filas y columnas ──────────────────────────────────────

    public void InsertRow(int beforeRow) {
        if (beforeRow < 0 || beforeRow >= CellAddress.MaxRow) return;

        // Mover datos hacia abajo desde la última fila
        for (int row = CellAddress.MaxRow - 1; row > beforeRow; row--) {
            for (int col = 0; col < CellAddress.MaxCol; col++) {
                var src = new CellAddress(col, row - 1);
                var dst = new CellAddress(col, row);
                MoveCell(src, dst);
            }
        }

        // Limpiar la fila insertada
        for (int col = 0; col < CellAddress.MaxCol; col++) {
            var addr = new CellAddress(col, beforeRow);
            _grid[col, beforeRow] = new Cell(addr);
        }

        // Ajustar referencias relativas en toda la planilla
        AdjustRefsAfterRowInsert(beforeRow);
    }

    public void InsertColumn(int beforeCol) {
        if (beforeCol < 0 || beforeCol >= CellAddress.MaxCol) return;

        for (int col = CellAddress.MaxCol - 1; col > beforeCol; col--) {
            for (int row = 0; row < CellAddress.MaxRow; row++) {
                var src = new CellAddress(col - 1, row);
                var dst = new CellAddress(col, row);
                MoveCell(src, dst);
            }
        }

        for (int row = 0; row < CellAddress.MaxRow; row++) {
            var addr = new CellAddress(beforeCol, row);
            _grid[beforeCol, row] = new Cell(addr);
        }

        AdjustRefsAfterColInsert(beforeCol);
    }

    private void MoveCell(CellAddress src, CellAddress dst) {
        var srcCell = _grid[src.Col, src.Row];
        var dstCell = new Cell(dst);

        // Copiar datos — las referencias relativas se ajustan después
        dstCell.Update(srcCell.RawInput, srcCell.CompiledExpr,
                       srcCell.Value, srcCell.Dependencies);

        _grid[dst.Col, dst.Row] = dstCell;

        // Limpiar origen
        _grid[src.Col, src.Row] = new Cell(src);
    }
}
```

## 9. Ajuste de referencias relativas al insertar

Al insertar la fila `R`, ocurren tres casos para cada referencia `(dc, dr)` en una celda en la fila `f` que apunta a la fila `t = f + dr`:

```
Caso A: f < R y t < R → nada cambia (ambos están por encima de la inserción)
Caso B: f < R y t >= R → la celda destino se desplaza, dr debe aumentar en 1
Caso C: f >= R y t < R → la celda origen se desplaza, dr debe disminuir en 1
Caso D: f >= R y t >= R → ambos se desplazan igual, dr no cambia
```

```csharp
// (continuación de la clase Spreadsheet)

private void AdjustRefsAfterRowInsert(int insertedRow) {
    for (int col = 0; col < CellAddress.MaxCol; col++) {
        for (int row = 0; row < CellAddress.MaxRow; row++) {
            var cell = _grid[col, row];
            if (cell.CompiledExpr is null) continue;

            var origin = new CellAddress(col, row);
            var newAst = AdjustRowRefs(cell.CompiledExpr, origin, insertedRow);

            if (!ReferenceEquals(newAst, cell.CompiledExpr)) {
                // Re-extraer dependencias del AST ajustado
                var deps = ExtractDeps(newAst, origin);
                cell.Update(cell.RawInput, newAst, cell.Value, deps);
            }
        }
    }
}

// Recorre el AST y ajusta los DeltaRow de las CellRefExpr
private Expr AdjustRowRefs(Expr expr, CellAddress origin, int insertedRow) {
    return expr switch {
        CellRefExpr(var rel, var text) => AdjustCellRefRow(origin, rel, text, insertedRow),
        BinaryExpr(var op, var l, var r) =>
            new BinaryExpr(op, AdjustRowRefs(l, origin, insertedRow),
                              AdjustRowRefs(r, origin, insertedRow)),
        UnaryExpr(var op, var e) =>
            new UnaryExpr(op, AdjustRowRefs(e, origin, insertedRow)),
        FunctionCall(var name, var args) =>
            new FunctionCall(name, args.Select(a => AdjustRowRefs(a, origin, insertedRow)).ToList()),
        _ => expr,
    };
}

private static Expr AdjustCellRefRow(CellAddress origin, RelativeRef rel,
                                      string text, int insertedRow) {
    int targetRow = origin.Row + rel.DeltaRow;

    int newDelta = rel.DeltaRow;

    // Caso B: origen arriba de la inserción, destino en o debajo → destino se desplazó
    if (origin.Row < insertedRow && targetRow >= insertedRow)
        newDelta++;
    // Caso C: origen en o debajo de la inserción, destino arriba → origen se desplazó
    else if (origin.Row >= insertedRow && targetRow < insertedRow)
        newDelta--;
    // Casos A y D: no cambia

    if (newDelta == rel.DeltaRow) return new CellRefExpr(rel, text);

    return new CellRefExpr(new RelativeRef(rel.DeltaCol, newDelta), text);
}

private void AdjustRefsAfterColInsert(int insertedCol) {
    for (int col = 0; col < CellAddress.MaxCol; col++) {
        for (int row = 0; row < CellAddress.MaxRow; row++) {
            var cell = _grid[col, row];
            if (cell.CompiledExpr is null) continue;

            var origin = new CellAddress(col, row);
            var newAst = AdjustColRefs(cell.CompiledExpr, origin, insertedCol);

            if (!ReferenceEquals(newAst, cell.CompiledExpr)) {
                var deps = ExtractDeps(newAst, origin);
                cell.Update(cell.RawInput, newAst, cell.Value, deps);
            }
        }
    }
}

private Expr AdjustColRefs(Expr expr, CellAddress origin, int insertedCol) {
    return expr switch {
        CellRefExpr(var rel, var text) => AdjustCellRefCol(origin, rel, text, insertedCol),
        BinaryExpr(var op, var l, var r) =>
            new BinaryExpr(op, AdjustColRefs(l, origin, insertedCol),
                              AdjustColRefs(r, origin, insertedCol)),
        UnaryExpr(var op, var e) =>
            new UnaryExpr(op, AdjustColRefs(e, origin, insertedCol)),
        FunctionCall(var name, var args) =>
            new FunctionCall(name, args.Select(a => AdjustColRefs(a, origin, insertedCol)).ToList()),
        _ => expr,
    };
}

private static Expr AdjustCellRefCol(CellAddress origin, RelativeRef rel,
                                      string text, int insertedCol) {
    int targetCol = origin.Col + rel.DeltaCol;

    int newDelta = rel.DeltaCol;

    if (origin.Col < insertedCol && targetCol >= insertedCol)
        newDelta++;
    else if (origin.Col >= insertedCol && targetCol < insertedCol)
        newDelta--;

    if (newDelta == rel.DeltaCol) return new CellRefExpr(rel, text);

    return new CellRefExpr(new RelativeRef(newDelta, rel.DeltaRow), text);
}

// Extraer dependencias de un AST dado
private static List<CellAddress> ExtractDeps(Expr expr, CellAddress origin) {
    var deps = new List<CellAddress>();
    ExtractDepsRec(expr, origin, deps);
    return deps;
}

private static void ExtractDepsRec(Expr expr, CellAddress origin, List<CellAddress> deps) {
    switch (expr) {
        case CellRefExpr(var rel, _):
            deps.Add(rel.Resolve(origin));
            break;
        case BinaryExpr(_, var l, var r):
            ExtractDepsRec(l, origin, deps);
            ExtractDepsRec(r, origin, deps);
            break;
        case UnaryExpr(_, var e):
            ExtractDepsRec(e, origin, deps);
            break;
        case FunctionCall(_, var args):
            foreach (var a in args) ExtractDepsRec(a, origin, deps);
            break;
    }
}
```

```csharp
public static Spreadsheet Load(string path) {

        // Ignorar líneas vacías y comentarios

            continue;
        }

        sheet.SetCell(addr, content);
    }
```

### Traza de propagación para el ejemplo circular

```
SetCell(X1, "=Y1 + 1"):
    X1.CompiledExpr = BinaryExpr(Add, CellRefExpr(Y1), NumberLit(1))
    X1.Dependencies = {Y1}
    Y1.Dependents   = {X1}
    PropagateChange(X1):
        affected = {X1} (Y1 no depende de X1 todavía)
        inDegree = {X1: 0}
        queue = [X1]
        Recalculate(X1) → Y1 vale StringVal(""), 0 + 1 = 1
        X1.Value = 1
        processed = {X1}

SetCell(Y1, "=X1 + 1"):
    Y1.Dependencies = {X1}
    X1.Dependents   = {Y1}
    PropagateChange(Y1):
        affected = {Y1, X1}  (Y1 y X1 están en el subgrafo de Y1)
        inDegree:
            Y1 depende de X1 → X1 está en affected → inDegree[Y1] = 1
            X1 depende de Y1 → Y1 está en affected → inDegree[X1] = 1
        queue = []  (ninguno tiene inDegree 0)
        processed = {}
        Restantes = {Y1, X1} → ambos son #circular ✓
```

## Resumen

```
Capas del sistema:
    ├── Modelo     → CellAddress, RelativeRef, CellValue, Cell
    ├── AST        → Expr jerárquico: NumberLit, StringLit, CellRefExpr,
    │                  BinaryExpr, UnaryExpr, FunctionCall
    ├── Lexer      → texto → tokens
    ├── Parser     → tokens → AST (RDP según la gramática de precedencia)
    ├── Evaluador  → AST → CellValue (con coerción automática)
    ├── Motor      → PropagateChange con algoritmo de Kahn
    └── Planilla   → API pública + serialización + inserción

Decisiones de diseño clave:
    ├── Referencias relativas  → las inserciones no requieren actualizar todo
    ├── AST en la celda        → recalcular sin re-parsear
    ├── Kahn para propagación  → orden correcto + detección de ciclos
    ├── si() lazy              → solo evalúa la rama elegida
    └── Coerción implícita     → strings↔números según contexto

Errores:
    ├── #error    → operación imposible (div/0, sqrt negativo, cadena no numérica)
    ├── #circular → ciclo en el grafo de dependencias
    └── #ref      → dirección fuera de rango (col/fila inválida)
```
